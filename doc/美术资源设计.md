# 百工归位 · Artisania — 美术资源设计清单

> 本文目标：把当前 demo 里**所有需要美术的子窗口与画面元素**逐一列清，供绘制像素 / 国风美术时按图索骥。
> 读完本文你应当能回答：游戏一共有几个窗口、每个窗口要画哪些图、每张图的尺寸 / 风格 / 数量、当前用什么占位、替换到哪里。
>
> **总体风格基调**：中国非遗题材 · 国风水墨 + 像素质感；主色取自 `src/index.css` 的 `:root` 调色板
> （靛蓝 `--indigo`、竹绿 `--bamboo`、朱砂、纸白 `--paper`、墨褐线条 `--line`）。
> **像素世界单格** `TILE = 32px`（见 `src/game/textures.ts`），UI 弹窗为常规网页矢量/位图皆可。
>
> **替换原则**：玩法零改动。Phaser 贴图只动 `game/textures.ts` 的加载映射；
> UI 窗口美术优先用 `<img>` 挂载真实 PNG，少量弹窗再用 CSS 背景兜底。真实素材建议放 `public/assets/`。
>
> **交接入口**：首次接手的图像组请先读 [图像组与代码组交接总览.md](图像组与代码组交接总览.md)，再按本清单和全地区规格拆分资产。
>
> **全地区口径**：地区地图、人物、建筑和交互点位的详细规格见 [全地区美术设计规格.md](全地区美术设计规格.md)。本轮按 11 个大地区一次性整理到可绘制、可验收的完整规格。

## 1. 调色与风格规范（绘制前先读）

- **像素世界**：`TILE=32px`。地砖/路面/水面为 1×1 格；建筑（作坊/产业点）当前按 3×3 格脚印摆放，主图统一 96×88；牌坊为 80×80。
  角色/NPC 为 1 格内的小人（约 16~24px 居中，留脚部阴影空间）。像素硬边、低饱和国风配色。
- **UI 弹窗**：跟随 `index.css` 既有视觉（纸白底、墨褐描边、朱砂强调、靛蓝标题）。图标建议 24~48px、
  描边风或单色剪影，避免与像素世界违和。
- **地区配色驱动**：地面/街道色由地区 `terrain.palette` 前两色运行时 `setTint`，所以**地砖底图请画成白/浅灰**
  （`TEX.ground` 即白底），让着色生效；不要把颜色画死。

## 2. 游戏世界（Phaser `StreetScene`）——核心像素资源

> `image_test` 分支已建立第一版清明上河图风格资产管线：`tools/build_qingming_assets.py` 先生成绿底源图到 `public/assets/art_sources/green_sources/`，再用色键抠像输出透明 PNG 到 `public/assets/game/`，并在 `public/assets/art_sources/` 输出透明底与绿底资产总览图。当前第二轮参考“精细像素资源表”的笔触、暗边、屋檐、摊具和低饱和古风配色重绘，但不直接使用参考图原文件。运行时由 `src/game/textures.ts` 的 `preloadArtTextures(scene)` 先加载 PNG，再由 `generatePlaceholderTextures(scene)` 兜底补缺失纹理。替换真实图时保持 `TEX` 键不变即可。

### 2.1 地面 / 地貌瓦片（1×1 格 = 32×32）

| 纹理键 `TEX.*` | 用途 | 占位 | 待绘要点 |
|---|---|---|---|

| `groundMoss` / `groundStone` / `groundSoil` | 苔地 / 青石 / 泥土地 | 已生成 PNG | 用于街角、铺装区和非道路空地，颜色需与通用地面同一低饱和体系 |

| `roadVertical` / `roadCross` | 竖向巷道 / 十字路口 | 已生成 PNG | 与 `road` 同材质，保证多街道网络拼接自然 |
| `wall` | 墙体 | 棕 | 砖墙/夯土墙 |
| `water` | 河面 / 海面（**阻挡**） | 蓝波纹 | 江南河水，波光；不可通行 |
| `waterEdgeLeft` / `waterEdgeRight` | 河岸边缘 | 已生成 PNG | 将河流拆成岸线与内部水面，避免整条河像一块蓝色长条 |

| `fence` | 栅栏（地块边界·**阻挡**） | 木栏 | 竹/木栅栏 |
| `tree` | 树木（点缀·**阻挡**） | 绿冠+棕干 | 江南树/松，带阴影 |
| `rock` | 山石（点缀·**阻挡**） | 灰三角 | 太湖石/山岩 |

> **地貌类型** `TerrainKind`（`water`/`mountain`/`coast`/`plain`）决定铺哪些瓦片，见开发者文档 §4.8 模块 1。

1. **每个地区先有整图母版**：江南、巴蜀、岭南、黔滇、荆楚、赣鄱、徽州、京畿、三晋、雪域、西域都要先确定完整土地构图、主色、道路/水路/山路关系和功能建筑分布。

3. **整图优先于瓦片**：先用风格化整图保证土地整体协调，再拆成 tiles、buildings、props、characters 进入地图编辑器和运行时。
4. **已有资源及时同步**：现有 UI、天气、开放世界、西域等资源继续保留，但后续验收要对齐 [全地区美术设计规格.md](全地区美术设计规格.md) 的地区完整度。
5. **最终验收看全局完整性**：11 个地区都具备“整图母版 + 人物表 + 建筑表 + 交互点位 + 缺少资源表”后，才算本轮美术文档整理完成。

## 10.1 地图编辑器素材规范

编辑器入口为 `http://127.0.0.1:5173/?editor=1`。它不是最终运行地图，而是给美术/策划人工摆图、导出 JSON 的工作台。新增素材验收时按以下规则同步：

- 地面、道路、水体：32×32 单格 tile，命名放在 `public/assets/game/tiles/` 或 `public/assets/game/weather/terrain/`，在 `PALETTE` 里登记为 `tile(id, name, category, asset, solid)`。
- 建筑：建议沿用 96×88 主图，对应编辑器默认 3×3 占格；建筑可有碰撞箱，通常不允许同格同层堆叠，以免同一格压两个房屋。
- 道具：按底部中心锚点思路绘制，编辑器可设置 1×2、2×3、3×1 等占格；柳树、茶摊、码头、船等默认允许与地面/道路/水岸组合。
- 单位：48×48、4 行 × 4 帧行走序列帧；编辑器可预览方向、静态帧与行走播放，并导出 `sheetCols/sheetRows/frame/direction/animated`。
- 标记：如玩家出生点、区域出入口、调试标记，导出 `interaction:"marker"`，通常不参与碰撞。

模型装配流程：点击「模型装配器」后从空白画布开始，左侧导入建筑主体、冬季雪顶、季节装饰、街景组件或自定义图片路径；右侧按上层到下层管理图层，逐层调整 X/Y、缩放、透明度和图片路径；中央设置整体模型分类、渲染层、占格、碰撞箱、堆叠规则和交互类型，点击「另存为模型」后会加入对应素材栏。该功能用于解决季节屋顶与常规建筑屋顶不对齐的问题，先导出 `modelDefinitions[]` 供后续运行时接入。

编辑器点击任意地图格子时，右侧「地块堆叠」会列出覆盖该格的对象层，支持选中、删除或插入当前模型。大型建筑、桥体、船只等多格模型的擦除和修正应优先通过这个堆叠列表处理。

导出 JSON schema 为 `artisania-map-editor/v2`。后续若某张图“看起来不对”，先替换 `public/assets/game/...` 下同名 PNG，再进编辑器复核尺寸、碰撞、动画、状态和堆叠，不必改玩法逻辑。
