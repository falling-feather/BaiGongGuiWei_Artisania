# 《百工归位 · Artisania》逐地区内容设计规划

> 版本：v0.3 讨论稿  
> 日期：2026-06-06  
> 目的：把“九州行脚生活模拟”的宏观方向拆成逐地区可落地内容，包括小地区、跳转关系、交互点、人物、工艺、小游戏形式、隐藏数值与古风描述词。  
> 基线：沿用当前 `src/data/regions.ts` 的 11 个大地区与 `src/data/crafts.ts` 的招牌手艺，不推翻现有架构。

---

## 0. 文档定位

上一篇 [开放世界生活模拟规划.md](开放世界生活模拟规划.md) 回答“游戏应该往哪里长”。本文回答“每个地区具体放什么”。

本文不直接改代码，但每个条目都应能在未来拆为数据：

- `RegionContentSpec`：地区内容总表。
- `SubregionContentSpec`：小地区场景、交互点、NPC、出入口。
- `ActivityDef`：非工艺生活交互，如诗词、练剑、烹茶、种田、赶集。
- `CraftProcessDef`：工艺链与小游戏映射。
- `ItemDescriptorRule`：隐藏数值转古风描述词。

## 2. 隐藏数值与描述词体系

玩家不应看到“攻击力 +12”“品质 87 分”。内部可以有量化参数，但外显必须是古风描述。

### 2.1 原料隐藏字段

以矿石为例：

| 隐藏字段 | 含义 | 外显词例 |
|---|---|---|
| `purity` 纯净 | 杂质多少 | 浊、净、莹、澄明 |
| `grain` 纹理 | 纹路稳定 | 散纹、细纹、霜纹、星纹 |
| `hardness` 硬度 | 坚硬程度 | 松脆、沉实、坚劲、铁骨 |
| `resilience` 韧性 | 折断风险 | 易裂、可锻、柔韧、百折 |
| `spirit` 灵韵 | 稀有气质 | 凡料、含光、寒意、异彩 |
| `flaw` 瑕疵 | 负面词条 | 砂眼、暗裂、火毒、脆心 |

### 2.2 成品描述维度

以剑为例，建议分三到四个外显维度：

| 维度 | 含义 | 词阶示例 |
|---|---|---|
| 形制 | 外观、纹理、气质 | 粗朴、清正、霜纹、仙逸、孤寒 |
| 手感 | 重心、趁手、使用感 | 坠手、生涩、趁手、如臂使指 |
| 锋芒 | 开刃与切割 | 钝涩、开锋、寒光、吹毛断发 |
| 韧骨 | 抗裂、耐用 | 脆心、可用、百折、刚柔并济 |

用户提出的“吹弹可破”很有画面感，但它更像“脆薄、易碎”的词，不太适合做锋利度高阶。建议保留为负面或矛盾描述：例如“锋芒极寒，却吹弹可破”，表示刀口薄利但韧性不足。锋利高阶更适合“吹毛断发”“霜刃无声”“断帛不滞”。

### 2.3 描述生成规则

```
隐藏数值 + 工艺步骤 + NPC 协作 + 材料词条
  → 生成维度词
  → 组合成短评
```

示例：

> 此剑形制清正，霜纹隐现，握之趁手；锋芒可断薄帛，唯火候稍急，韧骨未至圆熟。

## 3. 铸剑完整链路样板

龙泉山坊与巴蜀临邛铁场都可接入铸剑链路。

### 3.1 取材

地点：山溪铁矿 / 临邛铁场 / 大冶矿场 / 佛山冶坊。

小游戏：

- `rhythm`：凿矿节奏，节奏越稳，矿石完整度越高。
- `appraise_select`：从多块矿石中辨纹、听声、看色。
- 随机词条：星纹、暗裂、含砂、寒铁、沉铁、火毒。

产出：

- 铁矿实例：`ItemInstance(type=ironOre, origin=longquan, descriptors=[星纹, 暗裂])`。

### 3.2 冶炼

地点：铸剑坊 / 冶铁炉。

小游戏：

- `timing_hold`：控炉温。
- `ratio_mix`：铁矿、炭、助熔材料比例。

隐藏影响：

- 炉温过低：杂质残留。
- 炉温过高：火毒、脆心。
- 配炭得当：硬度上升。
- 反复精炼：纯净上升，工时增加。

产出：

- 铁锭描述：沉实、净铁、火色正、略带砂心。

### 3.3 锻打

地点：龙泉剑炉。

小游戏：

- `rhythm`：敲打节奏与落点。
- `aim_place`：折叠位置与锤击区域。

隐藏影响：

- 敲击次数不是越多越好。次数不足则纹理散；过度锻打则火力损耗、韧性下降。
- NPC 代理铸造更稳定但上限较低；玩家亲自锻造风险更高但可能出名品。
- 铸剑匠协作可提供“稳锤”加成，文人题名可提供“形制/精神”加成。

### 3.4 淬火与研磨

小游戏：

- `timing_hold`：淬火入水时机。
- `drag_path`：研磨开刃路径。
- `repeat_endure`：多轮细磨耐心。

产出描述：

- 形制：朴雅 / 峻直 / 仙逸 / 孤寒。
- 手感：坠手 / 趁手 / 如臂使指。
- 锋芒：开锋 / 寒光 / 吹毛断发。
- 韧骨：易裂 / 可用 / 百折 / 刚柔并济。

## 4. 种田与时间系统样板

种田应作为供应链和生活感的底盘，不应变成另一个独立游戏。

### 4.1 农事循环

```
翻地 → 播种 → 浇灌/施肥 → 除草/防虫 → 成熟 → 收获 → 留种/加工
```

### 4.2 时间影响

| 因素 | 影响 |
|---|---|
| 季节 | 决定可种作物和成熟周期 |
| 天气 | 雨天免浇水，暴雨可能伤苗 |
| 土质 | 影响产量与隐藏词条 |
| 玩家体魄 | 农事消耗 |
| 见闻 | 识别节气和地方种法 |
| NPC 指点 | 解锁更合适的地方作物 |

### 4.3 首批作物

| 作物 | 地区 | 用途 |
|---|---|---|
| 靛草 | 江南 / 黔滇 | 蓝染、蜡染 |
| 桑树 | 江南 / 岭南 / 荆楚 / 西域 | 生丝、织锦 |
| 茶树 | 江南 / 巴蜀 / 黔滇 / 徽州 | 制茶、赠礼、茶会 |
| 竹 | 巴蜀 / 江南 | 竹编、纸伞、藏纸 |
| 苎麻 | 赣鄱 | 夏布 |
| 药草 | 雪域 / 黔滇 / 江南 | 制香、医馆、药膳 |

## 18. 数据落地提醒

### 18.1 不建议把所有内容一次性编码

先建数据结构，再填江南样板。每个地区的内容表可以先作为 Markdown 或 JSON 草案存在，不急着全部接入场景。

### 18.2 建议新增的内容数据形状

```ts
interface RegionContentSpec {
  regionId: string;
  subregions: SubregionContentSpec[];
  routes: RouteSpec[];
  mainNpcs: NpcContentSpec[];
  activities: ActivityDef[];
  crafts: CraftActivityBinding[];
}
```

当前代码已先落地 `routes: RouteSpec[]`、`mainNpcIds` 与 `activityIds`；活动自身已可通过 `ActivityReward.routeIds` 写入路线情报，并把完成记录沉淀到 `completedActivities`；`subregions/mainNpcs/activities/crafts` 的完整内容规格仍需继续从 Markdown 草案迁入数据库。

### 18.3 关键规则

- 工艺规则仍归 `engine`，小游戏只返回 `quality`、`timing`、`mistakeTags` 等结果。
- NPC 文本可以读隐藏词条，但不能直接改资源。
- Phaser 只管“在哪个点触发什么活动”，不要把锻剑品质写进 `StreetScene`。
- 描述词系统应集中管理，避免每个工艺自己乱写一套词。
