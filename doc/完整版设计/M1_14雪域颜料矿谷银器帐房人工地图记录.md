# M1.14 雪域颜料矿谷与银器帐房人工地图记录

日期：2026-06-15

## 节点口径

本轮继续按 [程序大型化协作指引.md](程序大型化协作指引.md) 的 M1 口径推进，固定启用 6 个协作团队做只读审计。切片目标是关闭雪域章节剩余的直接人工地图断点：`xueyu-pigment-valley` 与 `xueyu-silver-tent` 已经有小地区、活动、NPC、产业/工艺数据，但运行时缺人工地图 JSON。

本轮不新增路线 ID，不改变巴蜀 / 西域进入雪域的路线落点；跨区路线仍落 `xueyu-snow-pass`。本轮也不关闭颜料矿谷 / 银器帐房人物回访、材料状态反馈或雪山驿路护商后日谈。

## 本轮完成

1. 新增 `src/data/mapLayouts/xueyu-pigment-valley.json`
   - 颜料矿谷正式进入运行时人工地图管线。
   - 点位包含 `harvest-pigment`、`grind-pigment`、`xy-pigment-valley` 与 `xy-shicai-tong`。

   - 保留巴蜀雪口路与西域驿线跨区 gate；正式迁移仍由街景 gate 触发。

2. 新增 `src/data/mapLayouts/xueyu-silver-tent.json`
   - 银器帐房正式进入运行时人工地图管线。
   - 点位包含 `harvest-silver-ore`、`refine-silver`、`tibetan-silver`、`xy-silver-tent` 与 `xy-baiyinshu`。
   - 区内通道连接 `xueyu-snow-pass`、`xueyu-thangka-court`、`xueyu-pigment-valley`。
   - 不新增本地铜料冶炼，继续保留藏银对跨区铜料供应的需求。

3. 补齐运行守卫
   - `mapLayout.test.ts` 将两张 JSON 纳入 shipped 顺序、关键对象断言和可达性守卫。
   - `priorityMapLayoutEntrypoints.test.ts` 将两张图纳入入口矩阵。
   - `priorityFrozenIds.test.ts` 冻结两张 layout、两条活动、两位 NPC 与 `tibetan-silver`。
   - 为 `tibetan-silver` 新增最小工艺交互规格，为 `xy-silver-tent` 新增活动挑战，避免正式入口只有名字没有玩法钩子。

4. 更新章节契约
   - `chapter-xueyu-thangka-snowpass` 仍保持 `needs-expansion`。
   - 雪域 gap 不再记录 `xueyu-pigment-valley` / `xueyu-silver-tent` 缺人工 JSON。
   - 后续缺口收敛为人物回访、材料状态反馈、雪山驿路护商后日谈和多入口章节 smoke。

## 剩余缺口

- 颜料矿谷 / 银器帐房人物回访仍未补成长线。
- 石彩童 / 白银叔仍缺读取材料、作品履历和订单反馈的材料状态链。
- 雪山驿路护商结果尚未进一步接入供展续单后日谈。
- 雪域多入口 `smokeBindings` 尚未扩展到矿谷 / 银帐专项入口。
- 本轮不处理最终美术资源、雪地瓦片和精细摆位。
