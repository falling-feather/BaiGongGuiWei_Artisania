# M1.5 黔滇礼银长线收束记录

日期：2026-06-14

## 节点目标

M1.5 关闭 M1 后续动作中“黔滇节事摊位 / 家访 / 协作”这一组长线缺口。目标不是把黔滇直接升为 `chapter-ready`，而是先让 `chapter-qiandian-silver-tea-road` 从一次活动即时订单，推进为可跑通的苗银礼俗长线：茶马会三阶段摊位、收束追单、阿岚礼银家访、复单交付、茶马客回访续单、礼银联作验样。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md)、[M1全地区核心章节入库记录.md](M1全地区核心章节入库记录.md) 与 [M1_4真实订单链补线记录.md](M1_4真实订单链补线记录.md)。

## 协作团队

按 M1 固定团队数继续启用 6 个只读审计团队。

| 团队 | 范围 | 关键结论 |
| --- | --- | --- |
| 1 集成与结构 | reducer、章节 hook、摊位/家访/联作 schema | 不需要改引擎，复用 `ActivityStallDef`、`HomeVisitDef`、`CollabRecipeDef` 即可闭环。 |
| 2 江南 / 巴蜀样板 | 成熟家访、联作、茶马会样板 | 黔滇应复制结构，不复制资源与文案；优先用家访三分支、联作两分支、茶马会三阶段。 |
| 3 岭南 / 黔滇 | 本区内容落点 | `qd-tea-horse-road` 是最适合的长线入口；阿岚需要 `homeVisit`，木骡子可承接订单。 |
| 4 荆楚 / 徽州 / 赣鄱样板 | 藏客回访与摊位追单 | 二跳家访必须以 `homevisit-referral-completed:*` 解锁，并配 blocked flag 防重复。 |
| 5 京畿 / 三晋 / 雪域 / 西域优先级 | M1 后续排序 | 黔滇之后优先转向京畿声望采办、三晋信用违约、雪域供展回访；西域暂可后置。 |
| 6 QA / 文档矩阵 | 验证、冻结 ID、浏览器 smoke | 主验收放在 Vitest/reducer；浏览器只做章节入口短 smoke。 |

1. `src/data/regionContent.ts`
   - `qd-yinniang-alan` 增加 `homeVisit` 功能，`qd-mu-luozi` 增加 `order` 功能。
   - `qd-tea-horse-road.reward.stall` 新增 `银染茶马会货摊`：
     - 3 个阶段：`open-ritual-sample`、`barter-silver-dye`、`ledger-route-close`。
     - 3 类客群：节礼人家、马帮经纪、港口样客。
     - 3 个组合：银染礼样组、茶马来路包、铜银回样台。
     - 3 条策略：先问用途、路引来历账、港口回样。

     - 基础联作读取 `silverOrnament` / `wutongSilver` / `tibetanSilver` / `filigreeOrnament`。

     - 展示礼银样后可选择 `silver-ritual-ledger`，生成 `礼银纹样复单`。

## 当前限制

本轮关闭黔滇“苗银礼俗长线”的第一条完整程序链，但黔滇仍保持 `needs-expansion`：

- `qiandian-dongchuan-copper` 仍缺人工 JSON。
- 东川铜矿缺矿口回访和铜银材料状态反馈。
- 丹青嫂蜡染线已作为联作伙伴接入，但还缺独立家访与后日谈。
- 茶马会已能追单到茶马路书和岭南港样，但岭南港口回访尚未继续展开。

## 当前判断

M1.5 让黔滇从“已有即时订单入口”推进为“已有一条可测试的长线章节骨架”。它已经足够支撑美术组理解：茶马驿道不只是交通点，而是苗银、蜡染、路引、摊位和 NPC 回访交织的章节核心。后续 M1.6 不应再重开黔滇主链，而应按 M1 总表继续补京畿声望采办、三晋信用违约、雪域供展回访，或补人工地图 JSON。
