# M1.18 徽州墨砚深巷人工地图记录

日期：2026-06-15  
分支：`codex/scoped-art-handoff-plan`  
节点来源：`程序大型化协作指引.md` 的 M1 后续地图缺口收束

## 1. 本轮边界

本轮只补 `huizhou-ink-alley` 的人工街景与最小活动挑战，目标是把徽州纸墨章节中“墨砚深巷没有运行入口”的缺口关闭。

不在本轮扩张的内容：

- `huizhou-she-stone` 歙石山坑人工地图。
- 徽商会馆荐藏回访长线、商会信用风险长线。
- `hui-ink` 专属三阶段 `CraftInteractionSpec`、缺陷、返修和订单拒收链。
- 跨区路线默认落点改造。

因此徽州 M1 缺口从“墨砚深巷 / 歙石山坑人工 JSON”收束为“歙石山坑人工 JSON + 商会长线信用内容”。

## 2. 六团队审计结论

M1.18 仍按大型化协作指引要求使用 6 个只读审计团队。六队结论一致：

- 直接补 `huizhou-ink-alley` 比补 `huizhou-she-stone` 更适合当前纸墨章节，因为现有活动、NPC、产业和工艺入口都已在数据层存在。
- 地图尺寸沿用徽州 56x30 人工图规格，保留三横三纵主路和玩家出生点。
- JSON 必须落入 `harvest-pine-soot`、`make-ink`、`hui-ink`、`hz-ink-workshop`、`hz-cheng-moshou`。
- 子区 gate 使用 `interaction: "gate"` + `runtimeInteraction: "subregionGate"`，跨区 gate 只指向地区 id。
- `route-jiangnan-huizhou-paper` 仍默认落到 `huizhou-paper-valley`，`route-ganpo-huizhou-merchant` 仍默认落到 `huizhou-merchant-hall`，不能因为墨巷接入而改变路线落点。
- `hui-ink` 本轮只冻结地图入口，后续补专属工艺交互时再纳入 craft 冻结清单。

## 3. 程序落点

新增并注册：

- `src/data/mapLayouts/huizhou-ink-alley.json`

- 工艺入口：`hui-ink`
- 活动：`hz-ink-workshop`
- NPC：`hz-cheng-moshou`
- 区内通道：`huizhou-paper-valley`、`huizhou-she-stone`、`huizhou-merchant-hall`
- 跨区通道：`jiangnan`、`ganpo`

## 4. 冻结与后续

本轮冻结 `huizhou-ink-alley`、`hz-ink-workshop`、`hz-cheng-moshou` 及其地图入口目标。最终美术资源、街巷陈设、建筑朝向、NPC 站姿和精确碰撞后置给美术替换与点位调整。

后续优先级：

1. 补 `huizhou-she-stone` 歙石山坑人工地图。
2. 把 `hui-ink` 从地图工艺入口扩成专属工艺交互规格。
3. 继续推进徽商会馆荐藏回访与信用风险长线。

## 5. 验收口径

- 单测需覆盖地图 JSON 注册、关键入口、冻结 ID、章节 gap 收窄、街景 spec 和导航 gate。
- 内置浏览器 smoke 使用 DEV-only `smokeSubregion=huizhou-ink-alley` 作为 QA 起点，只验证渲染和交互入口，不替代正式街景 gate 移动规则。
- 正式玩家移动仍必须通过街景通道、地区入口或可交互 gate，不使用总览地图直跳作为可达性证明。
