# M1.20 巴蜀临邛铁炉人工地图与入口收束记录

日期：2026-06-15

## 1. 本轮边界

本轮只关闭 `bashu-linqiong-iron` 的人工街景与运行时入口断点：铁矿采集、冶铁、`bs-linqiong-forge` 活动、`bs-deng-lusheng` NPC、通往锦里 / 青神竹海 / 茶马驿的区内 gate，以及巴蜀既有三条跨区 gate。

不扩展：临邛铁炉订单回流长线、龙泉剑铁料账、茶马会后日谈、蜀锦 / 青神竹编订单差异、M3 经济和最终美术摆位。

明确口径：`harvest-coal` 本轮只保留为炭仓装饰和资源需求背景，不作为巴蜀本地 live industry 冻结；巴蜀跨区路线默认落点仍由茶马驿承接，不把临邛铁炉改成路线默认落点。

## 2. 六团队审计结论

| 团队 | 结论 |
| --- | --- |
| 集成与地图注册 | 需要新增人工 JSON 并进入 `RUNTIME_MAP_LAYOUTS`，否则章节入口仍只是声明。 |
| 巴蜀章节与 NPC | 临邛铁炉必须同时有邓炉生 NPC、铁炉活动和章节 NPC 锚点。 |
| 工艺与活动 | 本轮适合补最小活动挑战，不扩大为铁器专属 `CraftInteractionSpec`。 |
| 路线与正式移动 | 区内移动走街景 gate；跨区 routeId 保持原落点，不通过总览地图证明可达性。 |
| 冻结 ID | 冻结 layout / activity / npc / industry / gate 目标，不冻结最终美术摆位。 |
| QA 与文档 | 需要把 35/43 人工地图口径更新为 36/43，并记录 `harvest-coal` 未冻结。 |

## 3. 程序落点

- 新增 `src/data/mapLayouts/bashu-linqiong-iron.json`，并注册到 `src/data/mapLayout.ts`。
- 临邛铁炉街景现在具备 `harvest-iron-ore`、`smelt-iron`、`bs-linqiong-forge`、`bs-deng-lusheng`。
- 新增活动挑战 `bs-linqiong-forge-fire`，用 `timing_hold` 承接看火、加炭和控风的最小可玩钩子。
- 章节数据把 `bashu-linqiong-iron` 纳入巴蜀入口，把 `bs-deng-lusheng` 纳入巴蜀章节 NPC，把下一步从“补临邛铁炉入口”改为“补临邛铁炉订单回流与铁锭供应读数”。
- `fullScope` 中巴蜀人工 layout 最低计数从 3 提到 4，后续缺口转向订单回流、茶马会后日谈和更多生活事件。

## 4. 冻结与后续

本轮冻结：

- 子区与 layout：`bashu-linqiong-iron`
- 活动：`bs-linqiong-forge`
- 活动挑战：`bs-linqiong-forge-fire`
- NPC：`bs-deng-lusheng`
- 产业入口：`harvest-iron-ore`、`smelt-iron`
- 区内 gate：`bashu-jinli`、`bashu-bamboo-sea`、`bashu-tea-horse`
- 跨区 gate 目标：`qiandian`、`jingchu`、`xueyu`

未冻结：

- `harvest-coal` 不作为巴蜀本地产业入口。
- 不新增铁器专属 `CraftInteractionSpec`。
- 不新增路线 ID，不改变 `route-bashu-qiandian-tea-horse`、`route-bashu-jingchu-river`、`route-bashu-xueyu-snow-pass` 的巴蜀侧落点。
