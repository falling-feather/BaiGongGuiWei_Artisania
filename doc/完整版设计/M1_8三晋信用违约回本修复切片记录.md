# M1.8 三晋信用违约回本修复切片记录

日期：2026-06-15

## 节点目标

M1.8 关闭 M1 后续动作中“三晋信用违约/回本修复”的第一轮程序闭环。目标不是把 `chapter-sanjin-piaohao-lacquer` 升为 `chapter-ready`，也不补煤铁窑塬/清徐醋坊人工地图，而是把雷掌柜信用单、逾期失约、推光样柜回访、票号漆档续订单和后续信用条款显式接成可测试链路。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md)、[M1全地区核心章节入库记录.md](M1全地区核心章节入库记录.md)、[M1_6京畿声望采办门槛记录.md](M1_6京畿声望采办门槛记录.md) 与 [M1_7雪域供展回访切片记录.md](M1_7雪域供展回访切片记录.md)。

## 本轮程序落点

1. `src/engine/reducer.ts`
   - 雷掌柜 `orderTrustScore` 读取 `credit-default:sj-lei-zhanggui`，未修复时对后续新单扣信用分。
   - 完成 `polish-credit-ledger` 来源的票号漆柜续订单且已存在雷掌柜失约时，写入 `credit-default-repaired:sj-lei-zhanggui`、`credit-default-recovered:polish-credit-ledger`、`sanjin-credit-payback-restored`。
   - 修复后雷掌柜后续新单读取恢复 flag，信用分回升，押金条款与文案显示“旧失约已由漆柜续订单补回”。
2. `src/engine/__tests__/sanjinCreditRecovery.test.ts`
   - 新增三晋专项守卫，覆盖章节 hook、雷掌柜信用失约、失约后押金恶化、票号漆档续单回本修复、修复后新单恢复，以及无失约时正向回访不误写修复 flags。
3. `src/data/regionChapters.ts`
   - 三晋 gap 从“缺长期违约/回本指标”改为“已有第一轮信用违约与回本修复；仍缺复违约利钱后日谈、更多账本分支和煤铁/醋坊人工 JSON”。
4. `src/data/__tests__/priorityFrozenIds.test.ts`
   - 纳入 `sanjin-piaohao`、`sj-lei-zhanggui`、`homevisit-pingyao-polish-room`、`homevisit-pingyao-client-return`、`collab-pingyao-hand-polish` 与 `sanjin-piaohao` layout。

## 当前限制

- `sj-piaohao` 活动生成的“票号押货推光漆单”仍保持 `route` 订单，不改为 `credit`，避免冲击 M1 活动订单语义。
- `homevisit-pingyao-client-return` 仍保留正向声誉链；没有雷掌柜失约时，票号漆档续单只写声誉与荐藏 flags，不写信用修复 flags。
- `credit-default:sj-lei-zhanggui` 仍是 NPC 级发生过 flag，不统计多次违约、累计本金或利钱。复违约、利钱和多轮账本属于后续 M3 经济深挖。
- `sanjin-coal-yard` 与 `sanjin-vinegar-yard` 仍缺人工 JSON；正式移动继续只通过街景 gate/通道触发，不通过总览地图直接迁移。

## 当前判断

M1.8 让三晋从“票号活动即时订单 + 雷掌柜信用失约 + 平遥漆婆正向续单”推进为“信用失约会影响后续新单，票号漆档续订单能把旧失约补回并恢复条款”的可测试章节骨架。后续三晋更适合补复违约利钱后日谈、票号多轮账本、煤铁窑塬/清徐醋坊人工地图和煤铁/醋坊人物回访，而不是继续在本轮扩大经济系统字段。
