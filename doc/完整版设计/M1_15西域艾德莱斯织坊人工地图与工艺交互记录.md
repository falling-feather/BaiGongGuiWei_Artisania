# M1.15 西域艾德莱斯织坊人工地图与工艺交互记录

日期：2026-06-15

## 本轮目标

本轮收束 M1 中西域章节的两个明确程序缺口：`xiyu-atlas-loom` 缺人工地图 JSON，`atlas-silk` 缺专属工艺交互规格。范围只覆盖织坊可达性、艾德莱斯绸工艺模拟、活动挑战和章节守卫；不扩成长线商路经济、新路线 ID、织坊藏客回访大章或最终美术摆位。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md) 与 [M1全地区核心章节入库记录.md](M1全地区核心章节入库记录.md)。

## 协作团队

按 M1 固定团队数启动 6 个只读审计团队，主线程负责实现与验证。

| 团队 | 范围 | 关键结论 |
| --- | --- | --- |
| 1 范围与验收 | M1.15 是否符合当前优先级 | 本切片应收窄为 `xiyu-atlas-loom` 人工地图 + `atlas-silk` 工艺规格，不扩展 M3 经济。 |
| 2 西域内容与 ID | 西域现有命名、NPC、订单钩子 | 继续使用 `xiyu-*` 子区/活动前缀、`xu-guli` NPC、`atlas-silk` 工艺和 `atlasSilk` 资源。 |
| 3 工艺系统 | craft interaction 数据缺口 | 缺的是数据规格，不需要改 `craftTechniques`、`CraftPage` 或工艺规则层。 |
| 4 地图与迁移 | 人工地图、gate、正式移动红线 | 织坊应通过巴扎/玉场/驿站的街景 `subregionGate` 到达；总览地图不作为正式迁移。 |
| 5 QA 与 smoke | 测试命令、DEV 参数、浏览器动作 | 使用 `?chapterSmoke=chapter-xiyu-bazaar-caravan&smokeSubregion=xiyu-atlas-loom` 做内置浏览器 smoke；截图不可作为唯一证据。 |
| 6 文档同步 | 文档、冻结、不能夸大项 | 记录为程序入口与数据链路完成，不能写成美术完成或 M1 全部完成。 |

## 程序落点

1. `src/data/mapLayouts/xiyu-atlas-loom.json`
   - 新增 52x28 西域织坊人工地图，沿用西域三横三纵路网。
   - 落入 `harvest-cocoon`、`sericulture`、`atlas-silk`、`xiyu-atlas-loom`、`xu-guli`。
   - 提供到 `xiyu-bazaar`、`xiyu-jade-yard`、`xiyu-caravan-post` 的区内 gate，以及到 `xueyu` 的正式跨区 gate。
2. `src/data/mapLayout.ts`
   - 注册 `xiyuAtlasLoomMap`，织坊进入 `RUNTIME_MAP_EDITOR_SNAPSHOTS` 与运行时 layout 管线。
3. `src/data/mapLayouts/xiyu-caravan-post.json`
   - 驼队驿站补到织坊的 `subregionGate`，让巴扎、玉场、驿站、织坊四点形成街景通道闭环。
4. `src/data/craftInteractions.ts`
   - 新增 `interaction-atlas-silk`，挂接 `atlas-silk`、`xiyu`、`xiyu-atlas-loom`、`xu-guli`。
   - 三阶段映射 `atlas-silk-prep`、`atlas-silk-a`、`atlas-silk-b`：缫丝牵经备染、扎经分段染色、织绸成匹。
   - 新增染色发浊、跳纹断气、边口松散三类缺陷和清洗复染、补扎经位、回机收边三类返修。
5. `src/data/activityChallenges.ts`
   - 新增 `xiyu-atlas-loom-tie-dye`，让织坊活动具备 `aim_place` 可执行挑战。
6. `src/dev/regionChapterSmokeScenarios.ts`
   - 西域章节 smoke 的工艺目标切到 `atlas-silk`、NPC 切到 `xu-guli`，便于 DEV smoke 直接验证织坊工艺链。
7. 守卫测试
   - `mapLayout.test.ts`、`priorityMapLayoutEntrypoints.test.ts`、`priorityFrozenIds.test.ts`、`craftInteractions.test.ts`、`craftTechniques.test.ts`、`fullScopeAudit.test.ts`、`regionChapterSmokeScenarios.test.ts` 同步更新。

## 未关闭项

- 西域仍保留后续长线缺口：巴扎后续单与长线商路风险联动、玉作与织坊作品的藏客回访和订单差异。
- `carpet`、`copperware` 仍缺专属 craft interaction spec，本轮不扩大范围。
- 最终织坊瓦片、织机装饰、NPC 站姿和细摆位仍待美术组替换；本轮只冻结程序 ID 与交互目标。
