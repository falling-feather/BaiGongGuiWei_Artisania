M1 用于把项目从“当前优先版五区主轴 + 六区骨架”推进为“11 区均有核心章节线”的程序结构。本记录是 M1 第一切片：先完成章节数据契约、引用守卫和审计入口，让后续补内容时不再以 Markdown 或口头判断为准。

## 本轮程序落点

新增/接入：

1. `src/data/regionChapters.ts`
   - 新增 `RegionChapterSpec`。
   - 11 区各有 1 条核心章节线。
   - 每章都含三类玩法支柱：`craft`、`life`、`tradeRoute`。
   - 每章都含三类人物角色：`artisan`、`trade`、`lifeCulture`。
   - 每章都记录订单/回访/联作/护商钩子、smoke 场景、下一步动作和 gap。
2. `src/engine/regionChapterAudit.ts`
   - 新增纯函数 `buildRegionChapterAudit`。
   - 输出章节 readiness、未知引用、缺人工地图入口、proposed 钩子和计数。
   - 对坏引用输出稳定字符串，如 `npc:missing-id`。
3. `src/data/__tests__/regionChapters.test.ts`
   - 守卫章节表通过数据统一出口导出。
   - 守卫 11 区覆盖、三类玩法、三类人物、真实 ID、订单钩子和 smoke 场景。
4. `src/data/__tests__/regionChapterMapEntrypoints.test.ts`
   - 守卫章节入口若缺人工 JSON，必须在 `gaps` 中点名小地区 ID。
   - 防止程序化 fallback 被误记为正式点位完成。
5. `src/engine/__tests__/regionChapterAudit.test.ts`
   - 守卫章节审计 11 行无未知引用。
   - 守卫坏引用能被稳定报出。
   - 守卫 `jiangnan-suhang`、`jiangnan-baigongyuan`、`ganpo-kaolin-hill`、`ganpo-river-wood` 等地图缺口可见。
6. `src/data/index.ts`、`src/engine/index.ts`
   - 导出章节数据与章节审计函数。
