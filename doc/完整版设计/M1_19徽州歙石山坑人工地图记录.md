# M1.19 徽州歙石山坑人工地图记录

日期：2026-06-15  
分支：`codex/scoped-art-handoff-plan`  
节点来源：`程序大型化协作指引.md` 的 M1 全地区核心章节入库后续收束

## 1. 本轮边界

本轮只补 `huizhou-she-stone` 的人工街景、采石入口和运行守卫，目标是关闭徽州纸墨 / 文房章节最后一个直接人工地图断点。

本轮不扩张：

- 徽商会馆荐藏回访长线。
- 商会信用风险、借贷押货、M3 级区域价格或债务模拟。
- `she-inkstone` 专属 `CraftInteractionSpec`、缺陷、返修和订单拒收链。
- `hui-ink` 专属工艺交互。
- 跨区路线默认落点改造。

因此徽州 M1 地图缺口从“歙石山坑仍缺人工 JSON”收束为“直接核心人工地图断点关闭”；徽州章节仍保持 `needs-expansion`，剩余转向文房订单差异、荐藏回访和信用长线。

## 2. 六团队审计结论

M1.19 按 M1 固定 6 团队执行审计。阶段性结论：

- `huizhou-she-stone` 已有小地区、`she-inkstone`、`hz-she-stone-pit`、`hz-xu-yanshi` 和活动挑战，适合做窄切片地图收束。
- 地图应沿用徽州 56x30 人工图规格，保持三横三纵路网与玩家出生点。
- 歙石山坑应只承接歙石、相石、开砚、许砚石和徽州内部回连；纸谷、墨巷、会馆信用与藏客回访不作为坑口主入口。
- 区内 gate 必须使用 `interaction: "gate"` + `runtimeInteraction: "subregionGate"`。
- `route-jiangnan-huizhou-paper` 仍落 `huizhou-paper-valley`，`route-ganpo-huizhou-merchant` 仍落 `huizhou-merchant-hall`。
- `she-inkstone` 本轮只冻结地图工艺入口，不冻结专属工艺交互规格。

## 4. 冻结与后续

本轮冻结 `huizhou-she-stone`、`hz-she-stone-pit`、`hz-xu-yanshi` 和 `harvest-she-stone` 的程序入口目标。最终山坑地形、砚石堆、磨砚台、溪桥、凿石棚、NPC 站姿和精确碰撞后置给美术替换与点位调整。

后续优先级：

1. 把 `she-inkstone` 扩成专属工艺交互规格，补相石、开璞、雕纹、磨池、缺陷、返修和订单拒收反馈。
2. 把徽商会馆荐藏回访与商会信用风险做成长线系统。
3. 继续补徽墨、宣纸、歙砚之间的订单差异和文房套装反馈。

## 5. 验收口径

- 单测需覆盖地图 JSON 注册、关键入口、冻结 ID、章节 gap 收窄、街景 spec、导航 gate 和 DEV-only smoke 起点。
- 内置浏览器 smoke 使用 `smokeSubregion=huizhou-she-stone` 只作为 QA 起点，不替代正式街景 gate 移动。
- 正式玩家移动仍必须通过街景通道、地区入口或可交互 gate，不使用总览地图直跳作为可达性证明。
