# M1.21 岭南佛山冶坊与端石坑口人工地图记录

日期：2026-06-16

## 1. 本轮边界

本轮关闭岭南章节中 `lingnan-forge` 与 `lingnan-duan-stone` 的人工街景入口断点：佛山冶坊落入采铁矿、冶铁、`ln-foshan-forge` 活动和梁铁线 NPC；端石坑口落入端砚 / 石湾陶地图工艺入口、`ln-duan-inkstone-pit` 活动和谭砚伯 NPC。两张图均通过街景 gate 回接珠江商港、香云纱晒场和黔滇商港路。

不扩展：佛山铁器修造订单回流、端砚 / 石湾陶专属 `CraftInteractionSpec`、端砚 / 石湾陶订单缺陷读取、骑楼夜市船期后日谈、最终美术摆位和 M3 经济。

明确口径：`duan-inkstone` 与 `shiwan-pottery` 本轮只作为端石坑口地图工艺入口冻结；端石采集仍由既有 `ln-duan-inkstone-pit` 活动产出 `duanStone`，不新增 `harvest-duan-stone` 产业入口。跨区路线仍通过既有黔滇商港路线落点承接，不把佛山冶坊或端石坑口改为路线默认落点。

## 2. 六团队审计结论

| 团队 | 结论 |
| --- | --- |
| 集成与地图注册 | 需要新增两张人工 JSON 并进入 `RUNTIME_MAP_LAYOUTS`，否则岭南四个章节小地区仍不完整。 |
| 岭南章节与 NPC | 佛山冶坊必须绑定梁铁线，端石坑口必须绑定谭砚伯，避免入口只有活动点没有人物锚。 |
| 工艺与活动 | 佛山冶坊适合补最小看炉挑战；端石坑口已有端石听声挑战，本轮不扩大端砚 / 石湾陶工艺规格。 |
| 路线与正式移动 | 区内移动走珠江商港 / 晒场 / 冶坊 / 端石坑口的街景 gate；跨区仍经黔滇商港路。 |
| 冻结 ID | 冻结 layout / activity / npc / craft 地图入口 / industry / gate 目标，不冻结最终美术摆位。 |
| QA 与文档 | 人工地图口径更新为 38/43，活动挑战从 27 更新到 28，并记录 `duan-inkstone` / `shiwan-pottery` 仅为地图入口。 |

## 3. 程序落点

- 新增 `src/data/mapLayouts/lingnan-forge.json` 与 `src/data/mapLayouts/lingnan-duan-stone.json`，并注册到 `src/data/mapLayout.ts`。
- 佛山冶坊街景现在具备 `harvest-iron-ore`、`smelt-iron`、`ln-foshan-forge`、`ln-liang-tiexian`。
- 端石坑口街景现在具备 `duan-inkstone`、`shiwan-pottery`、`ln-duan-inkstone-pit`、`ln-tan-yanbo`。
- 新增活动挑战 `ln-foshan-forge-fire`，用 `timing_hold` 承接看炉、续韧和铁器修造的最小可玩钩子。
- 章节数据把 `lingnan-forge` 与 `lingnan-duan-stone` 纳入岭南入口，把下一步从“补佛山冶坊或端石坑口入口”改为“补佛山铁器修造回流、端砚 / 石湾陶订单读缺陷和骑楼船期后日谈”。
- `fullScope` 中岭南人工 layout 最低计数从 3 提到 4，后续缺口转向订单、回访和长线商贸，而不是人工地图断点。

- 子区与 layout：`lingnan-forge`、`lingnan-duan-stone`

- 区内 gate：`lingnan-harbor`、`lingnan-gambiered-yard`、`lingnan-forge`、`lingnan-duan-stone`
- 跨区 gate 目标：`qiandian`

- 不新增端砚 / 石湾陶专属 `CraftInteractionSpec`。

- 不宣告佛山铁器修造订单回流、端砚 / 石湾陶订单缺陷读取或骑楼船期后日谈完成。
