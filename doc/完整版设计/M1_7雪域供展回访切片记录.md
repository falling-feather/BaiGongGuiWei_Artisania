## 本轮程序落点

1. `src/data/regionContent.ts`
   - `homevisit-losang-patron-return.requiredFlags` 从只要求 `homevisit-referral-completed:reverent-loan-hall`，扩展为同时要求 `activity-order-completed:xy-snow-pass`。
   - `homevisit-losang-patron-return.desc` 明确“净室供展复单与雪口供展补给单均交付后”才带藏客回访。
   - `reverent-loan-hall.referralOrder` 显式固定 `resourceId: 'thangka'`、`quantity: 1`、`minQuality: 0.64`，避免首跳复单随陈列品质量漂移。

2. `src/data/regionChapters.ts`
   - 雪域 gap 从“缺完整地区章节长线收束”更新为“已有雪口活动即时订单、唐卡失败分支和净室供展回访续单；仍缺颜料矿谷/银器帐房人物回访、雪山驿路后日谈和人工 JSON”。
   - 章节仍保留 `needs-expansion`。

3. `src/engine/__tests__/xueyuLongline.test.ts`
   - 新增雪域专项测试。
   - 覆盖章节 hook 绑定、双门槛 requiredFlags、只完成净室复单时阻断回访、完成雪口供展补给单后解锁藏客回访、生成并交付 `净室供展续单`。

4. `src/engine/__tests__/reducer.test.ts`
   - 同步旧雪域结局回声用例，在回访前显式加入 `activity-order-completed:xy-snow-pass` 前置，保留其原本“续订进入结局报告”的覆盖职责。

5. `src/data/__tests__/priorityFrozenIds.test.ts`
   - 冻结本轮提升为正式切片的雪域 ID：`xueyu-snow-pass`、`xy-snow-pass`、`xy-yak-captain`、`homevisit-losang-thangka-hall`、`homevisit-losang-patron-return`、`collab-losang-mineral-layer`。
   - 未把 `xueyu-snow-pass` 加入冻结 layout，人工 JSON 仍是 gap。

## 当前限制

本轮关闭雪域“唐卡供展回访”的第一条完整程序链，但雪域仍保持 `needs-expansion`：

- M1.13 已关闭 `xueyu-snow-pass` 人工 JSON；`xueyu-pigment-valley`、`xueyu-silver-tent` 仍缺人工 JSON。
- 颜料矿谷与银器帐房仍缺人物回访和材料状态反馈。
- `escort-snow-pass-windbreak` 仍是护商支线 hook，尚未把成功/失败结果接入供展续单后日谈。
- `collab-losang-mineral-layer` 仍作为质量/失败返修支线，不作为主链硬门槛。
