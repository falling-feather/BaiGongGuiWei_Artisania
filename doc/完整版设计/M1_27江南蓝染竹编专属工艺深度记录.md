# M1.27 江南蓝染竹编专属工艺深度记录

日期：2026-06-16

## 节点目标

M1.26 已关闭苏小茶 / 湖畔茶肆生活回访链，本节点继续关闭江南剩余程序缺口中的“蓝染 / 竹编专属工艺深度”。

本轮不新增美术资源、不改地图点位、不改变正式移动路线。苏杭水市与城郊百工院仍通过街景 gate / 区内通道到达，`smokeSubregion` 只作为 DEV 浏览器 QA 起点。

## 协作审计结论

- 按 M1 固定 6 团队口径启动只读审计，覆盖集成结构、江南 / 巴蜀工艺语义、跨工艺缺陷模式、订单 / 作品状态、章节审计和 QA smoke。
- 6 个只读团队均已关闭；本轮不保留空转子智能体。
- 审计确认 `bamboo-weaving` 可直接补 `CraftInteractionSpec`，复用现有阶段诊断、缺陷、返修和订单拒收链。
- 审计指出 `indigo-dyeing` 同时存在于江南和黔滇，若仍按 `craftId` 取第一条规格，会让黔滇苗寨蓝染误用江南阿蓝文本。本轮因此先补位置感知选择逻辑，再新增江南规格。

## 程序落点

位置感知工艺规格：

- `craftInteractionFor(content, craftId, { regionId, subregionId })` 已导出并按小地区优先、地区其次选择规格。
- `runProcess` 按当前大地区 / 小地区选择工艺规格；`createItemInstance` 生成作品时记录同一地区规格；`repairItem` 按作品产地选择返修方案。
- `CraftPage`、`CraftExperienceModal` 和 `InventoryModal` 改为共用 `craftInteractionFor`，避免 UI 展示与 reducer 结算不一致。
- 黔滇苗寨的 `indigo-dyeing` 在没有本地专属规格前不会误用江南蓝染缺陷和返修文本，后续可单独推进黔滇扎染 / 蜡染人物线。

江南蓝染：

- 新增 `interaction-jiangnan-indigo-dyeing`，绑定 `indigo-dyeing`、`jiangnan`、`jiangnan-suhang` 与 `jn-indigo-keeper`。
- 工序阶段覆盖 `调靛试色`、`建缸养缸`、`扎结防染`、`浸染氧化`，直接绑定现有 processChain：`harvest-indigo`、`build-vat`、`tie-resist`、`dip-dye`。
- 新增缺陷：`indigo-muddy-vat`（缸气发浑）、`indigo-blurred-resist`（花口糊散）、`indigo-floating-color`（浮色未净）。
- 新增返修：`indigo-reset-vat`（重养靛缸）、`indigo-retie-resist`（补扎花口）、`indigo-rinse-redye`（清洗复染）。
- 订单提示接入阿蓝和灯市布样验收语义，缺陷会进入作品评鉴、背包返修和订单拒收链。

江南竹编：

- 新增 `interaction-jiangnan-bamboo-weaving`，绑定 `bamboo-weaving`、`jiangnan`、`jiangnan-suhang` 与 `jn-bamboo-master`。
- 工序阶段覆盖 `选篾试手`、`理篾分丝`、`经纬编织`，直接绑定现有 processChain：`select-bamboo`、`split-strips`、`weave`。
- 新增缺陷：`jiangnan-bamboo-uneven-strip`（篾宽不匀）、`jiangnan-bamboo-loose-weave`（经纬松散）、`jiangnan-bamboo-rough-rim`（口沿毛涩）。
- 新增返修：`jiangnan-bamboo-resplit-strip`（重理篾宽）、`jiangnan-bamboo-tighten-weave`（回压经纬）、`jiangnan-bamboo-trim-rim`（修口打磨）。
- 订单提示接入周伯、灯市竹器和院藏日用验货语义，返修记录可留在作品履历。

章节与审计：

- 江南章节工艺柱纳入 `indigo-dyeing`、`bamboo-weaving`，章节人物纳入阿蓝与周伯。
- `FULL_SCOPE_REGION_REQUIREMENTS.jiangnan.m1Actions` 移除“补蓝染/竹编基础工艺的专属交互深度”。
- 江南章节剩余缺口收敛为多入口 smokeBindings、灯市后续单关系读数与最终美术摆位。
