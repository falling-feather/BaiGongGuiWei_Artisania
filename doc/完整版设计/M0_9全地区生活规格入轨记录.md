# M0.9 全地区生活规格入轨记录

日期：2026-06-14

## 节点目标

M0.9 接在当前优先版 N5 之后，用于把“当前优先纵切”转入“完整版大型生活模拟”的程序侧工作流。此节点不追求一次性补完 11 区全部内容，而是先把缺口转成可执行、可测试、可分工的矩阵，防止后续继续停留在文档愿景或零散加表。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md)。

## 本轮程序落点

本节点先补一层“完整版范围巡检”，让 M0.9 不是纯文档结论，而能被测试读取。

新增/接入：

1. `src/data/fullScope.ts`
   - 定义 11 个完整版地区的 M1 分组、章节目标、三类玩法支柱、最低覆盖指标和 M1 动作。
   - 保留当前优先五区与骨架六区的差异，但不再把六区只视为资料卡骨架。
2. `src/engine/fullScopeAudit.ts`
   - 提供纯函数 `buildFullScopeAudit`。
   - 按地区统计小地区、NPC、活动、路线、招牌工艺、工艺交互、工坊升级、人工地图、百工志、订单钩子、联作、家访和护商事件。
   - 输出 `ready / partial / gap`，并暴露稳定缺口字符串。
   - 加严未知引用守卫：检查 `REGION_CONTENT`、`SUBREGION_CONTENT` 和工艺交互中的 NPC、活动、小地区、工艺与导师引用。
3. `src/engine/__tests__/fullScopeAudit.test.ts`
   - 守卫 11 区都进入 M0.9 审计矩阵。
   - 守卫当前优先版不能被误判为完整版完成。
   - 固化示例缺口：赣鄱高岭矿丘/河运柴场两张人工地图已由 M1.16 关闭；西域艾德莱斯织坊和 `atlas-silk` 工艺深度已由 M1.15 收束，后续示例转向材料读数、河运风险与更长线回访。
4. `src/data/index.ts`、`src/engine/index.ts`
   - 导出完整版范围数据和审计函数，供后续 UI、测试和工具接入。

## P0/P1 程序动作

P0：

1. 新增 `RegionChapterSpec` 或同级结构，不直接硬扩当前优先版 `priorityJourney`。
2. 将 11 区每区至少一条核心章节线接入结构化数据，并守卫章节引用的 craft、activity、NPC、route、lore、homeVisit、collab 均存在。
3. 为正式移动补 gate 来源/近身凭证；后续验证不能只靠 reducer 直达或 `placeAt`。
4. 为跨区路线增加 `routeId -> landingSubregionId`，避免 `TRAVEL` 默认落到目标地区第一个小地区。
5. 建立 M1 级长局经济指标：金币流、维护缺缴、订单利润、库存积压、路线恢复、声望净变化。

P1：

1. 新增或拆出 `orderChains` 数据入口，避免订单链分散在活动、家访、NPC 功能和测试中。
2. 人工地图从 22/43 继续推进，优先补 `jiangnan-suhang`、`jiangnan-baigongyuan`、`huizhou-ink-alley`、`xueyu-snow-pass`；`ganpo-kaolin-hill`、`ganpo-river-wood` 已由 M1.16 关闭。
3. JSON 注册和 gate 语义增加守卫：新增文件必须进入运行时；gate 必须显式声明跨区或区内。
4. 工艺规格预留半实时桥字段：来源、前置、状态变量、每阶段输入类型、失败标签、返修窗口和社会用途。
5. 教程和失败态补维护费、断供、订单过期、押金/赊账、修复和失败恢复说明。

## M2/M3 前置记录

M2 工艺样板优先级：

1. 龙泉宝剑：火候、锤击、淬火、研磨。
2. 宣纸：水感、浆浓、荡帘、焙墙湿度。
3. 玉雕：相玉、开料、顺绺、抛光与不可逆取舍。

M3 经济指标必须从 M1 起就埋点或测试：

- 每 30/60/120 天金币 min/max/end、净现金流、破产或卡死天数。
- 工坊维护应缴/实缴/缺缴比例，连续缺缴次数和指标下滑。
- 订单接取/完成/过期/违约数量，平均利润，押金损失，品相失败和库存不足。
- 资源峰值、死库存、周转率、售卖/订单/修复消耗占比。
- 路线安全/紧张/危险天数，断供生成/解决/接受短缺和恢复时间。
- 每区声望 gain/loss，断供和订单对声望的净影响。
