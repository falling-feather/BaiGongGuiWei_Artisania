# 百工归位 · 全地区工艺交互模式规划

> 本文只做文档规划，不改代码、不新增美术资源。
>
> 目标：把“采集材料 + 进入工坊 + 完成工艺交互 + 生成作品品质”设计成可以长期游玩的页游系统，而不是一组互不相干的小游戏。玩家应感觉自己在一个真实运转的百工世界里学习、备料、请教、试错、返修、交付、陈列和成名。

---

## 0. 设计红线

### 0.1 不能做成 4399 式小游戏库

以下交互禁止作为核心工艺方案：

- 单纯找准时机按停，没有对应真实工序含义。
- 打地鼠、连连看、消除、无脑 QTE 等只考反应的空洞玩法。
- 每门工艺只换皮同一套小游戏，玩家只记住按钮，不记住工艺。
- 做完小游戏后只给“品质 +1”，没有材料、场所、人物、返修、订单和作品命运。
- 玩家失败后只有扣分，没有工艺上的原因和可理解的改进方向。

### 0.2 工艺交互必须具备的五件事

每门核心工艺至少要回答：

1. **材料从哪里来**：采集、购买、交换、远行、护商、断供如何影响材料。
2. **去哪里做**：对应的工坊、资源点、师傅、工具和天气/时段限制是什么。
3. **为什么这样做**：交互步骤必须映射真实工序，而不是抽象按钮。
4. **失败如何发生**：裂、翘、糊、偏色、走样、火候过、筋骨松、纹样断等都要有工艺原因。
5. **作品如何进入世界**：题名、作者、协作者、题跋、订单、赠礼、陈列、返修、售出和后日谈。

### 0.3 页游长期循环

核心循环不是“玩完一个小游戏”，而是：

```text
地区目标 / 订单 / 人物请求
  -> 查资料 / 请教师傅 / 看样
  -> 备料 / 鉴料 / 跑商 / 护商
  -> 进入对应工坊
  -> 多段真实工序交互
  -> 验收与缺陷诊断
  -> 返修 / 题名 / 陈列 / 交付 / 赠礼
  -> 地区声望、人物关系、工具熟练、图谱解锁
  -> 更复杂的订单和跨区联作
```

这个循环要能支撑几十小时：同一门工艺随着玩家工具、熟练度、师承、图谱和材料等级提升，出现更细的判断、更高阶的作品和更严苛的订单。

### 0.4 页游节奏红线

工艺交互要有厚度，但不能把玩家淹没在重复操作里：

- **首次学习完整交互**：第一次接触工艺时，让玩家完整走一遍真实流程，理解材料、工具和失败原因。
- **日常生产可排程**：熟练后，低阶订单允许批量排程、学徒协助、委托工坊或快速复现，只保留关键判断。
- **高价值作品再深度操作**：珍品、节令订单、NPC 专属联作、宫造采办、藏客荐藏等才要求完整多段交互。
- **重复要有状态变化**：天气、材料批次、师傅建议、工具磨损、市场要求和人物关系要改变每次工序，不做固定刷分。
- **失败要推进关系**：失败作品可以引出师傅教学、返修、改卖、收藏、笑谈或后日谈，不能只是“重来一次”。
- **玩家可以选择专注方向**：有人爱铸剑，有人爱纸墨，有人爱商路。系统应允许专业化，而不是强迫每门手艺都刷满。

## 1. 交互系统分层

### 1.1 四层交互

| 层级 | 玩家行为 | 设计目的 | 例子 |
|---|---|---|---|
| 生活准备层 | 看天气、查订单、问 NPC、备材料 | 让工艺嵌入生活循环 | 雨季伞单、窑期柴价、驼道水账 |
| 材料判断层 | 鉴料、分级、配比、预处理 | 让材料不是数字 | 相玉看绺裂、辨端石石眼、选青檀皮 |
| 工序操作层 | 按真实流程完成多段工序 | 形成核心可玩性 | 折叠锻打、捞纸荡帘、掐丝点蓝 |
| 作品命运层 | 验收、返修、题名、交付、陈列 | 让作品成为世界对象 | 宁辞秋题跋、珍品阁来访、票号信用单 |

### 1.2 品质维度

作品品质不只是一条总分。建议每件作品记录这些隐性或显性维度：

| 维度 | 含义 | 常见影响 |
|---|---|---|
| 材料纯度 | 原料是否合适、是否足量、是否来自好产地 | 颜色、强度、耐久、市场价 |
| 形制准确 | 器形、比例、结构是否符合图样 | 订单验收、师傅评价 |
| 工序完整 | 是否跳过关键工序，等待/阴干/发酵是否足时 | 传承值、长线耐久 |
| 火候/水候 | 温度、水分、湿度、日晒、风雪等环境控制 | 陶瓷、金工、漆器、纸、香云纱 |
| 纹样/线法 | 图样、针法、刻线、錾纹、掐丝是否连贯 | 审美、精神、NPC 题评 |
| 手感/实用 | 是否好用、称手、耐磨、平整 | 生活值、普通订单 |
| 故事痕迹 | 产地、协作者、题跋、返修记录 | 收藏、赠礼、结局报告 |

### 1.3 失败与返修

失败不能只是“品质低”。每次失败应生成可理解的缺陷词：

- 金工：夹渣、锤痕乱、焊点虚、淬裂、开刃偏。
- 陶瓷：坯裂、塌肩、釉泪、窑粘、火刺、色闷。
- 染织：色花、经线松、纬密乱、脱浆、晕边。
- 漆器：起泡、灰底不平、推光浮亮、描金断线。
- 纸墨文房：纸帘纹乱、墨胶重、砚堂涩、雕线崩。
- 玉石：顺绺走错、开窗伤肉、抛光发闷、题材压料。

返修要有选择：降级成日用、请 NPC 救场、追加材料返工、留作教学样本、转卖给不挑剔的客人、陈列为“失败也有故事”的作品。

## 2. 工艺交互类型库

这些是底层玩法模板，但每门工艺都必须按真实流程定制。

| 类型 | 玩家动词 | 适合工艺 | 不是怎么玩 |
|---|---|---|---|
| 鉴料辨性 | 观察纹理、重量、颜色、声响、裂绺，做取舍 | 玉、砚、瓷土、竹、铁、银、纸料 | 不是随机选 A/B/C，要有线索和后果 |
| 图样规划 | 画样、定稿、配色、分层、确定用途 | 剪纸、刺绣、唐卡、漆器、织锦 | 不是涂色小游戏，要影响后续工序 |
| 轨迹手势 | 沿真实工具路径操作，追求稳定而非速度 | 书法、刻纸、錾刻、拉坯、雕木 | 不是越快越好，而是稳、顺、不断线 |
| 配比水火 | 控制温度、水分、浓度、时间、环境 | 青瓷、景泰蓝、制茶、制墨、陈醋、香云纱 | 不是单点按停，而是多变量管理 |
| 结构编排 | 经纬、骨架、榫卯、掐丝、扎经、打结 | 竹编、织锦、地毯、油纸伞、花丝 | 不是拼图通关，而是结构会影响成品 |
| 递进耐心 | 多轮重复但每轮状态变化 | 捞纸、髹漆、推光、晒莨、发酵 | 不是疯狂连点，要能安排等待和复查 |
| 协作请教 | 与 NPC 分工、请师傅看样、让学徒辅助 | 宫造、云锦、唐卡、珍品联作 | 不是 NPC 加成按钮，要有意见与取舍 |
| 验收返修 | 根据缺陷选择补救路径 | 所有工艺 | 不是失败重开，而是作品进入世界 |

## 3. 全链路交互图谱

### 3.1 单件作品图谱

```text
需求来源
  订单 / NPC 请求 / 自选创作 / 节令活动 / 珍品阁邀约

材料链
  本地采集 -> 预处理 -> 鉴料分级
  外地材料 -> 路线情报 -> 护商/断供 -> 到货验收

设计链
  看样 -> 选器形/用途 -> 定纹样/配色/题材 -> NPC 建议

工序链
  粗备 -> 核心工序 A -> 中间检验 -> 核心工序 B
  -> 等待/阴干/发酵/烧制 -> 精修/装配/题名

验收链
  自检 -> 师傅点评 -> 订单验收 -> 返修选择

社会链
  交付 / 售卖 / 赠礼 / 陈列 / 参展 / 藏客转介绍
  -> 声望、关系、图谱、工具、路线、结局回声
```

### 3.2 地区图谱

每个地区都要有至少 3 条可持续链：

- **本地生活链**：玩家在本地也能做日常，不必只为主线而来。
- **代表工艺链**：本地至少一门工艺有完整流程和返修。
- **跨区供应链**：本地材料能流向别处，别处材料也能改变本地作品。

## 4. 江南交互方案

### 4.1 龙泉宝剑

**链路**：铁矿/铁锭 + 炭 -> 龙泉山坊 -> 定剑形 -> 折叠锻打 -> 修形开槽 -> 淬火 -> 研磨开刃 -> 题名/试剑/交付。

| 工序 | 真实逻辑 | 玩家交互 | 品质影响 |
|---|---|---|---|
| 选铁备炭 | 铁料纯度、炭火稳定决定剑体基础 | 鉴料辨性：看铁锭杂质、炭火耐烧，选择“韧骨/锋芒/礼器”方向 | 材料纯度、强度 |
| 定剑形 | 剑长、脊线、用途不同 | 图样规划：按订单用途定礼剑、行旅剑、镇宅剑 | 形制准确、市场 |
| 折叠锻打 | 反复折叠焊合，排杂并均匀组织 | 结构编排 + 火候曲线：安排折叠次数、回火窗口、锤击路径；过度追求次数会耗工时和韧性 | 筋骨、夹渣、精神 |
| 修形开槽 | 修出脊线、血槽或装饰线 | 轨迹手势：沿剑脊推线，保持左右对称 | 形制、观感 |
| 淬火 | 温度、介质、入水角度影响硬度与裂纹 | 配比水火：判断火色，选择水/油/药水和入水节奏；不是按停，而是观察火色变化和剑身状态 | 淬裂、锋芒、韧性 |
| 研磨开刃 | 粗磨到细磨，决定光泽与锋利 | 递进耐心：按粗砥、细砥、油石顺序磨，发现偏锋可返修 | 手感、锋芒 |

**可持续点**：高阶订单要求指定“锋芒/韧骨/礼器纹饰”；护商成功可带回异地好铁；乔照夜灯市可订“灯下礼剑”；宁辞秋可题名。

### 4.2 龙泉青瓷

**链路**：瓷土/高岭土 -> 淘泥陈腐 -> 拉坯修坯 -> 素烧 -> 施青釉 -> 装窑烧成 -> 开窑评色。

交互重点不是“按下烧窑”，而是泥性、器形、釉厚和窑位的连续判断。玩家要在拉坯时控制壁厚，在修坯时取舍轻薄与稳重，在施釉时决定釉层厚薄，在装窑时安排窑位。失败会生成坯裂、釉泪、色闷、窑粘等缺陷。

### 4.3 云锦 / 缂丝 / 蓝染 / 竹编 / 油纸伞

| 工艺 | 全流程交互 | 长期玩法 |
|---|---|---|
| 云锦/缂丝 | 理丝牵经 -> 读花本 -> 挑花/通经断纬 -> 修绒理面 -> 题样 | 花本收集、织机升级、沈云梭联作 |
| 蓝染 | 调靛试色 -> 建缸养缸 -> 扎结防染 -> 反复浸染氧化 -> 拆线晾布 | 靛缸状态随天气和维护变化 |
| 竹编 | 选竹 -> 刮青剖篾 -> 分丝定宽 -> 起底编织 -> 收口修形 | 周伯点评“篾宽、紧密、器形” |
| 油纸伞 | 选竹制骨 -> 糊纸 -> 绘面 -> 刷桐油 -> 晾伞绷面 | 雨季订单、林雨桥伞铺返修 |

## 5. 巴蜀交互方案

### 5.1 青神竹编

真实重点是竹材处理和细篾控制。交互应从“伐竹”开始：看竹龄、节距、青皮状态；剖篾时按用途选择篾宽；编织时用结构编排表现经纬、挑压和收口。不是拖线过迷宫，而是通过篾宽、湿润度、张力和器形保持度决定成品。

### 5.2 蜀锦 / 蜀绣 / 成都漆器

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 蜀锦 | 染丝 -> 牵经 -> 读花本 -> 提花织造 -> 修边 | 先读花本决定经纬组织，再在织造中处理断线和错花 |
| 蜀绣 | 上绷描稿 -> 分线配色 -> 施针 -> 片线光面 -> 收针 | 通过针法选择和线色过渡表现“绣面光亮” |
| 成都漆器 | 制胎刮灰 -> 髹漆阴干 -> 雕花/戗银 -> 填彩研磨 | 多日阴干与湿度管理，急做会起泡 |

### 5.3 井盐、临邛铁炉、茶马道

巴蜀的地区交互要承担“上游生产 + 远行”。盐井控火、铁炉冶炼和茶马护送都不是孤立小游戏，而是为江南剑、京畿宫造、雪域驿路提供材料与路线稳定。

## 6. 岭南交互方案

### 6.1 香云纱

**链路**：炼绸 -> 薯莨汁浸染 -> 晒莨 -> 河泥涂覆 -> 清洗复晒 -> 整理成绸。

玩家交互重点是“天候与等待”：晴天晒、雨天改期；薯莨汁浓度、河泥涂覆厚度、复晒次数共同影响色泽。失败不是扣分，而是色花、发硬、深浅不匀，可转为普通耐穿布或等待何云纱指导返修。

### 6.2 端砚 / 佛山冶坊 / 粤绣 / 石湾陶

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 端砚 | 采石 -> 相石 -> 开璞 -> 雕花 -> 磨堂 | 看石眼、纹理、裂线，决定砚堂位置 |
| 佛山冶坊 | 备铁 -> 熔炼 -> 浇铸/锻修 -> 打磨 | 炉温曲线和模具预热，影响铁器耐用 |
| 粤绣 | 描稿 -> 配线 -> 铺绒钉线 -> 缀珠点金 | 繁密构图管理，珠片过多会压住绣面 |
| 石湾陶 | 炼泥 -> 塑形 -> 修坯 -> 施釉 -> 烧成 | 造型重心和釉色取舍，适合人物陶塑 |

## 7. 黔滇交互方案

### 7.1 苗银

**链路**：银料/铜料 -> 熔银 -> 拉丝/压片 -> 锤錾纹样 -> 焊接装配 -> 清洗抛光 -> 礼俗验收。

玩家交互重点是“款式与礼法”：不是随便拼首饰，而是根据人物、节日、年龄和用途选择银饰部位。银娘阿岚可指出“纹样不合礼”“焊点虚”“压片太薄”。返修可补焊、改成小件、熔回银料但损耗。

### 7.2 蜡染 / 建水紫陶 / 乌铜走银

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 蜡染 | 熔蜡 -> 蜡刀点画 -> 靛染 -> 脱蜡 -> 晾布 | 蜡线温度、图案留白和染缸次数决定冰纹 |
| 建水紫陶 | 淘泥 -> 拉坯 -> 阴刻阳填 -> 烧成 -> 无釉磨光 | 刻填深度和打磨耐心决定“无釉也亮” |
| 乌铜走银 | 炼乌铜 -> 刻纹起地 -> 嵌银丝 -> 捂黑显纹 -> 磨显 | 银丝嵌入深浅和乌铜发色决定纹样清晰度 |

## 8. 荆楚交互方案

### 8.1 楚漆

**链路**：木胎 -> 刮灰 -> 髹漆多道 -> 阴干 -> 彩绘纹样 -> 罩漆 -> 修磨。

楚漆适合做“多日工序管理”。玩家要安排阴干、湿度、漆层厚薄和朱黑纹样。赶工会起泡、裂漆；漆层不足则不耐用；纹样可从渡市、古器、NPC 见闻中解锁。

### 8.2 湘绣 / 土家织锦 / 铜铁矿场

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 湘绣 | 上绷 -> 配线 -> 掺针晕色 -> 鬅毛点睛 -> 收面 | 毛发和眼神不是速度挑战，而是色阶判断 |
| 土家织锦 | 染线 -> 牵经 -> 腰机斜织 -> 挑花 -> 收边 | 反面挑花，玩家要读懂正反关系 |
| 铜铁采炼 | 采矿 -> 洗矿 -> 入炉 -> 出料 -> 运送 | 矿路风险和炉料配比影响京畿/岭南下游 |

## 9. 赣鄱交互方案

### 9.1 景德镇制瓷

**链路**：采高岭土 -> 淘洗练泥 -> 拉坯 -> 利坯 -> 晒坯 -> 绘青花/施釉 -> 装窑 -> 烧成 -> 开窑分级。

景德镇不应压缩成“拉坯小游戏”。它的核心是分工链和窑期。玩家可以亲手做关键工序，也可以请 NPC 分工。窑位、柴火、釉厚、坯体干湿共同影响结果。失败件可变成残瓷拼贴、日用粗瓷或学习样本。

### 9.2 夏布与河运柴场

夏布交互：剥麻刮青 -> 绩麻接线 -> 上机手织 -> 漂洗整理。重点是线的连续性和耐心，不是单纯重复点击。

河运柴场交互：签柴约 -> 查水位 -> 装船 -> 护运 -> 到窑场验柴。它服务窑火，而不是独立小游戏。

---

## 10. 徽州交互方案

### 10.1 宣纸

**链路**：青檀皮/稻草等原料 -> 蒸煮浸泡 -> 捶洗成浆 -> 荡帘捞纸 -> 压榨 -> 上墙焙干 -> 揭纸分级。

玩家交互重点是“水感与帘感”：荡帘不是左右拖拽，而是浆浓、入帘角度、摇帘节奏共同决定纤维均匀。焙墙阶段要判断湿度和揭纸时机。失败纸可用于练字、包装、低价订单，优质纸可触发宁辞秋题跋与藏客。

### 10.2 徽墨 / 歙砚 / 徽州木雕

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 徽墨 | 烧烟 -> 和胶 -> 捶打 -> 入模 -> 晾墨 -> 描金 | 胶烟比例和捶打程度影响墨色与裂纹 |
| 歙砚 | 采石 -> 辨纹 -> 开砚 -> 雕纹 -> 磨池 | 石纹决定设计，不能硬套图样 |
| 木雕 | 选木 -> 起稿 -> 粗雕 -> 透雕 -> 精修打磨 | 层次规划，刀路过深会崩线 |

---

## 11. 京畿交互方案

### 11.1 景泰蓝

**链路**：制铜胎 -> 掐丝 -> 点蓝 -> 烧蓝 -> 补蓝复烧 -> 磨光 -> 镀金 -> 官样验收。

玩家交互重点是“宫造标准”：掐丝不只是摆线，线条要贴合胎体曲面；点蓝要按色料流动与烧后收缩预留；烧蓝要多次补蓝。官署订单会要求纹样、色阶和金口完整度。失败可生成丝线翘、砂眼、色料塌、磨穿等缺陷。

### 11.2 花丝镶嵌 / 剔红雕漆 / 内画

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 花丝镶嵌 | 抽丝 -> 掐花丝 -> 攒焊 -> 錾活 -> 镶嵌 | 微结构搭建，焊点和宝石座决定牢固 |
| 剔红雕漆 | 制胎 -> 层漆百道 -> 养护 -> 剔刻 -> 磨显 | 漆层厚度和刀路层次决定浮雕感 |
| 内画 | 磨内壁 -> 反向起稿 -> 内壁勾线 -> 填彩 -> 封口 | 反向思维和手稳，适合高手挑战 |

### 11.3 蔚县剪纸作为外部模板

蔚县剪纸可作为后续京畿/燕赵年节扩展模板。推荐流程：

```text
题材采风 -> 画样 -> 分色定版 -> 打钉固定 -> 刻纸 -> 点染/套色 -> 熏样/做旧 -> 揭纸装裱 -> 年节交付
```

交互上，画样决定构图，打钉决定多层纸是否位移，刻纸考验刀路连续和转折，点染考验颜色渗透，装裱决定成品平整。它不能做成“沿线切割一次通关”，而应是设计、固定、刻制、染色和装裱的连续链。

## 12. 三晋交互方案

### 12.1 平遥推光漆器

**链路**：制胎 -> 刮灰 -> 髹漆多道 -> 阴干 -> 研磨 -> 手掌推光 -> 彩绘描金 -> 验面。

玩家交互重点是“慢”。每一道漆层都需要等待，推光不是点击次数越多越好，而是掌温、力度、方向和漆面状态配合。失败会出现浮亮未收、局部发乌、描金断线。平遥漆婆可以提供返修分支。

### 12.2 老陈醋 / 晋作家具 / 煤铁

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 老陈醋 | 蒸料 -> 拌曲 -> 发酵 -> 熏醅 -> 淋醋 -> 夏晒冬捞 -> 陈酿 | 跨季节管理，适合页游长线 |
| 晋作家具 | 选木 -> 烘干 -> 开榫 -> 组装 -> 雕饰 -> 打磨上蜡 | 榫卯结构和木性判断 |
| 煤铁窑塬 | 采煤 -> 选料 -> 供炉 -> 运料 | 服务铁器、宫造、瓷窑的燃料链 |

## 13. 雪域交互方案

### 13.1 唐卡

**链路**：绷布上胶 -> 度量起稿 -> 线描 -> 矿彩研磨 -> 层层设色 -> 勾金开脸 -> 装裱护画。

唐卡交互必须尊重语境。核心不是“涂色”，而是度量、线描、矿彩和修行式耐心。玩家要先由洛桑画师确认题材是否合适，再按度量线起稿。矿彩研磨、设色顺序和勾金会影响精神值与人物评价。

### 13.2 藏纸 / 藏银 / 藏香

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 藏纸 | 采料 -> 蒸煮 -> 捶浆 -> 浇帘抄造 -> 阴干压平 | 高原水温和纤维均匀度 |
| 藏银 | 锻银 -> 錾花 -> 镶石 -> 抛光 | 纹样和镶嵌牢固度 |
| 藏香 | 采药 -> 配伍 -> 研磨 -> 和泥 -> 搓制 -> 阴干 | 配方记忆、药材来源和阴干环境 |

---

## 14. 西域交互方案

### 14.1 和田玉雕

**链路**：采玉/购玉 -> 相玉看皮色与绺裂 -> 开窗 -> 切料 -> 设计题材 -> 粗琢 -> 细琢 -> 抛光 -> 巴扎鉴评。

玉雕的核心是“因材施艺”。玩家不能先定图再硬雕，而要根据玉料的皮色、裂、棉、脏、形状决定题材。开窗错会伤肉；抛光过度会发闷。玉师阿月可指导“顺绺避裂”。

### 14.2 艾德莱斯 / 地毯 / 铜器

| 工艺 | 工序链 | 玩家交互 |
|---|---|---|
| 艾德莱斯绸 | 缫丝 -> 牵经 -> 扎经 -> 分段染色 -> 织绸 -> 整理 | 扎经位置决定云霞纹样，不是随机染色 |
| 地毯 | 设计图样 -> 染线 -> 栽绒打结 -> 平剪 -> 洗毯整理 | 每排结的密度和图案读谱 |
| 铜器 | 备铜 -> 锤揲成器 -> 錾刻 -> 镶嵌 -> 抛光 | 锤痕均匀、器形稳定、纹样连贯 |

## 16. 文档落地格式

后续每门核心工艺都应补一张 `CraftInteractionSpec` 草案：

```ts
interface CraftInteractionSpec {
  craftId: string;
  regionId: string;
  workshopId: string;
  sourceReferences: string[];
  prerequisite: {
    materials: string[];
    npc?: string;
    seasonOrWeather?: string;
    toolLevel?: string;
  };
  stages: Array<{
    id: string;
    realStep: string;
    playerAction: string;
    stateVariables: string[];
    techniqueChoices: string[]; // 稳手 / 平衡 / 赶工等节奏取舍
    focusChecks: string[]; // 察火候 / 卡准点 / 压线抢手等下手窗口
    outcomeDimensions: string[]; // 本段会写入的控制、风险和品质维度
    failureTags: string[];
    repairOptions: string[];
  }>;
  output: {
    resourceId: string;
    qualityDimensions: string[];
    possibleDescriptors: string[];
    socialUses: string[];
  };
}
```

这张表不是要立即编码，而是保证每次扩工艺时都有资料、地点、人物、状态变量、失败词和作品去向。

当前代码落地（2026-06-13）：`CraftInteractionStage` 已作为运行时阶段规格，`RUN_PROCESS.techniqueChoices` 写手法记录，`RUN_PROCESS.focusChecks` 写专注校准记录，`craftStageOutcomes` 统一保存控制/风险/稳平险结果。后续实时小游戏只需要把输入结果汇总回这些阶段字段，仍由 reducer 负责品质、缺陷、返修、订单和作品命运。
