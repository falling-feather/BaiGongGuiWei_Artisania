# M1.2 路线落点与章节导航守卫记录

日期：2026-06-14

## 节点目标

M1.2 关闭 M1 后续动作中的“路线落点”和“章节导航守卫”两项。目标不是补美术排布，而是把正式跨区移动从“进入目标地区默认小地区”推进为“街景 gate 携带路线，reducer 按路线落到章节入口小地区”，并用测试证明章节目标经街景 gate/subregionGate 可达。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md) 与 [M1全地区核心章节入库记录.md](M1全地区核心章节入库记录.md)。

## 协作团队

按 M1 固定团队数继续启用 6 个只读审计团队。

| 团队 | 代号 | 范围 | 关键结论 |
| --- | --- | --- | --- |
| 1 路线落点结构 | Bohr | `RouteSpec`、`REGION_ROUTES`、`TRAVEL` | 落点应放入双向 `RouteSpec.landingSubregionIds`，不绑死在章节表。 |
| 2 章节导航测试 | Sartre | 街景导航测试结构 | 可以造章节进度，但不能直接造玩家位置；位置必须经当前街景 gate 行走。 |
| 3 地图布局与 gate | Anscombe | 人工 JSON 与 gate 可证明性 | 入口自身无人工 JSON 仍按 gap 记录；已有 JSON 的 gate 可作为证明。 |
| 4 数据契约兼容 | Hubble | 章节路线校验 | 章节引用路线时，落点必须属于本章节地区并落在章节入口集合内。 |
| 5 文档同步 | Boole | M1/当前优先记录 | M1 第 2、3 项可关闭，订单链、独立 smoke、人工地图仍保留。 |
| 6 风险与 smoke | Carver | 测试与浏览器 smoke | 本轮需跑定向 Vitest、全量测试、构建，并补内置浏览器短 smoke。 |

## 本轮程序落点

1. `src/engine/types.ts`
   - `RouteSpec` 新增 `landingSubregionIds?: Partial<Record<string, string>>`。
   - `TRAVEL` action 新增可选 `routeId`。
2. `src/data/regionContent.ts`
   - `REGION_ROUTES` 的所有正式路线均补双向落点。
   - 路线落点优先选择章节入口小地区，保留缺人工 JSON 的真实章节落点为后续地图任务。
3. `src/engine/reducer.ts`
   - `TRAVEL` 传入 `routeId` 时，按该路线的目标地区落点进入小地区。
   - 未传 `routeId` 时保留原默认小地区行为，服务调试和兼容旧测试。
4. `src/game/EventBus.ts`、`src/game/scenes/StreetScene.ts`、`src/App.tsx`、`src/game/navigationGuards.ts`
   - 街景 gate 事件携带 `routeId`。
   - React 只接受当前街景 spec 中真实存在的 gate，并用该 gate 的 `routeId` 派发正式迁移。
5. `src/engine/regionChapterAudit.ts`
   - 新增 `routeLandingGaps` 审计字段。
   - 章节路线缺落点、落点地区错误或落点小地区无效时，归入 `unknownReferences` 并使章节审计 invalid。
6. 测试
   - `regionChapters.test.ts` 守卫所有正式路线都有双向落点，并且章节路线落点落在章节入口集合内。
   - `reducer.test.ts` 守卫 `routeId` 驱动的正式跨区落点，同时保留无 `routeId` 的默认行为。
   - `navigationEntrypoints.test.ts` 守卫当前街景 gate guard 也返回 `routeId`。
   - `regionChapterNavigation.test.ts` 新增 24 条章节路线导航用例，从新档状态出发，经街景 gate、解锁、`TRAVEL(routeId)` 抵达章节落点。

## 当前判断

M1 已从“11 区章节数据契约”推进到“正式路线落点与章节导航可达性守卫”。这关闭了 M1 原后续动作中的路线落点和章节导航两项，但不代表 M1 全部完成。

仍需继续推进：

1. `needs-expansion` 章节真实订单链，优先黔滇、荆楚、徽州、京畿、三晋、雪域。
2. 独立章节 smoke 入口，避免完整版继续塞入当前优先版 `prioritySmokeScenarios`。
3. 人工地图 JSON，优先章节路线落点和缺口入口。
4. 巴蜀生活面、岭南端砚/石湾陶、西域艾德莱斯织坊等内容密度缺口。
