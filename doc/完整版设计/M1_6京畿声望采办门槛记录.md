# M1.6 京畿声望采办门槛记录

日期：2026-06-15

## 节点目标

M1.6 关闭 M1 后续动作中“京畿声望 / 商誉采办门槛”的第一刀。目标不是把 `chapter-jingji-palace-procurement` 直接升为 `chapter-ready`，而是先让官署门房和宋押司的采办入口读到地区声望、NPC 好感、商贸 / 人情属性、路线风险和宫造背书，从普通 NPC 订单推进为可测试的 `palace` 采办门槛。

执行依据：[程序大型化协作指引.md](程序大型化协作指引.md)、[M1全地区核心章节入库记录.md](M1全地区核心章节入库记录.md)、[M1_4真实订单链补线记录.md](M1_4真实订单链补线记录.md) 与 [M1_5黔滇礼银长线收束记录.md](M1_5黔滇礼银长线收束记录.md)。

## 本轮程序落点

1. `src/engine/reducer.ts`
   - `npcOrderTerms` 新增 `jj-song-yasi` 专用分支。
   - 采办商誉继续复用 `orderTrustScore` 的地区声望、NPC 好感、商贸 / 人情属性与路线风险。
   - `collector-reputation-palace-renewed`、`palace-order-ready`、`jingji-official-permit` 会作为宫造背书额外抬高商誉。
   - 商誉不足时生成 `官样采办预审单`，收名帖保金；商誉达标时生成 `官样采办许可单`，免押但仍按宫样 `palace` 验收。
   - `orderKind: 'palace'` 继续触发清洁作品验收和 `palace-order-completed:jj-song-yasi` 完成 flag。

2. `src/engine/__tests__/jingjiProcurement.test.ts`
   - 新增京畿专项测试。
   - 覆盖宋押司章节绑定、低商誉预审押金、高商誉 / 宫造背书免押许可、正式 palace 采办单交付与地区声望回升。

3. `src/data/regionChapters.ts`
   - 京畿 gap 从“缺声望 / 商誉门槛”更新为“已有宋押司 palace 采办商誉门槛，仍缺都门市口人工 JSON 与更长漕运 / 市口采办分支”。
   - 章节仍保留 `needs-expansion`，避免把地图与后续分支缺口误记为已完成。

## 当前限制

本轮只关闭“采办许可读取声望 / 商誉并影响押金、交期和验收”这一层：

- `jingji-market-gate` 仍缺人工 JSON。
- `jj-official-gate` 活动即时单仍保留为活动路线单，没有改造成活动级 `palace` 门槛，以免扩大押金会计和活动订单类型迁移。
- `homevisit-lan-palace-return` 已能生成宫样续单，但更长的市口背书、漕运复验仍未展开；担保过期惩罚已由 2026-06-16 M1.30 关闭。
- `expireOrdersForDay` 已在 2026-06-16 M1.30 扩展 `palace` 过期担保折损，宋押司采办单逾期会扣京畿地区声望并写入后续许可读数旗标。

## 当前判断

M1.6 让京畿从“已有官署活动即时采办单和宫样回访样板”推进到“宋押司能按玩家地区声望、商誉与宫造背书发放预审 / 正式 palace 采办单”。这条链足够支撑程序端继续向大型生活模拟扩展，但还不是京畿章节完成：下一步更适合转向雪域供展回访或三晋信用违约修复，同时保留京畿都门市口人工地图和漕运复验分支为后续 M1 切片。
