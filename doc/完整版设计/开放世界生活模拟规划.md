## 0. 结论先行

《百工归位》不应被继续定义为“非遗工艺模拟器”，也不应直接改成“古代中国版星露谷”。更准确的定位是：

**一款以中国古代地方生活、百工技艺、人物交往和跨区供应链为核心的 2D 九州行脚生活模拟游戏。**

玩家不是只做工艺，也不是只经营一个农场。玩家扮演一位接过“百工”匾额的行脚匠人，从江南起步，游历中国不同地区，在真实地貌、物产、人物与工艺网络中生活、采集、种植、锻造、交易、结交、学习和传承。最终，玩家的身份从“粗手生人”逐渐成长为“走方匠”“名匠”“宗匠”，乃至被人称作“神匠”。

### 设计判断

- **工艺仍是游戏的灵魂，但不该是唯一动作。** 工艺应该连接地图、资源、人物、订单、节令、传说与个人成长。
- **全国地图必须模块化。** “覆盖整个中国”不能等于真实比例铺满地图，而应拆成大地区节点，每个大地区再有 2-8 个小地区。
- **NPC 系统必须升级为核心玩法。** 现有好感和任务只是骨架，未来要让人物成为技能、剧情、订单、声望和地方知识的入口。
- **生活玩法必须服务百工主题。** 种田、钓鱼、练剑、书法、烹茶、读书、赶集，都应回流到“材料、属性、人物、地方身份”，不能只是堆玩法。
- **联网页游化应延后。** 先做高质量单人可玩闭环，再接账号、云存档、异步拜访、委托榜和玩家市集；不要一开始做实时 MMO。

## 1. 当前项目资产与系统基线

### 已经具备的基础

现有项目并不是空白原型，已经有一条值得保留的主干：

- 技术栈：React + TypeScript + Vite + Phaser + Zustand + Vitest。
- 架构原则：`engine` 纯函数结算、`data` 注入内容、Phaser 只管空间和移动、React 负责 HUD 与弹窗。
- 已有系统：四维数值、工序链、采料小游戏、背包、成就、剧情分支、结局报告、大地图、小地区、NPC 好感、任务、多存档。
- 已有世界数据：江南、巴蜀、岭南、黔滇、荆楚、赣鄱、徽州、京畿、三晋、雪域高原、西域共 11 个大地区。
- 已有供应链：raw 原料、material 半成品、product 成品三层资源，且已有供应链自足报告。
- 已有美术管线：江南、天气、开放世界扩展、西域地区素材，含建筑、瓦片、道具、行为图标和地图编辑器。

### 当前最大短板

- **可玩生活循环不完整。** 当前更像“街道探索 + 点位触发 + 弹窗结算”，玩家还没有真正“住在这个世界里”。
- **NPC 太薄。** 只有江南少量 NPC，且主要是对话加好感、交付任务，没有日程、性格、偏好、人物线和地方关系网。
- **玩家属性缺席。** 目前有全局四维，但缺少“玩家这个人”的成长维度。玩家不知道自己为什么从新手变强。
- **开放世界素材先于玩法。** 剑馆、农家、书院、静修堂等素材已经准备了，但对应行为还没有形成统一系统。
- **全国地图尚未产生“游历理由”。** 有地区和供应链，但需要更强的任务、人物、节令、材料、声望和地方知识来驱动旅行。

## 2. 外部参照提炼

本规划参考同类游戏和非遗资料时，只取结构经验，不复制题材。

### 《Stardew Valley》的可借鉴点

它的核心不是“种田”，而是多条生活循环相互咬合：种植、采矿、钓鱼、采集、战斗、烹饪、手作、NPC 日程、节日和社区修复共同组成长期生活感。我们应学习的是：

- 玩家每天有有限时间和体力，因此选择有分量。
- 每种活动都能积累技能和资源，不会变成一次性小游戏。
- NPC 有日程、生日、偏好和关系事件，世界因此像有人在生活。
- 节令和天气改变资源、人物活动和事件。

### 《东方夜雀食堂》的可借鉴点

它的结构非常清楚：白天采集、买货、跑任务、推进角色，晚上开店服务顾客。我们可以借鉴“昼夜双循环”：

- 白天用于行脚探索、采料、拜访、学习。
- 黄昏或夜间用于工坊制作、市集摆摊、订单交付、宴饮社交。
- 每个地区有自己的顾客、材料、口味和故事线。

### 非遗资料的约束

非遗内容必须建立在可信资料之上，尤其是传统技艺、传统美术、传统制茶、宣纸、云锦、龙泉青瓷、木构营造等项目。游戏可以做虚构人物与虚构城镇，但工序、材料、地理关系和文化语境不能随意拼贴。

## 3. 新定位：九州行脚生活模拟

### 玩家幻想

玩家是一个初出茅庐的行脚匠人。你不是天降救世主，而是在不断走访、拜师、做活、交朋友、还人情、跑商路、攒名声中，把散落九州的百工线索重新接起来。

### 核心动词

| 动词 | 含义 | 产出 |
|---|---|---|
| 行脚 | 在大地区和小地区之间移动，解锁地方知识 | 地图、见闻、商路、人物 |
| 采集 | 采矿、伐竹、割漆、采茶、采桑、采药 | 原料、体魄、地方声望 |
| 种植 | 在家园或租地中种靛草、桑、茶、稻、药草 | 稳定材料、季节收益 |
| 精炼 | 冶铁、炼铜、建靛、缫丝、制纸、制墨 | 半成品、工艺质量 |
| 制作 | 按工序链完成器物、织物、食饮、武器 | 成品、声望、订单 |
| 交往 | 与 NPC 对话、赠物、切磋、代办、联作 | 好感、任务、技能、故事 |
| 交易 | 接订单、摆摊、跑商、寄售、换货 | 货币、商誉、跨区资源 |
| 修习 | 书法、练剑、读书、打坐、烹茶、听曲 | 属性、心境、NPC 入口 |
| 建设 | 升级工坊、家园、仓库、车马、铺面 | 自动化、容量、地区影响 |

## 4. 世界结构

### 地图层级

```
九州全国图
  → 大地区模块（江南 / 巴蜀 / 西域 / 京畿 ...）
    → 小地区（2-8 个，如金陵、临安、龙泉、太湖）
      → 场景点（街市、工坊、田圃、书院、码头、山路）
        → 内景或小游戏（工序、对话、事件、修习）
```

### 大地区设计原则

每个大地区必须有：

- 一套地貌主题：颜色、地形、障碍、建筑和气候。
- 2-8 个小地区：主城、工坊、郊野、资源点、商路节点。
- 3-10 个主要人物：至少 1 位匠人、1 位商人、1 位文人或官署人物、1 位生活类人物。
- 一组地方资源：原料和半成品来源。
- 2-5 门招牌工艺：本轮在文档层面对 11 个地区全部列清，标出供应链、人物线和美术缺口；后续工程可拆分并行。
- 1 条地区主线：为什么玩家要来到这里，解决什么地方矛盾。
- 1 个节令或地方大事件：灯市、茶会、赛巧、开窑、巴扎、庙会、雪山转场等。

## 6. 江南地区模块

江南是 11 个并行设计地区之一，定位为水乡商埠、文人雅集、茶伞丝瓷、铸剑青瓷。现有代码中江南已有“苏杭水市、金陵城、临安水市、龙泉山坊、太湖织埠、城郊百工院”等小地区；本节保留其细节，并与其它地区按同一规格验收。

### 6.1 金陵城

定位：文人、织造、金箔、灯市、码头贸易。

交互点：

- 秦淮灯市：灯彩、夜市、节令订单。
- 云锦局：云锦织造、织机协作小游戏。
- 金箔作：锤金箔节奏小游戏，可供应高级装饰材料。
- 乌衣书院：诗文、书法、识字、地方见闻。
- 江平码头：跑商、寄售、外地货物。
- 城南菜圃：首个轻量种植点。

主要 NPC：

| 人物 | 定位 | 玩法入口 |
|---|---|---|
| 宁辞秋 | 文人书法家，复合型宋代文人原型 | 诗词问答、书法小游戏、题跋订单 |
| 沈云梭 | 云锦织造师 | 云锦线、织机联作、织造声望 |
| 顾薄金 | 金箔匠 | 锤金节奏、宫廷订单、材料精炼 |
| 乔照夜 | 灯彩匠 | 灯市节令、灯彩设计、夜间事件 |
| 方季衡 | 码头商人 | 物流、寄售、外地材料 |
| 柳拂衣 | 采买夫人 | 日常订单、流行风向、生活度反馈 |
| 小满 | 菜圃少年 | 种植教程、节气作物 |

宁辞秋说明：他可以“参考苏轼式的文人气质”，但不应硬说成“苏轼原型在金陵”。苏轼与金陵并非最强绑定，直接拼贴名人容易显得轻浮。更好的做法是做“复合文人原型”：豪放、懂诗、会书法、关心民生，也会对手艺商业化提出尖锐问题。

宁辞秋关系线：

- 初识：看不起玩家，称你“手上有力，心里未必有墨”。
- 熟络：请求玩家带一张好纸、一锭好墨，开启文房供应链。
- 知己：玩家在书法小游戏中完成题跋，为自己的作品提升精神度与声望。
- 终章：若玩家过度商业化，他写文讽刺；若玩家守艺过度，他劝你“器物要入人间”。

### 6.2 临安水市

定位：茶、伞、食饮、湖港生活。

交互点：龙井茶园、油纸伞铺、湖畔茶肆、船坞、市桥鱼摊、药铺。

玩法重点：

- 烹茶与制茶成为生活社交入口。
- 油纸伞连接竹篾、纸、桐油和雨季订单。
- 湖港能解锁水路运输，降低江南内部物流成本。

### 6.3 龙泉山坊

定位：青瓷、宝剑、山溪矿冶。

交互点：龙泉剑炉、青瓷窑、山溪铁矿、练剑场、柴炭场。

玩法重点：

- 武器锻造不是单纯攻击力系统，而是“器物系统”：剑可作为订单、礼物、身份象征、护商工具。
- 练剑提高体魄和声望，可解锁山路护送、武人 NPC 和更高风险采矿。
- 青瓷和宝剑共享燃料、窑火、矿冶，形成江南内部产业链。

### 6.4 太湖织埠

定位：缂丝、桑蚕、渔桑、湖港商贸。

交互点：桑园、织埠、渔村、湖船、染线坊。

玩法重点：

- 玩家可以种桑养蚕，稳定供应生丝。
- 缂丝需要长期熟练度和高心境，不适合快节奏刷产物。
- 太湖渔获可用于烹饪、馈赠和夜市订单。

### 6.5 城郊百工院

定位：玩家家园与长期建设中心。

设施：

- 小田圃：种靛草、桑、茶、稻、药草。
- 工棚：低阶制作、修理、工具升级。
- 仓房：材料分类、保鲜、珍品陈列。
- 客舍：邀请 NPC 暂住或来访。
- 书案：研究图谱、整理百工谱。
- 炉台：烹饪、烹茶、药膳。

## 7. 玩家属性体系

现有四维数值“传承 / 市场 / 生活 / 精神”应保留为**世界与手艺的生态指标**。新增玩家属性，用于描述“玩家这个人”。

| 属性 | 含义 | 影响 |
|---|---|---|
| 手艺 | 手上功夫与工序理解 | 制作质量、失败率、可学工艺 |
| 见闻 | 地方知识、材料辨识、历史掌故 | 解锁对话、地图事件、鉴定 |
| 人缘 | 与人打交道的能力 | NPC 好感收益、议价、求助 |
| 商誉 | 诚信、交付能力、市场信任 | 高级订单、寄售、商路优惠 |
| 体魄 | 远行、采集、练剑、抗疲劳 | 山路、矿洞、护商、长途行脚 |
| 心境 | 耐心、审美、精神稳定 | 书画、制瓷、刺绣、唐卡等精细工艺 |

### 称号体系

称号不要只是等级名，而应是 NPC 对玩家的社会评价。

| 阶段 | 称号 | 触发印象 |
|---|---|---|
| 0 | 粗手生人 | 会做粗活，但不懂门道 |
| 1 | 入门匠 | 能独立完成基本工序 |
| 2 | 走方匠 | 能跨地区找材料、接订单 |
| 3 | 名匠 | 某门手艺有代表作 |
| 4 | 宗匠 | 能教徒弟、修复断链工艺 |
| 5 | 神匠 | 多地区、多工艺、多人物线共同承认 |

注意：“粗人”可以作为个别 NPC 的刻薄称呼，但不建议作为系统正式称号，容易显得羞辱玩家，也不符合我们想传达的温度。

## 8. 日程与核心循环

建议采用“日 + 时段 + 季节”的生活循环，而不是完全实时 MMO。

### 单日循环

```
清晨：查看天气、体力、信件、订单、NPC 日程
上午：采集 / 种植 / 拜访 / 修习
下午：制作 / 精炼 / 行脚 / 交易
黄昏：市集 / 摆摊 / 交付 / 社交
夜间：整理背包 / 读书 / 记录百工谱 / 休息
```

### 单局节奏

- 1 天：5-12 分钟。
- 1 季：7 或 10 天。
- 1 个地区主线：2-4 季。
- 每个大地区章节：文档层面先具备 2-4 季的主线、生活循环、人物线和可持续订单；具体实装时可并行拆任务。
- 长线存档：几十小时逐步解锁地区、人物、工艺和家园。

### 为什么不要一开始做真实时间

真实时间页游会带来签到、体力、冷却、错过事件等压力，容易毁掉题材本来的沉浸感。首版更适合“玩家推进时间”，后续联网版再做异步委托和每日市集。

## 9. 背包与资源系统

当前资源分层很好，应继续扩展为“材料有来源和品质”的系统。

### 背包字段建议

| 字段 | 示例 | 作用 |
|---|---|---|
| id | `ironIngot` | 资源键 |
| 数量 | 3 | 基础库存 |
| 品质 | 粗 / 良 / 精 / 臻 | 影响制作上限 |
| 产地 | 龙泉 / 临邛 / 大冶 | 影响 NPC 评价、订单偏好 |
| 新鲜度 | 茶青、鱼、药草 | 推动季节和时间管理 |
| 标记 | 任务物 / 礼物 / 收藏 | 防误卖、驱动剧情 |

### 仓储设计

- 背包：随身容量有限。
- 家仓：材料主仓库。
- 工坊架：当前制作常用材料。
- 珍品阁：代表作陈列，提高声望，也可触发 NPC 参观。
- 商栈寄存：跨区运输中转，需费用或商誉。

---

## 10. NPC 系统

NPC 不是“好感条”，而是地方生活的核心。

### NPC 数据模板

每个主要 NPC 至少包含：

- 基础：姓名、职业、地区、小地区、活动范围。
- 性格：守旧、豪爽、精细、势利、温厚、孤僻等。
- 日程：平日、雨天、节日、事件期间在哪里。
- 偏好：喜欢什么物、讨厌什么物、看重什么属性。
- 关系阶段：初识、熟络、信任、知己。
- 功能：教技能、给订单、开商路、给线索、参与联作。
- 个人矛盾：为什么需要玩家。
- 结局影响：玩家如何改变他/她的命运。

### 交互方式

| 交互 | 不应只是 | 应该成为 |
|---|---|---|
| 对话 | 每日刷一句 | 获取地方情报、改变关系、触发事件 |
| 赠物 | 万能送礼 | 根据职业、审美、季节和产地判断 |
| 代办 | 跑腿 | 连接地图、供应链和人物矛盾 |
| 联作 | 交材料 | NPC 参与工序，改变成品风格 |
| 切磋 | 加好感 | 书法、锻剑、烹茶、诗会、棋局等小游戏 |
| 邀请 | 叫到家园 | 让百工院成为人物关系的汇聚地 |

### NPC 对属性的反馈

- 手艺低：匠人不愿教核心工序。
- 见闻低：文人不与你谈掌故，只说客套话。
- 人缘低：商人不给好价，官署人物不放行。
- 商誉低：订单需要押金，失败会降声望。
- 体魄低：山路、矿洞、护商任务受限。
- 心境低：书画、刺绣、制瓷等精细工艺更容易失败。

## 11. 工艺与制作系统

工艺必须保持“真实工序链 + 可玩微交互 + 统一结算”的原则。

### 工艺流程

```
地方知识 / NPC 指点
  → 获得图谱或工序
  → 准备材料
  → 逐道工序微交互
  → 生成品质、风格、瑕疵
  → 订单 / 馈赠 / 陈列 / 联作 / 解锁
```

### 随机性来源

随机性不是纯抽奖，应来自可理解因素：

- 玩家属性。
- 材料品质与产地。
- 工具等级。
- 天气与季节。
- NPC 是否协助。
- 工序是否省略。
- 玩家小游戏表现。

### 成品字段

成品不应只是一种 `product` 数量，至少高级成品应有实例化字段：

- 名称：如“秋水纹青瓷盏”。
- 工艺：青瓷。
- 品级：良 / 精 / 臻 / 绝。
- 风格：守正、趋时、民用、雅玩、宫造。
- 瑕疵：火候偏、釉色浅、纹样乱等。
- 作者：玩家与协作 NPC。
- 题跋或故事：由文人 NPC 或事件生成。

---

## 12. 武器锻造与武艺

武器锻造可以加入，但不能把游戏带偏成刷怪 RPG。

### 正确方向

- 武器是“器物”和“身份”，不是单纯攻击力。
- 剑、弓、刀、甲可以作为订单、礼物、护商凭证、武人关系线和收藏品。
- 练剑、射艺、护商提高体魄与声望，开启高风险地图点。

### 武器工艺示例：龙泉宝剑

工序：

1. 选铁备炭：铁锭产地决定韧性。
2. 折叠锻打：节奏小游戏，稳定度影响纹理。
3. 淬火回火：火候小游戏，失败会脆裂。
4. 研磨开刃：拖拽/耐心小游戏，影响锋利与美观。
5. 装具题名：可找文人题名、金工装具、漆工做鞘。

用途：

- 卖给武馆或镖局。
- 赠给武人 NPC。
- 用于护送商路，降低断供风险。
- 作为名作陈列，提升声望。

## 13. 种田与家园

种田应服务古代生活和供应链，而不是复制现代农场经营。

### 作物方向

| 作物 | 用途 |
|---|---|
| 靛草 | 蓝染、蜡染 |
| 桑树 | 蚕丝、织造 |
| 茶树 | 制茶、社交、礼物 |
| 稻麦 | 烹饪、酿造、节令任务 |
| 苎麻 | 夏布、纸纤维替代 |
| 药草 | 医馆任务、香、药膳 |
| 花木 | 染料、香料、庭院美观 |

### 家园建设路线

- 初始：一间破工棚、一小块田、一只背篓。
- 中期：仓房、茶炉、织机、砧台、纸槽、晒场。
- 后期：百工院，可邀请 NPC 联作、办展、收徒。

### 自动化边界

可以有轻度自动化，如水车、晒架、雇工、学徒代劳，但不要让玩家完全脱离工序。主题是“手艺”，不是全自动流水线。

## 14. 经济与订单

订单系统应从“交一个成品换钱”升级为多层市场。

### 订单类型

| 类型 | 示例 | 特点 |
|---|---|---|
| 日常订单 | 竹篮、蓝布、茶、纸伞 | 稳定收入，生活度高 |
| 匠人委托 | 某 NPC 要指定材料或器物 | 提升关系，带故事 |
| 商路订单 | 外地商人批量采购 | 受市场度与运输影响 |
| 宫造订单 | 景泰蓝、花丝、雕漆 | 高价值，高门槛 |
| 节令订单 | 灯市、茶会、开窑、婚礼 | 时限强，带事件 |
| 修复订单 | 古器、旧谱、老宅构件 | 见闻和手艺要求高 |

### 市场波动

- 地区偏好：江南喜雅物，京畿喜重器，西域重换货。
- 季节偏好：雨季纸伞、夏季凉席、冬季香与漆器修复。
- 事件偏好：文旅热、官府采办、断供、流行风向。
- 声望影响：同一件物，名匠价与粗工价不同。

## 16. 主线叙事结构

主线不应是“拯救全国非遗”的口号，而是“补全百工谱”的具体旅程。

### 主线阶段

1. **半边匾**：老师傅留下“百工”匾，玩家在江南站稳脚跟。
2. **第一门手艺**：通过蓝染或竹编理解工序、材料和人。
3. **第一条商路**：发现本地做不出好东西，需要外地材料。
4. **第一位知己**：NPC 不再只把玩家当外人。
5. **第一件名作**：成品被题名、收藏或用于地方事件。
6. **百工谱残页**：每个地区主线给一页残谱，记录一种“地方如何养手艺”。
7. **归位不是复古**：终章要求玩家回答：让手艺归位，是回到过去，还是进入生活？

### 结局方向

- 守正宗匠：传承极深，但市场与生活需要玩家承担代价。
- 市井名匠：手艺进入日常，可能牺牲部分古法纯度。
- 商路巨匠：供应链强大，百工繁荣，但容易商业异化。
- 山野隐匠：精神极高，人物关系深，但影响范围有限。
- 空名浮匠：声望高、实质浅，被结局报告尖锐批评。
- 神匠归位：不是全数值拉满，而是关键人物、代表作、供应链和传承都形成活态平衡。

## 17. 联网页游化路线

“中小型页游”可行，但应分阶段，不能一开始就做实时大世界。

### 阶段 A：单机长线存档

- localStorage / IndexedDB。
- 完整江南章节。
- 支持离线游玩。

### 阶段 B：账号与云存档

- 登录、云端存档、多设备同步。
- 云端只保存可序列化 `GameState`，延续现有 `StorageAdapter`。
- 不改变核心玩法。

### 阶段 C：异步社交

- 拜访好友百工院。
- 查看他人代表作陈列。
- 给好友下委托单。
- 市集榜单、节令活动榜、作品展。

### 阶段 D：轻量多人经济

- 玩家寄售材料或成品。
- 系统限制价格区间，防刷钱。
- 珍品可展示但不可随意交易，避免破坏养成。

### 阶段 E：实时活动，仅作为后期

- 节令灯会、开窑共赏、拍卖会。
- 小规模房间或实例，不做全服同屏大地图。

## 18. 技术落地建议

现有架构方向是正确的，应继续坚持：

- `engine`：所有数值、时间、资源、NPC 关系、任务、订单、种植、制作结算。
- `data`：地区、NPC、作物、工具、订单、节令、图谱、食谱。
- `game`：Phaser 场景、寻路、碰撞、交互点、角色移动。
- `components`：HUD、背包、日程、NPC 对话、工坊、家园、地图。
- `storage`：本地和云端存档适配器。

### 新增数据类型建议

| 类型 | 作用 |
|---|---|
| `PlayerProfile` | 玩家属性、称号、技能经验 |
| `CalendarState` | 日期、季节、时段、天气 |
| `NpcSchedule` | NPC 日程与特殊事件位置 |
| `RelationshipStage` | 好感阶段和人物事件 |
| `CropDef` / `FarmPlot` | 作物与田地 |
| `ItemInstance` | 高级成品实例化 |
| `OrderDef` / `ActiveOrder` | 订单系统 |
| `RecipeDef` | 食物、茶、药膳、工具配方 |
| `RegionKnowledge` | 地方见闻和解锁线索 |

### 输入与界面

- 保持 WASD / 方向键移动，E 交互。
- 新增快捷轮盘：背包、地图、日程、百工谱、工具。
- 文本重的界面继续用 DOM，避免塞进 canvas。
- 手机端暂不优先，先保证桌面 Web 体验完整。

## 19. 里程碑

- 补一份“当前玩法流程说明”。

验收：新玩家能从主菜单进入、采料、制作、接订单、与 NPC 交谈、去大地图。

### M0.9 全地区生活规格

目标：让 11 个地区都具备“能生活”的文档级闭环。

- 列清每个地区的日程：清晨、上午、下午、黄昏、夜间。
- 列清玩家属性在每个地区的作用：手艺、见闻、人缘、商誉、体魄、心境。
- 列清田圃、资源点、工坊、市场、交通点和家园/客栈等生活入口。
- 每个地区至少 3-10 个主要人物，至少 2 条人物线方向。
- 每个地区写明天气对采集、NPC 位置、订单和活动的影响。

验收：玩家在任一地区都能形成“今天要做什么”的自然计划。

### M1 全地区核心章节规格

目标：每个地区都有可成为章节主线的核心内容。

- 每个地区一张核心地图或子地区母版。
- 每个地区至少一条代表人物线。
- 每个地区至少三类玩法点：工艺、生活、交易/路线。
- 每个地区至少一个节令或地方事件。
- 代表作实例化与题名、作者、题跋、陈列、赠礼、售出关系在全地区可映射。

验收：玩家能在每个地区完成一条地方主线，并产出一件有名字、有作者、有地区痕迹的作品。

### M2 全地区地区闭环

目标：11 个地区都达到“可长期停留”的设计密度。

- 每个地区的小地区、工坊、资源点、交通点和家园/客栈入口列清。
- 每个地区至少 2-5 门招牌工艺形成材料、工序、订单和人物关系。
- 每个地区有地区结局、声望段落和 NPC 人情回声。
- 每个地区完成美术缺口清单，与 [全地区美术设计规格.md](全地区美术设计规格.md) 对齐。

验收：任一地区都能支撑持续游玩、订单、人物推进、作品收藏和路线联动。

### M3 跨区供应链

目标：地图真正活起来。

- 同步打通铁、竹、银、铜、靛蓝、蚕丝、茶、瓷土、纸墨、颜料、玉料等跨区供应链。
- 加入运输成本、商路风险、护商、断供和地区声望联动。
- 每条关键供应链都标出上游产地、中转路线、下游工艺和美术表现。

验收：玩家为了做一件高级作品，会主动在多个地区之间寻找材料、人物和订单，而不是被单线任务硬拖走。

### M4 开放世界生活层

目标：自由玩法成型。

- 家园建设。
- 稳定种植、钓鱼、烹饪、练剑、读书、烹茶。
- NPC 拜访家园。
- 地区节令循环。

验收：即使不推主线，玩家也能在世界中自然生活 1-2 小时。

### M5 云存档与异步社交

目标：页游化第一步。

- 账号和云存档。
- 百工院拜访。
- 玩家委托板。
- 作品展与节令榜。

验收：社交增强长期留存，但不破坏单人叙事。

## 20. 必须警惕的问题

### 20.1 范围过大

覆盖整个中国、全链路工艺、种田、NPC、锻造、联网，这些放在一起是一个巨型项目。当前应先用统一文档规格控制范围：11 个地区同时列清地图、人物、工艺、建筑、交互点和美术缺口，再拆成可并行实装任务。

### 20.2 美术资源不能继续独自前进

美术已经走得很快，但玩法骨架没有跟上。后续每批美术都应绑定一个可验证玩法目标。例如“西域素材”必须对应“巴扎交易 + 驼队运输 + 玉石采集”，否则只是漂亮素材库。

### 20.3 不要把工艺降级为材料合成

如果玩家只是收集 `ironIngot x2` 点按钮出剑，就失去项目特色。至少旗舰工艺必须有工序、人物、产地、品质和故事。

### 20.4 不要迷信真实地图

真实中国地图过大且历史时期复杂。应该使用“文化地理模块”，避免硬套现代行政区。江南、巴蜀、黔滇、京畿等是玩法地图，不是地理考试。

### 20.5 名人原型要谨慎

可以有复合型人物气质，但不要把历史名人随意移植。宁辞秋可以有苏轼式的豪放和书法趣味，但应是独立虚构人物，并与金陵地方线自洽。

### 20.6 民族和宗教内容要慎重

苗银、唐卡、藏香、艾德莱斯等内容不能只做“异域装饰”。要有资料来源、工序尊重、人物语境和避免刻板化的文本规范。

### 20.7 “粗人”称号要控制语气

从粗到精的成长很好，但系统称号最好用“粗手生人”等更具时代感、少羞辱感的表述。刻薄评价可以放到个别 NPC 台词里，作为关系改善的对比。

### 20.8 联网经济风险

一旦玩家市场开放，就会有刷钱、囤货、脚本、通胀、交易纠纷和服务器成本。异步社交应先从展示、拜访和委托开始，不急着开放自由交易。

### 全地区验收标准

玩家进入任一地区后，都应能自然经历：

- 认识本地街市或资源点，知道这个地区为什么长成这样。
- 接触至少一位本地匠人、一位路线/商贸人物、一位生活或文化人物。
- 完成一次本地采集或资源准备，理解材料从哪里来。
- 完成一次代表工艺或生活活动，知道自己操作的道具和工序意义。
- 遇到一次地方订单、节令活动、护商/断供或人物请求。
- 产出或改造一件带地区痕迹的作品，并决定卖出、赠予、陈列或继续联作。
- 在地图上看见通往其它地区的路线理由，而不是只从菜单跳转。

只有 11 个地区都满足这组文档验收，才算全国地图值得进入下一轮实装拆分。
