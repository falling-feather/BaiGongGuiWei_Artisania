# M1.11 三晋煤铁醋坊人工地图记录

日期：2026-06-15

## 目标范围

本轮承接 M1.10，关闭三晋直接程序缺口中的 `sanjin-coal-yard` 与 `sanjin-vinegar-yard` 人工地图 JSON。目标不是最终美术摆位，而是让煤铁窑塬与清徐醋坊从“票号/漆院通道指向的目标”变成运行时真实可落地、可交互、可被守卫测试覆盖的小地区。

正式移动红线继续保持：总览地图只作调试和路线提示，玩家正式迁移仍需在街景中靠近 `subregionGate`、城门、商路节点或交互点后触发。

## 协作团队

按 `程序大型化协作指引.md` 的 M1 固定团队数，本轮启动 6 个只读审计团队：

1. 集成与结构团队：审计 runtime layout schema、注册与测试规则。
2. 江南 / 巴蜀样板团队：审计成熟人工地图的道路、工坊、采集点、NPC 和活动点写法。
3. 岭南 / 黔滇样板团队：审计生活链、商贸链、工艺链在布局中的表达。
4. 荆楚 / 赣鄱 / 徽州样板团队：审计资源采集、窑口、文房与渡口布局密度。
5. 京畿 / 三晋 / 雪域 / 西域团队：审计三晋现有 ID、票号/漆院图和本轮应关闭的 gap。
6. QA 与浏览器团队：设计自动化和内置浏览器 smoke 路线。

| 团队 | 审计结论 |
| --- | --- |
| 集成与结构 | schema、注册点和测试清单正确；`regionChapters.nextActions` 中旧的“补煤铁/醋坊人工图”口径已同步改为后续复验与 M3 经济。 |
| 江南 / 巴蜀样板 | `jiangnan-longquan` 适合煤铁/冶炼布局，`bashu-jinli` 适合醋坊生活街；本轮采用 56x30、多横街和三处区内 gate 的成熟写法。 |
| 岭南 / 黔滇样板 | 煤铁图需要突出重货、火候和账房，醋坊图需要突出民生日用、封坛和日账；当前点位与文案符合两条链路。 |
| 荆楚 / 赣鄱 / 徽州样板 | 资源点、窑口/作坊和文房式账本点位可以共存；当前图不需要额外系统扩写，只需保证入口可达。 |
| 京畿 / 三晋 / 雪域 / 西域 | 本轮应收束在 `sanjin-coal-yard`、`sanjin-vinegar-yard` 两个 layout ID，不扩大到 M3 多轮经济。 |
| QA 与浏览器 | 除 shipped layout、入口、冻结 ID 外，新增 NPC 站位可达和三晋四街景 gate 闭环守卫；浏览器 smoke 需从票号实际走 gate 进入新增图。 |

## 程序落点

- 新增 `src/data/mapLayouts/sanjin-coal-yard.json`
  - 包含 `harvest-coal`、`harvest-iron-ore`、`smelt-iron` 三个产业点。
  - 包含活动 `sj-coal-iron-yard`。
  - 包含 NPC `sj-yaoyuan-han`。
  - 包含通往 `sanjin-piaohao`、`sanjin-lacquer-yard`、`sanjin-vinegar-yard` 的区内 gate，以及通往 `jingji` 的地区 gate。
- 新增 `src/data/mapLayouts/sanjin-vinegar-yard.json`
  - 包含工艺 `aged-vinegar`。
  - 包含活动 `sj-vinegar-yard`。
  - 包含 NPC `sj-cu-langzhong`。
  - 包含通往 `sanjin-piaohao`、`sanjin-coal-yard`、`sanjin-lacquer-yard` 的区内 gate，以及通往 `jingji` 的地区 gate。
- 更新 `src/data/mapLayout.ts`
  - 注册两张 JSON 到 `RUNTIME_MAP_EDITOR_SNAPSHOTS`，由运行时统一转换为 `RUNTIME_MAP_LAYOUTS`。
- 更新地图与冻结测试
  - `mapLayout.test.ts` 固定两张图的 shipped 顺序和关键交互点。
  - `priorityMapLayoutEntrypoints.test.ts` 增加 M1 shipped layout 覆盖。
  - `priorityFrozenIds.test.ts` 冻结两张新 layout。
- 更新 `src/data/regionChapters.ts`
  - 三晋章节 gap 不再记录煤铁/醋坊人工 JSON 缺失，改为记录已入库并保留 M3 级多轮本金/利钱统计缺口。
