## 0. 一句话定位

**《百工归位》是一款以中国非物质文化遗产（传统手工艺）为题材的 Web 端策略经营 + 卡牌模拟游戏。** 玩家不是「保护某一个非遗项目」，而是经营一整座活着的「百工镇」——让木匠、染坊、陶工、皮影师、竹编师、银匠等百业在传承、市场、生活、精神四股张力中重新「归位」，最终生成一份属于玩家自己的「百工镇命运报告」。

设计母题（来自[简介.md](../简介.md)）：
> 非遗不是「放进博物馆就保住了」，也不是「做成爆款就成功了」。

我们要把「非遗传承」做成一个**可玩的复杂系统**，而不是一部宣传片。

---

## 1. 设计支柱（Design Pillars）

一切玩法、内容、数值取舍都必须服务于以下四根支柱。任何新功能若不能落到某根支柱上，应被质疑。

1. **张力而非答案（Tension, not Answers）**
   四维数值没有「全部拉满」的最优解。玩家永远在「保住手艺」与「养活手艺」之间做痛苦权衡。
2. **活态生态（Living Ecosystem）**
   手艺之间相互依赖、相互组合（竹编+灯具、蓝染+可穿戴、皮影+数字剧场）。一门手艺的兴衰会沿生态链传导。
3. **人的故事（People, not Props）**
   学徒、匠人有性格、有成长、有离去。传承的载体是「人」，不是「工序数据」。
4. **千镇千面（A Town of One's Own）**
   随机事件 + 玩家抉择 + 多结局，使每一局的百工镇命运都不同，结局生成可分享的叙事报告。

## 2. 核心玩法循环

### 2.1 宏观循环（回合 = 一个「季」）

```
观察镇情（四维数值 + 事件） 
   ↓
决策（培养学徒 / 改造产品 / 接订单 / 办展演 / 技艺联动）
   ↓
结算（资源、数值、口碑、解锁判定）
   ↓
随机事件（市场风向、天灾、人来人往）
   ↓
进入下一季 …… 直至触发结局条件 → 生成命运报告
```

### 2.2 每回合玩家的关键抉择（来自简介）

1. 培养哪位学徒？
2. 把哪门手艺改造成现代产品？
3. 是否接商业订单？
4. 是否牺牲传统工序换取销量？
5. 是否举办展演、课程、联名、旅游体验？

### 2.3 微观循环（单门手艺的工艺流程模拟）

每门手艺拥有一条**工序链（Process Chain）**，这是「模拟工艺流程」主题的落点：

> 例：蓝染 = 采蓝 → 制靛 → 建缸（养缸） → 扎结 → 浸染（多次） → 氧化 → 漂洗 → 晾晒

- 玩家可在工序级别做权衡：**省略/简化工序** → 提升产量与市场度，但损耗传承度与精神度。
- 工序需要**资源**（植物染料、水、布料）与**人力**（匠人/学徒的工时与熟练度）。
- 工序质量决定产物品级（粗品 / 良品 / 精品 / 臻品），品级影响售价与口碑。

## 3. 四维数值系统（游戏的灵魂）

| 数值 | 含义 | 太低 | 太高 |
| --- | --- | --- | --- |
| 传承度 Heritage | 技艺的纯正与延续 | 技艺断代 | 过度保守、脱离时代 |
| 市场度 Market | 商业生存能力 | 活不下去 | 沦为廉价商品 |
| 生活度 Life | 与普通人日常的贴近 | 离普通人太远 | 被日常消费稀释 |
| 精神度 Spirit | 文化内核与意义感 | 失去灵魂 | 变成空洞仪式 |

设计要点：
- **没有「赢」的稳态**：四维各自有「健康区间」（如 30–70）。低于下限触发**危机事件**，高于上限触发**异化事件**。
- **此消彼长**：几乎每个行动都同时正向和负向地推动多个数值（trade-off 矩阵），杜绝无脑最优解。
- **镇级 vs 手艺级**：四维既有「全镇总览值」，也有「每门手艺各自的四维」，全镇值由各手艺加权汇总。
- **结局由四维的最终组合 + 历史轨迹决定**，而非单一分数。

> 数值的实现见 [开发者文档.md](开发者文档.md) 的「数值引擎」章节与 `src/engine/metrics.ts`。

## 4. 内容系统

### 4.1 手艺（Craft）

每门手艺是一张「数据卡 + 工序链 + 生态依赖」：

| 字段 | 说明 |
| --- | --- |
| id / 名称 / 地域 | 唯一标识、显示名、所属省份/文化区 |
| 依赖资源 | 如蓝染依赖：植物染料、水、布料 |
| 工序链 | 有序工序节点，每节点有资源/人力/质量影响 |
| 传承风险 | 触发危机的条件（原料消失、审美老化、无人学等） |
| 四维基线 | 该手艺的初始/固有四维倾向 |
| 联动标签 | 用于跨手艺组合（见 4.4） |
| 解锁条件 | 何时对玩家开放 |

首批参考手艺（取自简介 + UNESCO/国家级名录调研）：
蓝染（蓝印花布/扎染）、皮影、陶艺（龙泉青瓷）、木作（榫卯）、竹编、银饰锻造、油纸伞、修谱（雕版/活字印刷）、香料/制香。
后续扩展池（已调研有据）：南京云锦、宋锦、缂丝、苏绣/蜀绣、宣纸、桑蚕丝织、黎族纺染织绣、紫砂、景泰蓝、木拱桥营造、制茶、年画。

### 4.2 学徒 / 匠人（Apprentice / Artisan）

- **性格特质**：影响成长曲线与事件触发（如「守旧」加传承、减市场；「灵巧」易上手现代产品；「躁动」可能出走）。
- **熟练度成长**：通过参与工序累积，分手艺独立成长。
- **师徒制**：老匠人带学徒，存在「传承窗口」——若老匠人离去前未完成传承，技艺可能断层。
- **人来人往**：学徒会因生活度/精神度过低而离开，制造叙事张力。

### 4.3 事件系统（Event）

- **市场事件**：审美潮流变化、爆款风口、原料涨价、外来资本。
- **天时事件**：天灾、节令、政策扶持、文旅热度。
- **人事事件**：匠人病老、学徒出走、媒体报道、联名邀约。
- 事件以**卡牌**形式呈现，多为「带后果的选择」，强化「张力而非答案」。

### 4.4 技艺联动（Craft Synergy）—— 耐玩性核心

- 通过**联动标签**匹配，解锁组合产品：
  - 竹编 + 灯具 → 现代竹编灯具
  - 蓝染 + 可穿戴 → 蓝染数码配饰
  - 皮影 + 数字剧场 → 沉浸式皮影演出
- 联动既是**玩法深度**来源，也是**长线解锁/进阶**主线的脉络。

### 4.5 解锁、拓展与升级（长线驱动）

呼应「清明上河图式」愿景：
- **地域拓展**：以省份/文化区为单位逐步开放，玩家从一镇出发，向全国铺展。
- **技艺解锁树**：完成特定成就/联动后解锁新手艺与新工序。
- **升级**：手艺可升级工坊、工具、品牌，影响产能与品级上限。
- **代币/积分经济**：镇内流通货币 + 跨局成就积分，用于解锁与长期养成。

---

## 5. 结局与叙事

- **结局条件**：到达回合上限 / 关键数值崩盘 / 达成特定愿景。
- **命运报告**：基于四维终值、历史轨迹、存活手艺、学徒结局、关键抉择，**程序化生成**一段叙事文本 + 数据图谱，可截图分享。
- 报告是「千镇千面」支柱的兑现点，也是自然的社交传播钩子。

## 6. 技术架构总览

> 详细落地见 [开发者文档.md](开发者文档.md)。此处只给「为什么这么选」。

| 层 | 选型 | 理由 |
| --- | --- | --- |
| 前端框架 | React 18 + TypeScript + Vite | 生态最大、组件化强、长期可演进 |
| 状态管理 | Zustand | 轻量、贴合游戏状态机、易测试 |
| 样式 | Tailwind CSS | 原型期快速布局 |
| 核心引擎 | 纯 TypeScript（`src/engine`） | **框架无关**、纯函数、可单测、可移植到后端 |
| 内容数据 | 数据驱动（`src/data`） | 内容与逻辑解耦，支撑「全工艺全地域」海量扩张 |
| 存档 | `StorageAdapter` 接口（localStorage 实现） | 现在单机，未来无缝切到后端云存档 |
| 后端（蓝图） | Node + TS + Fastify + PostgreSQL/Prisma | 同语言栈、低心智负担；支撑账号/排行榜/云存档 |

**最关键的架构原则：游戏规则（engine）与渲染（React）彻底解耦。** 引擎是一组纯函数 `(state, action) => newState`，UI 只负责「展示状态 + 派发行动」。这保证了：可单元测试、可换皮、可上后端做权威结算（反作弊）、可被任意新开发者快速理解。

## 7. 迭代路线图（Roadmap）

里程碑按「可玩性递增」组织，而非按功能堆砌。

### M0 · 最小内核（当前 demo 目标）
- 1–2 门手艺、四维数值、回合循环、1 个工序链、若干事件、结局报告雏形。
- 纯前端 + localStorage 存档。
- **验收**：能完整玩通一局并看到结局报告。

### M1 · 生态成形
- 3–4 门手艺、学徒系统、技艺联动 1.0、事件库扩充、资源经济闭环。

### M2 · 长线驱动
- 解锁树、手艺/工坊升级、代币经济、地域拓展（首个跨省）。

### M3 · 联网与社交
- 后端接入（账号、云存档、排行榜）、命运报告分享、成就系统。

### M4 · 走向「清明上河图」
- 多地域全工艺铺开、主线剧情、节令/文旅大事件、内容工具链（让非工程师也能产出手艺数据）。

## 8. 风险与对策

| 风险 | 说明 | 对策 |
| --- | --- | --- |
| 内容体量爆炸 | 全工艺全地域意味着海量数据 | 严格数据驱动 + 内容 Schema + 校验脚本；先做内容模板 |
| 数值平衡难 | 四维 trade-off 容易失衡 | 引擎纯函数化 + 可重放测试 + 数值表外置可调 |
| 题材失真 | 非遗题材需尊重事实 | 内容以权威名录为据，标注来源；避免戏说 |
| 范围蔓延 | 愿景宏大易失焦 | 严守里程碑，M0 只做最小内核 |
| 单人/小团队产能 | 工作量极大 | AI 协作 + 文档驱动交接 + 模块化，保证任意环节可转手 |

---

## 9. 文化呈现原则

- **有据可查**：手艺设定尽量对齐 UNESCO/国家级非遗名录与公开资料，关键条目在数据中标注来源。
- **尊重而不猎奇**：呈现工艺的真实难处与价值，不做空洞符号堆砌。
- **传承的复杂性**：通过「张力数值」让玩家体会到传承没有标准答案，这正是教育意义所在。

## 12. 工艺大类抽象（游戏数据模型）

> 上面上百个具体项目，在引擎里**归并为有限的「工艺大类」**，
> 每个大类共享一套「工序模拟模板」，具体项目只是换皮（材料/配方/产物/地域）。
> 这样既能海量铺内容，又不需要为每个项目单独写玩法代码。

| 工艺大类 `CraftCategory` | 共享工序模板 | 代表项目（换皮） |
|---|---|---|
| `dyeing` 染织 | 备料→建缸/染液→上图案→反复染→固色晾晒 | 蓝染、蜡染、扎染、香云纱 |
| `weaving` 织造 | 选丝→染线→挑花/编程→投梭织造→整理 | 云锦、宋锦、黎锦、土家织锦 |
| `embroidery` 刺绣 | 描稿→劈丝→配色→针法绣制→装裱 | 苏绣、湘绣、苗绣、缂丝 |
| `ceramics` 陶瓷 | 练泥→拉坯→修坯→施釉→装窑→烧成 | 景德镇瓷、紫砂、龙泉青瓷、建盏 |
| `weaving_craft` 编结 | 选料→剖丝/刮青→起底→编织→收口 | 竹编、草编、藤编、柳编 |
| `woodwork` 木作 | 选材→画线→开榫凿卯→雕刻→打磨/髹饰 | 东阳木雕、榫卯、红木家具、根雕 |
| `metalwork` 金工 | 熔炼→锻打/拉丝→成型→錾刻/掐丝→点蓝/抛光 | 苗银、花丝镶嵌、景泰蓝、铁画 |
| `lacquer` 漆艺 | 制胎→刮灰→髹漆（多遍）→雕刻/镶嵌→推光 | 雕漆、脱胎漆器、螺钿 |
| `paper` 纸艺 | 备料→（捞纸/刻版/扎制）→印刷/上色→装裱 | 宣纸、剪纸、年画、风筝、油纸伞 |
| `stone_jade` 玉石 | 相料→设计→粗雕→精雕→抛光 | 玉雕、寿山石、砚台 |
| `food_brew` 食酿 | 选料→处理→发酵/火候→陈化→成品 | 制茶、酿酒、酱醋、糕点 |
| `folk_art` 民艺造型 | 起形→塑造/雕刻→上色→点睛 | 皮影、面塑、糖画、泥塑、唐卡 |

> 实现层面：`Craft` 增加 `category` 字段；工序模拟逻辑按 `category` 分发到对应「工序小游戏」。

## 13. 工艺流程模拟设计（当前缺口的补全）

> 现状：玩家只能走到作坊、按 E 打开面板做数值结算，**没有真正的「上手做」体验**。
> 目标：为每道工序设计一个轻量「微交互小游戏」，让玩家**真的动手**，
> 操作质量（quality 0–1）再反馈到隐藏的四维与产出品质。

### 13.1 微交互原子（可复用的小游戏组件）
| 原子 `MiniGameType` | 操作 | 适配工序 |
|---|---|---|
| `rhythm` 节奏点击 | 在节拍/光标到位时按键 | 锻打、投梭、舂捣、刻版 |
| `drag_path` 描线/运笔 | 沿轨迹拖动鼠标，越贴合越好 | 刺绣针迹、雕刻走刀、画唐卡、剪纸 |
| `ratio_mix` 配比调和 | 拖滑块凑近目标比例 | 染液配方、釉料、配色、和泥 |
| `timing_hold` 火候/时机 | 在温度/进度条的绿区松手 | 烧窑、炒茶、发酵、染色时长 |
| `aim_place` 定位摆放 | 把元素拖到正确位置 | 掐丝、镶嵌螺钿、扎结防染、拼竹篾 |
| `repeat_endure` 重复耐心 | 多次重复并保持稳定度 | 反复浸染、髹漆多遍、捞纸 |

### 13.2 工序 → 微交互映射（示例）
- **蓝染·建缸养缸** → `ratio_mix`（靛+石灰+酒/草木灰配比）+ `timing_hold`（发酵活性）
- **蓝染·扎结防染** → `aim_place`（扎点布局影响花纹）
- **蓝染·反复浸染** → `repeat_endure`（浸染遍数 vs 色牢度，过度浪费工时）
- **景德镇·拉坯** → `drag_path`（手随转盘塑形，抖动降品质）+ `timing_hold`（含水度）
- **景德镇·烧成** → `timing_hold`（窑温曲线，过火/生烧报废）
- **竹编·经纬编织** → `rhythm` + `aim_place`（挑压规律，错一根则纹样乱）
- **苗银·锻打** → `rhythm`；**掐丝/点蓝** → `aim_place` + `ratio_mix`
- **苏绣·绣制** → `drag_path`（针脚走向）+ `ratio_mix`（劈丝配色过渡）
- **制茶·杀青** → `timing_hold`（锅温与翻炒节奏）

### 13.3 品质结算桥接
```
微交互得分 quality(0–1)
   │
   ├─→ 产出品质等级（次品/良品/精品/神品）→ 影响订单收益、声誉
   └─→ 缩放该工序的 metricImpact（做得越好，正向四维收益越高；偷工/失误则惩罚）
```
> 关键：微交互只产出一个 `quality` 标量，**仍由 `reducer.ts` 统一结算四维与经济**，
> 不把规则写进小游戏组件（延续「规则只在引擎」的铁律）。新增 action：
> `RUN_PROCESS` 携带 `quality` 与 `stepResults`。

## 15. 内容实现优先级（分批落地路线）

> 原则：先把「一个大类的工序模拟」打磨成可玩闭环，再横向复制换皮，避免一次铺太宽。

- **第 1 批（夯实模板）**：染织（蓝染，已有）+ 编结（竹编，已有）→ 补齐两者的微交互小游戏与品质结算。验证 §13 框架。
- **第 2 批（旗舰扩容）**：陶瓷（景德镇）、金工（苗银）、刺绣（苏绣）、纸艺（年画/剪纸）——每个大类落 1 个旗舰项目 + 1 套微交互。
- **第 3 批（地图升级）**：实现「城市」层级，把一条街扩成一座城（金陵城样板），引入街↔城出入口与小地图。
- **第 4 批（横向铺量）**：每个大类补到 3–5 个换皮项目（蜡染/扎染、龙泉青瓷/紫砂、湘绣/苗绣…），填充城市与大区。
- **第 5 批（国家地图）**：九州地图与七大区集群，城市间巡游，区域事件与地方订单。
- **第 6 批（人与叙事）**：学徒成长、匠人传记、地方事件链、结局报告接入地图成就。

### 15.1 数据扩展约定（落地时遵循）
- 新工艺只改 `src/data/*.ts`（加 `Craft`，含 `category`、`processChain`、产物、地域、风险）。
- 新体验点只改 `StreetScene.points`（或对应城市场景的点位表）。
- 新微交互在 `src/game/minigames/` 下按 `MiniGameType` 实现，按 `category` 分发；**不写数值规则**。
- 工序的四维/资源结算一律在 `engine/reducer.ts`，新增字段同步 `engine/types.ts`。

> 资料溯源：上述工艺与流布地依据中国国家级非物质文化遗产名录（传统技艺、传统美术类）
> 及联合国教科文组织人类非遗名录（如中国传统制茶技艺、南京云锦、宣纸、龙泉青瓷等）整理。
> 后续逐项实现时，应核对该项目权威介绍以确保工序步骤与地域准确。

# 第三部分 · 地区优先的世界设计

> **设计转向**：游戏是**按地区**组织的，不是按工艺种类。§11 的工艺大类表从此降级为「索引/换皮库」，
> 真正的世界结构以「一方水土养一方手艺」为骨架——每个地区有自己的**地貌、物产、招牌产业（含非遗与非非遗）**，
> 工艺只是这片土地的产物之一。地区之间通过**资源与供应链**彼此依赖，驱动玩家跨区巡游。

## 16. 核心理念：一方水土养一方百工

1. **地貌决定物产**：多山出竹木矿石，沿海出盐与商路，高原出金银与矿物颜料，平原出桑麻五谷。
2. **物产决定产业**：有铁矿才有冶铁锻造，有瓷土才有窑场，有蚕丝才有织造，有靛草才有染坊。
3. **产业不只非遗**：除了非遗手艺，地区还有**基础产业**（冶铁、采铜、采银、制盐、伐木、种桑、采茶、采矿、磨颜料），它们产出**原料/半成品**，是手艺的上游。
4. **供应链驱动探索**：高级手艺需要外地原料——想做景泰蓝得先有铜，想打好剑得先有好铁。玩家必须**解锁产地→在当地手搓原料→运回**，地图因此「活」起来。

## 17. 地区档案（地貌 · 物产 · 产业）

> 下表是世界的主干。每个地区标注：**地貌特征 / 基础产业（产原料）/ 招牌非遗工艺 / 特产资源**。
> ★为建议首发地区。说明：部分矿冶设定取自真实史地（如汉代临邛冶铁、明清佛山铸铁、湖北大冶铜铁、云南东川铜矿、和田玉等），
> 个别为可玩性做了适度演绎，实现时再做史实校核。

### ★17.1 江南（浙江 · 苏南）— 水乡商埠
- **地貌**：河网密布、沿海、丘陵少山；石桥、乌篷船、白墙黛瓦、桑基鱼塘。
- **基础产业**：种桑养蚕（生丝）、海盐晒制、龙泉采铁/淘瓷土、茶园采青。
- **招牌非遗**：龙泉青瓷、龙泉宝剑（铁）、缂丝、油纸伞、西湖龙井、绍兴黄酒。
- **特产资源**：`蚕丝`、`海盐`、`瓷土(龙泉)`、`铁(龙泉)`、`茶青`。
- **地区性格**：富庶、重商；商路发达，是资源集散与销货枢纽（卖价高、信息灵）。

### ★17.2 巴蜀（四川 · 重庆）— 山盆竹海
- **地貌**：四面环山的红盆地、竹海、梯田、栈道、江流湍急；多雾多雨。
- **基础产业**：**临邛冶铁（铁锭）**、井盐熬制、伐竹、丹砂开采、茶马。
- **招牌非遗**：蜀锦蜀绣、青神竹编、瓷胎竹编、成都漆艺、荣昌折扇/陶。
- **特产资源**：`铁(临邛)`、`井盐`、`竹`、`丹砂(朱砂颜料)`、`茶青`。
- **地区性格**：物产厚但多山难行（移动/运输成本高）；是**铁与丹砂的主产地**。

### 17.3 岭南（广东 · 广西）— 海贸冶铸
- **地貌**：珠江三角洲、海岸线长、骑楼商街、码头商船；湿热。
- **基础产业**：**佛山冶铁/铸铁（铁器、铁线）**、种桑、莨薯种植（香云纱染料）、外销贸易。
- **招牌非遗**：广绣/潮绣、香云纱、石湾陶、端砚、坭兴陶、壮锦、外销瓷。
- **特产资源**：`铁(佛山·精铸)`、`桑丝`、`莨薯`、`端石`、`陶土`。
- **地区性格**：海贸门户，**出口订单**与外来奇货多；冶铸技术强（铁器质优）。

### 17.4 黔滇（贵州 · 云南）— 高原银铜
- **地貌**：云贵高原、群山梯田、苗寨吊脚楼、侗寨鼓楼、喀斯特；民族聚居。
- **基础产业**：**采银（银矿）**、**东川采铜（铜料）**、种靛蓝草、制茶（普洱）、陶土。
- **招牌非遗**：苗族银饰、蜡染、扎染（白族）、建水紫陶、乌铜走银、侗锦。
- **特产资源**：`银矿`、`铜(东川)`、`靛蓝草`、`陶土`、`茶青(普洱)`。
- **地区性格**：**贵金属与铜的主产区**；多民族联动工艺丰富，但深山交通不便。

### 17.5 荆楚（湖北 · 湖南）— 江湖铜铁
- **地貌**：长江中游、云梦泽水乡、丘陵；江湖纵横。
- **基础产业**：**大冶采铜/采铁**、割生漆、种桑、黑茶。
- **招牌非遗**：楚式漆器、湘绣、土家织锦、安化黑茶。
- **特产资源**：`铜(大冶)`、`铁(大冶)`、`生漆`、`桑丝`、`茶青`。

### 17.6 瓷都·赣鄱（江西）— 窑火高岭
- **地貌**：丘陵盆地、鄱阳湖、樟林、河运便利。
- **基础产业**：**高岭土开采（瓷土）**、釉料配制、伐樟、苎麻（夏布）。
- **招牌非遗**：景德镇手工制瓷、樟木家具、夏布。
- **特产资源**：`高岭土(瓷土)`、`釉料`、`樟木`、`苎麻`。
- **地区性格**：**瓷土与制瓷中心**；窑业兴盛、技艺分工极细（拉坯/利坯/画青/烧窑各为专户）。

### 17.7 徽州（皖南）— 群山文房
- **地貌**：黄山余脉、深山幽谷、徽派民居、溪水。
- **基础产业**：采青檀皮（造纸）、烧松取烟（制墨）、采歙石、伐木、毛峰采茶。
- **招牌非遗**：宣纸、徽墨、歙砚、徽派三雕（砖木石）、徽州毛峰。
- **特产资源**：`青檀皮`、`松烟`、`歙石`、`木材`、`茶青`。
- **地区性格**：**文房之乡**（笔墨纸砚原料集中）；徽商外出，资本回流。

### 17.8 燕赵·京畿（京津 · 河北）— 宫造重器
- **地貌**：华北平原、太行东麓、都城宫苑、漕运。
- **基础产业**：遵化采铁、海盐、桑蚕、矿物颜料。
- **招牌非遗**：景泰蓝（需铜）、花丝镶嵌（需金银）、潍坊/杨柳青年画与风筝（邻接齐鲁）、衡水内画、雕漆。
- **特产资源**：`铁(遵化)`、`海盐`、`颜料`、（金银/铜**依赖外区**）。
- **地区性格**：**宫廷重工**，做奢侈精品；但金、银、铜需从黔滇/雪域/西域调运——典型的**供应链下游**。

### 17.9 三晋（山西）— 煤铁醋漆
- **地貌**：黄土高原、沟壑、古城墙、窑洞、票号街。
- **基础产业**：**采煤、采铁**、高粱（酿醋酿酒）、割漆。
- **招牌非遗**：平遥推光漆器、老陈醋、晋作家具；晋商票号（金融）。
- **特产资源**：`煤`、`铁`、`高粱`、`漆`。
- **地区性格**：**能源与金融**——煤铁供给冶炼，票号提供借贷/汇兑（经济系统接口）。

### 17.10 雪域·高原（西藏 · 青海）— 金银唐卡
- **地貌**：高寒雪山、经幡、藏式碉房、草甸；含氧低、移动耗体力。
- **基础产业**：**采金、采银**、研磨矿物颜料、采狼毒草（藏纸）。
- **招牌非遗**：唐卡、藏纸、藏银器、藏香、氆氇。
- **特产资源**：`金`、`银`、`矿物颜料`、`狼毒草`。
- **地区性格**：**金银与天然颜料源头**；路途艰险，原料珍稀高价。

### 17.11 西域（新疆）— 绿洲玉路
- **地貌**：沙漠戈壁、绿洲、葡萄架、巴扎集市；商路要冲。
- **基础产业**：**和田采玉**、桑蚕、牧羊（羊毛）、采铜。
- **招牌非遗**：和田玉雕、艾德莱斯绸、地毯、铜器。
- **特产资源**：`和田玉`、`桑丝`、`羊毛`、`铜`。
- **地区性格**：**美玉与丝路集市**；东西货物在此交汇（稀有材料黑市/换货）。

## 18. 资源与供应链系统（让地图「活」起来）

### 18.1 资源分层
```
原矿/天然物（产地限定，靠基础产业采集）
   铁矿 铜矿 银矿 金 高岭土 蚕丝 苎麻 木棉 竹 木材 生漆 靛草
   井盐/海盐 茶青 玉石 端/歙/寿山石 煤 矿物颜料 牛皮 桦皮 沉香 松烟 青檀皮
        │  经「基础产业」加工
        ▼
半成品/材料（可跨区运输与交易）
   铁锭 铜料 银料 瓷土 染液(建靛) 生丝/染线 漆料 纸 颜料 盐 茶
        │  经「非遗手艺」工序链
        ▼
成品（订单交付 / 展览 / 馈赠）
   青瓷 宝剑 苗银饰 景泰蓝 蜀锦 蓝染布 唐卡 紫砂壶 …
```

### 18.2 跨区供应链（玩法主线动力）
- 每件成品的 `processChain` 声明所需材料；材料若**本区不产**，会标记为「需外购/自采」。
- 玩家可：① 在产地**手搓**原料（基础产业小游戏，如冶铁炉 `timing_hold` 控火→出铁锭）；② 或在商埠/巴扎**买入**（更贵，受市场度影响）。
- 典型链路示例：
  - **景泰蓝（京畿）** ← `铜料`（黔滇东川 / 荆楚大冶采铜）+ `金银`（雪域/西域）+ `颜料`。
  - **龙泉宝剑（江南）** ← 优质 `铁锭`（巴蜀临邛 / 岭南佛山精铸更佳）。
  - **苗族银饰（黔滇）** ← `银料`（本区银矿，自给）→ 顺差可外销京畿花丝。
  - **景德镇青花（赣鄱）** ← `瓷土`（本区）+ `青料/钴蓝`（西域/外购）。
- **解锁逻辑**：未解锁产地 → 只能高价买入且有断供风险事件；解锁后可自采，**降本＋稳定供应**，形成「探索→解锁→降本→做更高级成品」的正循环。

### 18.3 与隐藏经济（四维）的耦合
- 自采原料：省钱但耗工时/体力（生活度↓风险）；过度采矿可触发「竭泽」事件（生活/精神↓）。
- 外购原料：快但贵（市场度高时更贵），依赖商路（岭南/西域/江南枢纽）。
- 供应链稳定本身成为一种「市场度」表现；断供/囤积是事件与策略空间。
- 结算仍**全部在 `engine/reducer.ts`**；新增资源键与产业 action（如 `GATHER_RESOURCE`、`SMELT`）同步 `engine/types.ts`。

### 19.2 地形机制（不只是贴图）
- **可通行/障碍层**：水、山崖、墙为障碍；桥、栈道、街巷为通路（用 Tilemap collision 层）。
- **地貌影响移动**：山地/雪域移动慢或耗体力，水路需渡船/桥，沙漠有视野/补给限制（轻度，避免劝退）。
- **资源点散布在地貌里**：矿洞在山中、盐井在盆地、桑园在水乡、玉矿在戈壁——采集要「走到对的地形」。
- **地标即功能点**：窑房=陶瓷工坊、铸铁炉=冶铁、染缸院=染坊、票号=借贷/汇兑、巴扎/当铺=交易。

### 19.3 地图分级（衔接 §14，细化）
- **作坊内景**（可选）：进入体验点后的小室内场景，做工序小游戏。
- **街/坊**：一条非遗街，密集功能点（现有层级，将套用地貌主题）。
- **城**：多街区 + 城门 + 集市；地区首府（如金陵、成都府、广府、大理）。
- **地区（路/州）**：一张区域地图，散布数座城、资源点、关隘；体现该地貌主题。
- **九州**：全国地图，地区为节点，商路/驿道相连，按解锁推进。

### 19.4 实现路径（美术与工程）
- 用 **Tiled 编辑器**导出 JSON tilemap，Phaser `this.make.tilemap` 加载；地貌主题=一套 tileset。
- `textures.ts` 的占位生成保留为「未配图时的回退」；逐地区替换为真实 tileset（延续 §4.5 美术替换点约定）。
- 地区数据（地貌主题、资源点、功能点、连接关系）放 `src/data/regions.ts`，**数据驱动**，新增地区不改引擎。

## 20. 地区优先的实现优先级（取代 §15 的横向铺量顺序）
