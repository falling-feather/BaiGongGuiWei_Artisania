# 《百工归位》当前进度与规划差距盘点

> 盘点日期：2026-06-07  
> 交接整理：2026-06-13  
> 目的：给后续推进一个清晰坐标，说明当前项目大概走到哪一步、距离文档中的大型端游化目标还差多少，以及玩家验证时应重点看什么。代码组正式开工前，应先读 [代码组交接与收口清单.md](代码组交接与收口清单.md)；图像组和综合协作再读 [图像组与代码组交接总览.md](图像组与代码组交接总览.md)。

## 1. 一句话结论

当前项目已经不是早期 demo：**核心引擎、地区框架、NPC 交互、订单、天气、时段、田圃、地区声望、商路稳定、护商危机、断供危机、夜市摊位、珍品阁来访、专属联作和结局报告都已进入可运行链路**。

但它也还没有达到“只差美术资源和点位调整”的阶段。按最终目标“类端游、地区内容大型完善、工艺模拟深入可玩”来衡量，当前主要仍缺：

- 更真实的工艺小游戏操作，而不是大多由按钮/选项结算。
- 每个地区足够密的 NPC 日程、人物事件链、后续订单和回访线。
- 手工布置的地图点位、通道、工坊、资源点和区域内动线。
- 大量平衡、节奏、引导、异常态和长局压力测试。
- 后端/云存档/异步社交等页游化能力。

粗略判断：**核心系统骨架约完成 60%–70%，全地区生活与美术规格约完成 35%–45%，全国大型内容目标约完成 25%–35%，最终商业级表现与内容密度约完成 15%–25%。**

给代码团队的交接结论：**美术资源套用与最终点位美化先降到最低优先级；非美术系统还不能宣称基本完成。** 接手后的主线应是补齐工艺模拟深度、全地区 NPC/活动/订单密度、人工地图 JSON 到运行时的正式接入、街景通道式移动守卫、长局经济平衡和数据完整性工具。若某一项暂时不能完成，必须继续在本文记录为明确缺口，而不是归入“等待美术”。

## 2. 当前项目所在阶段

按 `doc/项目规划.md` 的 M0–M4 路线看：

| 里程碑 | 文档目标 | 当前判断 |
|---|---|---|
| M0 基础内核 | 基础手艺、回合循环、工序链、事件、结局报告、可完整玩一局 | 已超过。当前已有多地区、多手艺、多系统链路和结局报告。 |
| M1 生态成形 | 3–4 门手艺、资源经济、技艺联动、事件扩充 | 基本超过。手艺与资源规模已大幅扩展，联作与供应链已有实际结算。 |
| M2 长线驱动 | 解锁树、手艺升级、代币经济、地域拓展 | 部分完成。地域拓展、路线、地区声望、订单、供应链和 11 区两层工坊升级/季维护/空间扩建已跑通；正式解锁树、更深工坊空间经营和长期经济仍弱。 |
| M3 联网与社交 | 账号、云存档、排行榜、报告分享、成就 | 成就和本地存档已有，联网/云端/排行榜基本未做。 |

按 `doc/开放世界生活模拟规划.md` 的目标看，当前最接近的是：**江南生活模拟样板 + 全国供应链骨架 + 若干外地代表 NPC/订单/联作样例**。还没有达到“玩家自然生活 1–2 小时仍持续有新内容”的密度。

## 3. 已实现的主链路

### 3.1 基础运行链路

- React + Phaser + Vite 前端可运行。
- Zustand 管理游戏状态。
- `engine/reducer.ts` 维持纯函数式规则结算。
- `localStorage` 多槽存档已支持版本迁移。
- 新游戏、时段推进、季节/日推进、事件、结局报告都已接入。
- 当前测试覆盖 158 项。

### 3.2 地区与移动

- 已定义 11 个大地区：江南、巴蜀、岭南、黔滇、荆楚、赣鄱、徽州、京畿、三晋、雪域高原、西域。
- 地区定义内已有 43 个小地区，`SUBREGION_CONTENT` 已覆盖 44 条小地区内容作用域。
- 大地图当前主要作为路线总览/调试入口保留。
- 实际玩法原则已经转向：**地区移动应通过街景中的区内通道、出入口和交互点触发**。
- 已新增迁移入口静态守卫：`WorldMapModal` / `RegionPanel` 不得直接派发 `TRAVEL`、`TRAVEL_SUBREGION`、`UNLOCK_REGION`，正式迁移继续限定在 `StreetScene -> EventBus -> App` 的 gate 链路。
- HUD 提示条新增“互动”按钮，通过 `emitCommand({ type:'interact-nearby' })` 回到 `StreetScene.interactNearby()`，与键盘 E 共用附近点位/NPC/通道/出入口判定，避免 UI 绕过街景空间规则。
- 商路稳定、路线情报、开路成本折扣、护商次数、断供记录都已落到状态。

仍缺：各地图人工点位和动线需要继续手工化；通道/入口还需要从程序生成推进到精确摆放。

### 3.3 手艺与资源

- 已有 50 门手艺数据。
- 约 81 类资源/产物数据。
- 蓝染、竹编有更细工序；其余招牌工艺多数采用统一三段工序模板。
- 2026-06-12 新增 `CraftInteractionSpec` 数据化样板：11 个大地区各有一门代表工艺写入真实阶段、品质维度、缺陷触发和返修方案；江南已继续扩展“龙泉青瓷”、临安“油纸伞”、太湖“缂丝”、巴蜀“蜀锦”、岭南“香云纱”、黔滇“苗银”、荆楚“楚漆”、赣鄱“景德镇瓷”、徽州“宣纸”、京畿“景泰蓝”和三晋“平遥推光漆器”，接入坯形/釉色、伞骨/伞面、断纬/绒面、断花/翘边、浮色/泥痕、焊痕/压身、起泡/纹样发闷、坯体走样/惊釉、纸筋弱/帘纹乱、脱丝/塌蓝、光面发虚/描金浮缺陷、返修方案和对应师傅授艺。
- 采集、精炼、工序运行、材料消耗、品质产物和作品实例已跑通。
- 上游材料品质可写入成品来源和描述。
- 作品实例现可携带 `qualityDimensions`、`defects` 与 `repairHistory`；低质材料、省略关键备料或低品质成品会留下可返修缺陷，背包可直接触发 `REPAIR_ITEM` 返修。
- 订单、NPC 鉴评、赠礼、题名、陈列和珍品阁来访已开始读取作品缺陷；`itemEffectiveQuality` 会让缺陷折损验收，`orderDeliveryIssue` 会拦下高信任/高品相订单中的重缺陷作品。
- NPC 授艺开始读取 `CraftInteractionSpec.mentorNpcIds`：师傅会把代表工艺阶段、常见缺陷和返修方向写入 `knownTopics`/flags；后续 `RUN_PROCESS` 会把这些师傅建议转成品质微增、重点维度加成、同类缺陷严重度压低、`师傅看样` 描述和 `craft-mentor-used:*` 标记，`REPAIR_ITEM` 会把已请教的返修法转成工时减免、师承返修履历和 `mentor-repair-*` flags，形成“请教 -> 试作 -> 诊断/返修”的前置学习链路；当前已有宁辞秋、叶青盏、林雨桥、沈云梭、卓锦娘、何云纱、银娘阿岚、熊朱漆、窑头老温、汪纸娘、蓝大器、平遥漆婆十二个样板入口；数据守卫已校验所有 `mentorNpcIds` 都指向现有 NPC。
- 2026-06-12 新增 `WORKSHOP_UPGRADES` 一阶工坊整备树：11 个大地区各有一条代表工艺升级，购买会走 `UPGRADE_WORKSHOP` 纯函数结算，校验手艺解锁、成品/资源成本、地区声望或属性门槛，写入 `GameState.workshopUpgrades`、`workshop-upgrade:*` / `workshop-upgrade-craft:*` flags，并在后续 `RUN_PROCESS` 中影响工时折扣、成品品相、作品描述词、品质维度和对应重缺陷严重度压低；`CraftPage` 已显示“工坊整备”区块，旧档会自动补齐 `workshopUpgrades`。
- 2026-06-13 浏览器烟测追加：实际走街景通道时发现跨子区后旧“按 E 前往……”提示会残留，已在 `App.tsx` 的 `interact-subregion-gate` / `interact-gate` 分支主动清空 hint；同时修正街面交互优先级和 vendor 站位，避免师傅 NPC 贴住工艺点导致“进入工坊”被“与 NPC 交谈”长期挤掉。
- 2026-06-13 继续扩展 `WORKSHOP_UPGRADES` 为 22 条两层升级树：11 区代表工艺各新增一条二阶建筑/工具/品牌/物流升级，`WorkshopUpgradeRequirement.upgrades` 会显式校验前置升级，`CraftPage` 会显示缺少哪项前置整备；二阶成本消耗成品样件与材料，效果可与一阶叠加到工时、品质、描述词、维度、地区指标和缺陷压低。
- 2026-06-13 继续接入工坊长期维护：`END_TURN` 会按升级层级结算季维护费（一阶默认少量钱，二阶默认钱+工时），维护成功写入 `maintenancePaid/lastMaintainedTurn`，付不起会写 `maintenanceMissed/lastMissedTurn`、`workshop-maintenance-missed:*` flags、日志，并对对应工艺 market/spirit 施加轻度压力；`CraftPage` 已显示每项升级的“季维护”成本。
- 2026-06-13 继续收紧小地区工坊点：`RUN_PROCESS`、`TAKE_ORDER`、`UPGRADE_WORKSHOP` 和 `EXPAND_WORKSHOP_SPACE` 现在会读取 `SubregionContentSpec.craftIds`，不在当前小地区开放的工艺不能隔空制作、交单、扩建或安置工坊；`CraftPage` 与回退用 `CraftExperienceModal` 会显示应返回的本地工坊点。
- 2026-06-13 继续加入工坊空间经营：新增 `workshopSpaces` 与 `EXPAND_WORKSHOP_SPACE`，每门工艺默认 1 格整备空间，二阶工坊会先被空间容量拦下；玩家必须在本地工坊点消耗文钱/工时扩建后，才能继续安置第二项整备。`CraftPage` 已展示空间已用/容量、扩建成本和“空间不足”原因，`RegionMapSpec.crafts[].workshop` 会把容量/扩建/可安置摘要下发给街景工坊点，`StreetScene` 接近提示会显示“空间 1/1 / 可扩建 / 可安置整备”等状态；存档版本升至 `SAVE_VERSION=16` 并兼容旧档补齐 `workshopSpaces: []`。
- 2026-06-13 继续补工艺操作层：新增 `CraftTechniqueSelection` / `CraftTechniqueRecord` 与 `craftTechniques.ts`，`CraftPage` 和 `CraftExperienceModal` 会按 `CraftInteractionStage` 展示“稳手细作 / 平衡推进 / 赶工压时”三种手法；`RUN_PROCESS.techniqueChoices` 会把手法转成工时增减、品相变化、焦点品质维度变化、工艺四维变化、作品鉴评与 `craft-technique:*` flags，省略工序后对应手法不再参与结算。

仍缺：旗舰工艺的独立动作小游戏深度不足。龙泉剑、青瓷、油纸伞、缂丝、蜀锦、香云纱、苗银、云锦、推光漆、唐卡、玉雕等已开始有数据化阶段/缺陷/返修、师傅授艺提示和两层工坊整备，但还需要更贴近工艺的实时操作、现场干预、失败演出、更深工坊扩建与长期经营压力。

### 3.4 NPC 与人物交互

- 手写核心 NPC 约 20 个，并引入占位 NPC 扩展。
- NPC 已支持功能标签、赠礼偏好、关系阶段台词、地方见闻、人物矛盾、结局影响。
- 已实装攀谈、赠礼、任务、授艺、指路、护商、切磋、订单、联作、鉴评、家园来访。
- NPC 见闻会写入 `knownTopics` 与 flags，能影响后续叙事和结局回声。
- 天气日程已支持，如雨天伞铺、晴天晒场。
- NPC 日程与自定义见闻已从样板扩到全国主线匠人/路线人物：15 个地区 NPC 已配 `NpcScheduleRule`（含晨/午/夜分时与雨/晴/雪天气分支），14 条手写 `NpcIntelDef` 覆盖竹丝顺纹、雪口规矩、苗银礼俗、楚漆潮养、宣纸水檀、推光掌温、唐卡度量、玉作顺绺、海贸船期、票号信用、青花分水、窑火窑期、宫造料路等地方门道，地图规格会按“当前时段 + 当前天气”把这些 NPC 安排到对应小地区。

仍缺：黔滇/荆楚/西域等地仍有部分 thin NPC（铜山客、冶叔、丝娘古丽等）只靠自动生成日程与见闻；需要更多人物冲突事件、阶段分支、失败回收和后日谈，把“代表性样例”补成“完整生活网络”。

### 3.5 活动、节令与夜市

- 活动挑战已有 24 个左右。
- 秦淮灯市已从单活动升级为三阶段夜市链：开灯夜、旺市夜、收灯夜。
- 西域绿洲巴扎已升级为第二个三阶段节令摊位链：开市晨、旺市晌、散市暮，复用 `ActivityStallDef` 全套顾客群（过往商队/识玉行客/绿洲东道）、组合（玉佩护件/云霞绸卷/驼队补给包）、主动策略与散市选择，证明节令摊位 schema 可跨地区泛化。
- 夜市/巴扎可售出库存/作品，按主动策略、顾客群、摊位组合、品相和赶集/小灯期结算。
- 灯谜已支持三轮选择。
- 收灯/散市选择已接入 `ActivityStallClosingModal`，会生成后续订单（来年灯谜底本预约、来季巴扎货约预约等）并进入结局亮点。

仍缺：节令深度仍主要集中在秦淮灯市与绿洲巴扎两处，巴蜀茶马会、赣鄱开窑、黔滇歌圩、京畿年节等其他地区节令尚未实装同等体量的事件链。

### 3.6 订单、商贸与供应链

- `ActiveOrder` 已接入具体资源、数量、最低品相、赏钱、地区声望、路线风险、交期、押金和信用分。
- NPC 可生成订单，玩家可在 NPC 面板交付。
- 订单逾期、押金没收、信用失约已经有 flags 和声望后果。
- 方季衡寄售单、雷掌柜票号信用单已具备样例。
- 高风险路线会触发断供危机，玩家可选择垫资补货、派人护路或承受短缺。
- 断供处置会进入账簿，可复盘稳路；拖延会形成余波。

仍缺：经济平衡还未长期校准；跨地区供需、价格、库存压力和玩家策略空间需要大规模长局验证。

### 3.7 护商危机

- 已有 `EscortEncounterDef` 数据驱动护商事件链。
- 茶马南线、雪山驿路、高原丝路驿线已有首段/后续事件；赣鄱柴运河路（赣徽山货商路/江南窑柴河路/江湖瓷柴路）已有“窑柴赶期簿”选择型危机与“窑期对账”后续，京畿漕运（运河北上路/北地票号路）已有“漕运料账簿”选择型危机。
- 茶马负重簿、驼道水账、窑柴赶期簿、漕运料账簿均已接入选择型危机；`EscortEncounterDef.requiredFlags/blockedFlags` 已让茶马、赣鄱柴运后续从“只按押运次数”升级为“按前置选择分叉”：重排茶篓盐包会进入“夜铃认路”，趁雾压坡会进入“雾口补账”，跑过窑柴会进入“窑期对账”。
- 待处理危机会阻断时辰推进/结季，选择后才结算工时、赏钱、路线稳定、NPC 见闻和成功/失败 flags。
- 断供危机为路线通用机制：`END_TURN` 会遍历所有已通商路，按风险与产地声望从 `RegionDef.localResources` 推断断供资源，京畿漕运（颜料矿物/铁矿）与赣鄱河路的宫造/窑火料路已自动纳入断供苗头与 `SupplyCrisisModal` 处置。

仍缺：护商还需要更多非赣鄱/京畿路线的多步危机、专属小游戏化操作，以及断供危机与具体工艺工期（如补蓝复烧、开窑）之间的更强联动。

### 3.8 代表作、专属联作与珍品阁

- `ItemInstance` 已支持题名、作者、协作 NPC、题跋、陈列、赠出等状态。
- 10 条专属联作配方已经接入：云锦/竹编/染织/漆艺/纸墨/推光漆/唐卡/玉作，以及赣鄱景德镇瓷“窑火分级联作”和京畿景泰蓝“掐丝点蓝联作”。
- 每条专属联作都有选择型分支。
- 分支可配置协同 NPC：汪纸娘“留题位”会把宁辞秋写为协作者，何云纱“赶市定色”会把伍海潮写为船期协同方，窑头老温“分水画青”会把画坯师蓝釉生写为协同方，蓝大器“花丝攒边/官样验收”会把金琐娘或宋押司写为宫造协同方。
- 高风险分支可配置失败返修，例如平遥漆婆“慢镜推光”和洛桑画师“度量补线”会按试手分数失败，扣材料、降品相、追加返修题评与失败记忆。
- 珍品阁来访已接入参观、题跋、收藏三类选择。
- 收藏分支可生成荐藏转介绍订单。

## 6. 目前最值得验证的玩家流程

这次你亲自跑时，建议重点验证这些链路：

1. 新开局后进入江南，观察 HUD 的日期、时段、天气、地区/小地区显示是否正常。
2. 在城郊百工院种植靛草/桑/茶，推进天气，看雨天免工时润田、雪天封土是否好理解。
3. 在江南小地区采集、精炼、跑一次蓝染或竹编工序，查看背包作品是否生成。
4. 找江南 NPC 攀谈、赠礼、接任务，观察见闻/好感/任务按钮是否清楚。
5. 体验秦淮灯市，尤其是灯谜三轮、夜市摆摊、收灯选择和后续订单。
6. 制作并命名一件高品质作品，陈列到百工院，触发珍品阁/题跋/收藏相关入口。
7. 尝试 NPC 专属联作，观察选择分支、材料/品相提示、协同 NPC 和失败返修提示。
8. 靠近街景手艺点、NPC 或通道时点击 HUD“互动”，确认它与键盘 E 都走同一条附近目标判定，并检查大地图只作为总览、实际移动尽量走场景通道/地区入口。
9. 尝试护商或等待断供危机，确认待处理危机会阻断推进，选择后才结算。
10. 推进到结局报告，查看地区段落、NPC 人情回声、夜市/珍品阁/断供/护商亮点是否出现。

- 蓝染、竹编、龙泉剑、云锦/缂丝、蜀锦、香云纱、苗银、楚漆、景德镇瓷、宣纸、景泰蓝、推光漆、唐卡、玉作等旗舰工艺都要先有文档级工序与美术需求。

### 7.4 内容工具链

- 做数据校验/可视化表，检查 NPC、活动、订单、联作、来访、点位和美术缺口是否缺字段。
- 后续可以把逐地区内容从 Markdown 草案迁移成结构化数据表，但迁移范围应面向 11 个地区整体。

## 8. 当前风险判断

- 最大风险不是架构，而是内容量和体验密度。
- 当前系统很多，但玩家第一次进入可能还不容易知道“应该追哪条线”。
- 最大风险是 11 个地区标准不一致，导致有的地区像完整土地、有的地区只像资源点；需要用同一张规格和验收表控制密度。
- “大地图只作总览/调试，迁移靠场景通道”的原则需要继续守住，否则会削弱探索感。
- 工艺小游戏如果不补深，项目会滑向材料合成游戏；这是最需要优先避免的方向。

## 9. 结论

当前可以定义为：

**大型生活模拟游戏的核心系统原型已成形，全国地区骨架已铺开；但距离最终文档中的端游级大体量，还需要一次性补齐 11 个地区的地图点位、人物长线、建筑与交互、美术缺口、小游戏手感、地区密度、经济平衡、内容工具链和联网能力。**

下一步不宜只继续加表项。最有效的路线是：**把江南、巴蜀、岭南、黔滇、荆楚、赣鄱、徽州、京畿、三晋、雪域和西域同时整理到完整设计规格，再按统一规格拆分实装任务。**
