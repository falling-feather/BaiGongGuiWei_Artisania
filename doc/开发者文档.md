> 本文目标：让任何一位新接手的程序员 / AI，**无需口头交接**即可独立续开发本项目。
> 读完本文你应当能回答：代码放在哪、出问题先看哪、各模块怎么连、加内容/加功能从哪下手。

## 0. 一分钟速览

- **是什么**：以中国非物质文化遗产「手艺工序」为核心的 **2D 像素风俯视街道游览** 游戏。玩家操控角色在街道（未来拓展到城市 / 国家地图）上自由行走，走到不同的手艺体验点亲手体验非遗工艺。手艺背后是一套隐藏的「传承 / 市场 / 生活 / 精神」四维数值与经济系统，支撑探索与经营。
- **技术栈**：React 18 + TypeScript 5 + Vite 5；**游戏世界用 Phaser 3**（像素风俯视、tilemap、物理碰撞）；状态管理用 Zustand；测试用 Vitest；界面样式为纯 CSS。
- **架构一句话**：**纯函数引擎（engine，隐藏数值/经济）+ 注入式内容（data）+ Zustand 桥接（store）+ Phaser 游戏世界（game）+ React HUD/弹窗（components）+ 可替换存档（storage）**。
- **核心原则**：游戏规则（数值结算）**只**存在于 `src/engine/reducer.ts`。Phaser 只管「空间与移动」，React 只管「HUD 与弹窗」，二者都不直接改规则。

## 3. 数据流：一次玩家操作如何贯穿全栈

```
玩家在街上走近「蓝染坊」，按 E
      │
      ▼
StreetScene.ts  →  emitBus({ type:'interact-craft', craftId })   // Phaser 只发事件，不碰规则
      │
      ▼
App.tsx  →  onBus 监听 → setActiveCraftId(craftId)            // React 接管，弹出体验面板
      │
      ▼
CraftExperienceModal 点「亲手制作」
      │
      ▼
store/gameStore.ts  →  gameReducer(state, action, content)     // 纯函数求新状态
      │                         └─ 读取 content（crafts/apprentices/events）
      ▼
set({ state: next })  +  localStorageAdapter.save(next)        // 更新 + 异步存档
      │
      ▼
HUD / 弹窗订阅的切片自动重渲染（四维、手艺状态…）
```

> **提示事件**：玩家走进/离开体验点感应范围时，StreetScene 会 `emitBus({type:'hint', text})`，
> App 监听后把提示文本传给 `Hud`（「按 E 进入…」）。Phaser 与 React 间的所有通信都走 `game/EventBus.ts`。

**关键不变量**：
1. 规则只在 `reducer.ts`。改数值平衡、加结算逻辑都来这里。
2. `gameReducer` 是纯函数，**无副作用**（不碰 localStorage、不 fetch）。副作用（存档）由 store 在 reducer 返回后执行。
3. 内容是**注入**的（`GameContent` 作为第三参数传入），引擎本身不 import 任何具体手艺/事件，因此引擎可独立测试、可移植到后端。
4. **Phaser 不碰数值**：游戏世界只负责空间/移动/触发，手艺的数值结算一律走引擎。

## 4. 核心类型速查（详见 `engine/types.ts`）

| 类型 | 作用 |
|---|---|
| `MetricKey` | 四维键：`heritage` 传承 / `market` 市场 / `life` 生活 / `spirit` 精神 |
| `Metrics` | `Record<MetricKey, number>`，取值 0–100 |
| `MetricZone` | 区间评价：`crisis/low/healthy/high/excess`，健康区 30–70 |
| `ProcessStep` | 一道工序：资源/人力成本、正常影响、是否可省略、省略影响 |
| `Craft` | 一门手艺：工序链 `processChain`、四维基线、联动标签、风险、`outputResourceId`（产出的成品资源键，供应链终端） |
| `Apprentice` | 学徒：性格特质、所学手艺、熟练度、留存意愿 |
| `GameEvent` / `EventChoice` / `GameEffect` | 随机事件、选项、结构化效果 |
| `CraftState` | 某手艺在本局的动态状态（四维、是否解锁、产出数） |
| `GameState` | **单一事实来源**：种子、回合、四维、资源、手艺、学徒、待处理事件、日志、状态、报告、`unlockedRegions`、`currentRegion`、`achievements`、`seenStory`（已看过的剧情节点）、`flags`（叙事标记，驱动分支）、`playerName`（玩家名号）、`devMode`（开发者模式） |
| `GameReport` | 结局命运报告：标题/概述/最终四维/存续手艺/亮点/`epilogue`（个性化尾声，呼应名号与拉择走向） |
| `ResourceDef` / `ResourceTier` | 资源定义与分层（raw 原料 → material 半成品 → product 成品）；product 可选 `value`（订单交付基准售价） |
| `IndustryDef` / `MiniGameType` | 基础产业定义与其采料小游戏类型（六种） |
| `RegionDef` / `TerrainTheme` | 地区档案与地貌主题 |
| `AchievementDef` | 成就定义：`id/name/desc` + `predicate(state)=>boolean`（纯函数） |
| `StoryBeat` / `StoryChoice` | 剧情节点：`id/title/lines/trigger(state)=>boolean` + 可选 `choices`（每个 `StoryChoice` 含 `setFlags?`/`logMessage?`） |
| `GameAction` | 所有可派发动作（见下表） |

### GameAction 一览

| action | 含义 | 由谁派发 |
|---|---|---|
| `NEW_GAME{seed?,playerName?}` | 开新局（名号决定剧情称呼，`fallingfeather` 触发开发者模式） | MainMenu「开启新局」/ 结局「再启新局」 |
| `RUN_PROCESS{craftId,skipStepIds}` | 按工序链开工，可省略部分工序 | CraftPanel |
| `TAKE_ORDER{craftId}` | **交付订单**：消耗 1 件成品库存换收入，售价随该手艺传承品质浮动（`orderPrice`）；无成品则拒绝 | CraftExperienceModal |
| `HOLD_EXHIBITION` | 举办市集/展演 | App「镇务」 |
| `RESOLVE_EVENT{choiceId}` | 处理当前随机事件 | EventModal |
| `END_TURN` | 结束本季，结算/补给/抽事件 | App「结束本季」 |
| `GATHER_RESOURCE{industryId,quality?}` | 在当地跑一项基础产业，quality 来自小游戏手感 | MiniGameModal |
| `TRAVEL{regionId}` | 前往一个已解锁地区 | WorldMapModal / RegionPanel / 场景出入口 |
| `UNLOCK_REGION{regionId}` | 花 30 文解锁相邻地区 | WorldMapModal / RegionPanel / 场景出入口 |
| `SEEN_STORY{storyId,choiceId?}` | 标记某剧情节点已看；带 `choiceId` 时把该抉择的 `setFlags` 写入 `flags` 并记日志 | StoryModal |

> **成就检测**：顶层 `gameReducer` 是 `checkAchievements(reduce(...))` 的包装——每次 action 结算后
> 遵一遍 `content.achievements` 的 `predicate(state)`，新达成的追加进 `state.achievements` 并写日志。仍为纯函数。

## 4.5 2D 游戏世界（Phaser 层）

> 这一层是本次重做的核心：把原文本玩法包进一个可自由行走的像素世界。
> 设计上**刻意全部用程序生成的占位图**，方便日后逐个替换为真实美术（见「美术替换点」）。

### 模块职责

| 文件 | 职责 |
|---|---|
| `game/PhaserGame.tsx` | React 容器组件。`useEffect` 里 `new Phaser.Game(config)` 挂到 div，卸载时 `game.destroy(true)`。开发态把实例挂到 `window.__game` 供浏览器调试。 |
| `game/gameConfig.ts` | Phaser 配置：`pixelArt:true`、800×500 基准分辨率、FIT 缩放、arcade 物理（无重力）、`scene:[StreetScene]`。 |
| `game/EventBus.ts` | 单例 `Phaser.Events.EventEmitter`，**双向**：`emitBus`/`onBus`（Phaser→React：`interact-craft`/`interact-industry`/`interact-gate`/`hint`/`scene-ready`/`player-pos`(玩家瓦片坐标+地图尺寸)/`region-points`(小地图点位)/`zoom-changed`(当前倍率)）与 `emitCommand`/`onCommand`（React→Phaser：`enter-region` 附 `RegionMapSpec`、`zoom` 附 `delta`/`absolute`）。 |
| `game/regionSpec.ts` | `buildRegionSpec(regionId, state)`：把数据层+当前状态翻译成场景可用的 `RegionMapSpec`（地面配色、产业点含层级、已实现手艺点、相邻地区出入口）。场景因此与数据层解耦。 |
| `game/textures.ts` | **★美术替换点★** `generatePlaceholderTextures(scene)` 用 `Graphics.generateTexture` 画占位贴图（幂等，切区不重生成）。地面用白底 + 运行时 `setTint` 按地区 palette 着色。`TILE=32`。 |
| `game/scenes/StreetScene.ts` | 唯一场景，**地区驱动**。`create()` 建玩家/相机/输入并订阅 `enter-region`/`zoom` 命令；`buildRegion(spec)` 清空旧地图后按 spec 铺地（palette 着色）、沿街摆放产业/手艺/出入口点（列数随点数动态扩展、地图宽度随之放大避免槽位重叠），末尾 `emitBus('region-points')` 下发小地图点位。`update()` 处理 8 向移动、按 Y 排深度、检测最近点、按 E 发交互事件、+/- 键缩放、变动时 `emitBus('player-pos')` 上报坐标；`applyZoom` 统一限幅(1~4×)并 `emitBus('zoom-changed')`。 |

### 完整 demo 数据流（地图切换 / 小游戏 / 资源传递）

```
场景就绪 → emitBus('scene-ready')
  → App 收到后 buildRegionSpec(currentRegion) → emitCommand('enter-region', spec) → 场景铺图

走近产业点按 E → emitBus('interact-industry', industryId)
  → App 开 MiniGameModal → 玩家「出手」得 quality(0..1)
  → dispatch GATHER_RESOURCE{industryId, quality} → 引擎结算：扣原料/工时，产出 = round(yield*(1+quality))

走近出入口按 E → emitBus('interact-gate', {regionId, unlocked})
  → 已解锁 → dispatch TRAVEL；未解锁 → dispatch UNLOCK_REGION(扣30文)，成功则接着 TRAVEL
  → currentRegion / unlockedRegions 变化 → App 的 store 订阅检测到 signature 变化 → 重新 emitCommand('enter-region') → 场景切图

资源传递：resources 是全局单一池，跨区携带即「运输」。
  例：江南缺煤 → 解锁邻区赣鄱(有 coal) → 采煤 → 回江南冶铁(铁矿本地+煤) → 锻剑成品。

相机缩放 / 小地图（V0.5.12）：
  +/- 键·小键盘加减·Shift+滚轮·小地图按钮 → 场景 `applyZoom` 限幅 1~4×。
  玩家移动→`player-pos`→App 状态→`Minimap` 实时渲染玩家点与坐标；切区重建→`region-points`→小地图重画点位。
```

> 产业分三层：采集(harvest，仅耗工时，由地区 `localResources` 授权) → 精炼(refine，原料→材料) → 制作(product，材料→成品)。
> 采集业的可用性在引擎 `gatherResource` 里特判：`input` 为空且 `output ∈ region.localResources` 即放行，无需写进 `region.industries`。

### 关键约定 / 踩坑记录

- **不要用 `React.StrictMode`**：它会双挂载，导致 Phaser 生成两个 canvas。`main.tsx` 里已移除（注释说明了原因）。
- **单场景胜于 Boot+切换**：早期 `BootScene → StreetScene` 的 `scene.start` 静默不触发，已合并为单一 `StreetScene` 在 `create()` 里自己生成贴图。新增场景时把它加进 `gameConfig` 的 `scene` 数组。
- **`window.__game` 调试钩子**：仅开发态（`import.meta.env.DEV`）挂载，可在浏览器控制台直接读写玩家位置/速度，测试移动与交互。
- 交互点感应范围 `INTERACT_RANGE = TILE*1.8`；进入范围发 `hint`，按 E 按点类型发 `interact-craft`/`interact-industry`/`interact-gate`。

### 如何新增一个交互点（产业 / 手艺 / 出入口）

地图内容**完全由数据驱动**，新增点不改场景：

1. **产业**：在 `data/industries.ts` 加一条 `IndustryDef`；把 id 写进某地区 `region.industries`（采集业则只需让其 `output` 出现在该地区 `localResources`）。
2. **手艺**：在 `data/crafts.ts` 加一门手艺，并把其 id 放进某地区 `signatureCrafts`（只有已实现的 craft 才会出现在地图上）。
3. **出入口**：在 `data/regions.ts` 给地区互填 `neighbors` 即可；已解锁→可前往，未解锁→花 30 文解锁。
4. 配色：地区 `terrain.palette` 前两色作为 [地面色, 街道色]，场景运行时 `setTint`。

### 美术替换点（日后逐个替换）

当前所有视觉都是占位色块。替换真实像素美术时按以下顺序，**只动 `game/textures.ts` 与场景里加载逻辑，玩法零改动**：

| 占位 | 替换为 | 做法 |
|---|---|---|
| `TEX.player` 单色方块 | 玩家 4 向行走精灵表 | 把 `generateTexture` 改为 `scene.load.spritesheet('player', url, {frameWidth,frameHeight})`，并在 `StreetScene` 建立 `anims`（walk-up/down/left/right），`update()` 里按移动方向播放。 |
| `TEX.ground` / `TEX.road` | 真实地砖/路面 tile | 换成 tileset 贴图；进一步可改用 Phaser Tilemap（Tiled 导出 JSON）替代手铺。 |
| `TEX.shopIndigo` / `TEX.shopBamboo` | 作坊建筑像素图 | 每个体验点一张建筑图，`texture` 键对应换图即可。 |
| `TEX.marker` | 「按 E」气泡/箭头 | 换成动画精灵。 |

真实素材建议放 `public/assets/`（或 `src/game/assets/` 经打包），在 `StreetScene.preload()` 里 `load`。替换是**增量**的：可以先只换玩家精灵，其余仍用占位。

### 未来地图分层（来自产品愿景）

街道 → 城市 → 国家三级地图。实现思路：每级一个 Phaser 场景（或同场景切换 Tilemap），出口瓦片触发 `scene.start` / 切图；体验点数据从 `data` 注入，保持引擎与 UI 不变。

## 4.6 元系统：大地图 / 背包 / 成就 / 主菜单 / 新手引导

## 4.7 玩家名号 / 开发者模式 / 剧情分支系统

> 这一层给 demo 添上叙事与代入感：玩家取名→剧情称呼→抉择分叉。全部以**数据驱动 + 引擎纯函数 selector** 实现，引擎逻辑零硬编码。

### 玩家名号与开发者模式（V0.5.3）

- 开局页输入名号→`NEW_GAME{playerName}`→`createInitialState(...,playerName)` 存入 `state.playerName`。
- `state.ts` 常量 `DEV_NAME = 'fallingfeather'`；`devMode = playerName.trim().toLowerCase() === DEV_NAME`。
- **开发者模式**：`unlockedRegions` = 全部地区、`resources` = `DEV_RESOURCES`（各 999999）；END_TURN 时 `labor` 不衰减（保持海量工时）。纯函数判定，不开后门。

### 剧情节点与抉择分支（V0.5.4 / V0.5.5）

| 模块 | 职责 |
|---|---|
| `data/story.ts` | **剧情表** `STORY_BEATS`。每个 `StoryBeat` 用 `trigger(state)` 纯函数决定何时出现（按回合/成就/已看节点/`flags`/解锁地区数/开发者模式）；有 `choices` 的为抉择节点。`renderStoryLine(line,name)` 把 `{name}` 替换为玩家名（空名回退「无名匠人」）。 |
| `engine/reducer.ts` 的 `nextStoryBeat(state,content)` | 选取首个 `!seenStory.includes(id) && trigger(state)` 的节点；无则返回 null。纯函数 selector，不改状态。 |
| `engine/reducer.ts` 的 `SEEN_STORY` | 追加 storyId 进 `seenStory`；带 `choiceId` 时从 `content.story` 找到节点与抉择，把 `choice.setFlags` 并入 `flags`、推 `choice.logMessage`（支持 `{name}`）。 |
| `components/StoryModal.tsx` | 读 `nextStoryBeat`；`pendingEvent` 存在或无节点时返回 null。有 `choices` 则逐项渲染按钮派 `SEEN_STORY{storyId,choiceId}`；否则单一「继续」派 `SEEN_STORY{storyId}`。 |

**分支机制（零引擎逻辑改动）**：`GameEffect.setFlags` + `StoryChoice.setFlags` 写入叙事标记 → 后续 `StoryBeat.trigger` 读 `flags` 分叉。示例：「立心」节点选「守正」→`oath-tradition`，选「趋时」→`oath-market`；第 4 季分到 `path-tradition` / `path-market`。随机事件「审美风向突变」选项写 `chased-trend`/`kept-tradition`，趋流触发「风过之后」回响节点。

**App 挂载**：`{!tutorialOpen && <StoryModal />}`——引导期不出剧情；剧情与随机事件互斥（`pendingEvent` 优先）。

### 个性化结局尾声（V0.5.7）

- `generateReport` 调 `generateEpilogue(state, m)` 写入 `GameReport.epilogue`。
- 根据 `flags` 的 `oath-tradition`/`oath-market` 走向，叠加最终传承/市场度，给出「体面回答 / 退了几步」或「热闹是真热闹 / 没等到回暖」的呼应或反差；再叠 `kept-tradition`/`chased-trend` 的风向回响句，玩家名号嵌入正文。
- `GameOverReport` 渲染 `.report__epilogue`（朱砂左边线、分段缩进）。纯函数，不读外部数据。

### 手艺工序链 / 供应链终端（V0.5.8）

> 打通「采料(raw) → 精炼(material) → 手艺(product)」三层：手艺 RUN_PROCESS 现在**消耗半成品、产出成品**。

| 环节 | 说明 |
|---|---|
| `Craft.outputResourceId` | 手艺的成品资源键（product 层）。蓝染→`indigoCloth`、竹编→`bambooWare`。缺省则不产实物。 |
| 工序 `resourceCost` | 改用 material 键（`indigoVat`/`bambooSplit`），与精炼产业的 `output` 对接。 |
| `reducer.ts` 的 `runProcess` | 汇总工序耗料→校验人力/物料→扣除→产出 1 件 `outputResourceId` 入 `resources` 池，日志取成品名（由 `content.resources` 注入）。 |

#### 订单经济闭环（V0.6.0）

> 闭环：采料(raw) → 精炼(material) → 手艺 RUN_PROCESS 产出成品 → **TAKE_ORDER 交付成品换取市场收入**。售价与品质挂钩，鼓励维持传承而非偼工减料。

| 关键 | 说明 |
|---|---|
| `ResourceDef.value` | 成品基准售价（文）。青瓷 32 / 宝剑 36 / 银饰 28 / 景泰蓝 40 / 蓝染布 20 / 竹编器 18 / 织锦 34 / 唐卡 38。 |
| `orderPrice(baseValue, heritage)` | 纯函数定价：传承 0→0.6×、50→1.0×、100→1.4×，取整且不低于 1。engine 与弹窗共用同一公式。 |
| `reducer.ts` 的 `TAKE_ORDER` | 校验 `outputResourceId` 库存 ≥1；不足则仅记日志返回。足额则扣 1 件成品 + 进账 `orderPrice`，该手艺 market+6/heritage+2/spirit+1。 |
| `CraftExperienceModal` | 产出行显示「库存 N」与「订单价 X 文」；接订单按钮无成品时禁用并提示。 |
| `GameContent.resources` | 新增可选资源词典，仅用于日志/展示取名；由 `gameStore` 注入 `RESOURCES`。 |
| `state.ts` 初始/开发者资源 + 每回合补给 | 同步改为 material 键（`indigoVat`/`bambooSplit`）。 |
| `CraftExperienceModal` | 新增耗料/产出预览与不足预警；料/工不足时禁用「亲手制作」。 |

### 全局设置 / 地图缩放 / 小地图（V0.5.12）

> 给街道探索补上「可返回主页 + 可缩放 + 可定位」三件表层体验，全走 EventBus，不耕入引擎。

| 模块 | 职责 |
|---|---|
| `components/SettingsModal.tsx` | 左上齐轮按钮 / `Esc` 唤出（App 全局 keydown 监听，仅 `status==='playing'` 时生效）。提供「返回主页」：`setView('menu')`，存档已自动留存，主菜单「继续上局」可续。 |
| `components/Minimap.tsx` | 右下角缩略图：按 `player-pos` 的 `mapW/mapH` 按比例摆点，产业橙/手艺蓝/出入口绿 + 玩家红点，顶部显示瓦片坐标与 −/＋ 缩放按钮（`emitCommand('zoom',{delta})`）。 |
| `StreetScene.applyZoom` | 唯一缩放入口：按 `ZOOM_STEP=0.25` 对齐、`Clamp(1,4)`，变化才 `setZoom` + `emitBus('zoom-changed')`。输入源：`+/-` 主键盘与小键盘、Shift+滚轮、`zoom` 命令。 |
| `App` 状态 | `settingsOpen`/`playerPos`/`mapPoints`/`zoom` 由 `onBus` 驱动；zoom 跨 `view` 保留（返回主页再续局倍率不丢）。 |

| 现象 | 先看 |
|---|---|
| 数值结算不对 / 平衡问题 | `engine/reducer.ts`（runProcess、endTurn、applyEffect） |
| 四维着色/健康区判定错 | `engine/metrics.ts` 的 `zoneOf`、`HEALTHY_LOW/HIGH` |

| 画面出现两个 canvas / 重影 | `main.tsx` 是否误加了 `React.StrictMode`（必须不加） |
| 角色不动 / 按 E 无反应 | `StreetScene.update()` 的按键读取；浏览器控制台用 `window.__game` 调试 |
| Phaser↔React 不通信 | `game/EventBus.ts` 的事件类型 + `App.tsx` 的 `onBus` 订阅 |
| 存档丢失/读档失败 | `storage/localStorageAdapter.ts`（注意 `SAVE_VERSION` 变更会丢旧档） |
| 大地图节点位置/状态不对 | `data/mapLayout.ts` 坐标 + `WorldMapModal.tsx` 的 `kindOf/isReachable` |

| 菜单/引导不出或重复弹 | `App.tsx` 的 `view` 与 `artisania:tutorial-seen` 标记 |
| 剧情不出/重复/分支不对 | `data/story.ts` 的 `trigger`/`flags` + `reducer.ts` 的 `nextStoryBeat`/`SEEN_STORY` + `StoryModal.tsx`（是否被 `pendingEvent` 拦下） |
| 开发者模式不生效 | `state.ts` 的 `DEV_NAME`/`devMode` 判定与 `DEV_RESOURCES`；名号需完全匹配 `fallingfeather`（不区大小写） |
| 界面不刷新 | 检查组件是否用 `useGameStore(s => s.切片)` 订阅了正确切片 |
| 按钮该禁用没禁用 | 各组件里的 `playing` / `hasEvent` 判定 |
| TS 报错 unused | tsconfig 开了 `noUnusedLocals/Parameters`，删掉未用 import |

## 6. 如何扩展内容（最常见的开发任务）

### 加一门新手艺
1. 在 `src/data/crafts.ts` 的 `CRAFTS` 数组里新增一个 `Craft` 对象：填 `id`、`name`、`region`、`blurb`、`processChain`（每道工序的成本与四维影响）、`baseMetrics`、`synergyTags`、`risks`。
2. 若希望开局就有它的动态状态，确认 `createInitialState` 会为每个 craft 生成 `CraftState`（默认已遍历传入的 crafts）。
3. 完成。引擎与 UI 自动识别，无需改其它文件。

### 加一个随机事件
1. 在 `src/data/events.ts` 的 `EVENTS` 数组新增 `GameEvent`：`id/title/description/choices[]/weight`。
2. 每个 `choice.effect` 用 `GameEffect` 描述对四维/资源/某手艺的影响和日志文本。
3. `weight` 越大越容易被抽中。

### 调整平衡数值
- 初始资源、每回合补给、人力上限：`engine/state.ts`（`INITIAL_RESOURCES`、`LABOR_PER_TURN`）与 `engine/reducer.ts`（`TURN_RESOURCE_REGEN`、收益公式 `incomeBase`、`REGION_UNLOCK_COST`）。

### 加一条成就
1. 在 `src/data/achievements.ts` 的 `ACHIEVEMENTS` 加一条 `AchievementDef`：`id/name/desc` + `predicate(state)`（纯函数，只读 `GameState`）。
2. 完成。reducer 每次结算后自动检测解锁，成就册与 toast 自动识别。

### 加一个地区
1. 在 `src/data/regions.ts` 加一条 `RegionDef`（产业/特产/邻区）。
2. 在 `src/data/mapLayout.ts` 给它补一条 `{x,y}` 坐标，即可在大地图上出现。
3. 互填 `neighbors` 打通商路；引擎/场景不改。

### 加一个剧情节点 / 分支
1. 在 `src/data/story.ts` 的 `STORY_BEATS` 加一条 `StoryBeat`：`id/title/lines` + `trigger(state)`（纯函数，只读 `GameState`，可读 `flags`/`seenStory`/`turn` 等）。
2. 要分支则加 `choices`，每个 `StoryChoice` 用 `setFlags` 写叙事标记；后续节点的 `trigger` 读该标记分叉。
3. 若要随机事件也能打标记，在 `data/events.ts` 的 `choice` 上加 `setFlags`。引擎/UI 不改。

## 7. 存档与未来后端

- 当前实现：`LocalStorageAdapter` 把整份 `GameState` 序列化进 `localStorage`（键 `artisania:save`），带 `version` 字段（当前 `SAVE_VERSION=4`；因 `GameState` 新增 `playerName/devMode/seenStory/flags` 而从 3 提升）。
- `hasSave()` 探测是否有版本兼容的可续存档（供主菜单「继续上局」启用与否）。
- 版本不兼容（`SAVE_VERSION` 提升）时，目前直接丢弃旧档；如需迁移，在 `load()` 里按 version 写迁移逻辑。
- **接后端时**：实现一个 `RemoteStorageAdapter implements StorageAdapter`（save/load/clear 改为 HTTP 调用），在 `gameStore.ts` 替换 `localStorageAdapter` 即可。账号/排行榜/云存档都从这一层接入，UI 与引擎零改动。

## 11. 给下一位接手者的提醒

- 改规则只动 `reducer.ts`，别把逻辑写进组件或 Phaser 场景。
- 保持引擎纯净：不要在引擎里 import 具体数据或调用浏览器 API。
- **Phaser 只管空间/移动，所有手艺数值结算走引擎**；Phaser↔React 只通过 `game/EventBus.ts`。
- **绝不加 `React.StrictMode`**（会让 Phaser 生成双 canvas）。
- 美术是增量替换的：从 `game/textures.ts` 的占位图逐个换成真实像素（见第 4.5 节「美术替换点」）。
- 中文文件用 UTF-8（无 BOM）；PowerShell 下避免用 `Set-Content`/`Get-Content` 处理含中文的文件，优先用编辑器工具。
- 提交信息格式：`V{大}.{中}.{小} 中文说明`。
