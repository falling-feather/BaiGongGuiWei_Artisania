# 程序收束巡检与浏览器新档深烟测记录

更新时间：2026-06-14

## 本轮目标

文档设计进入审核后，程序侧开始快速收束缺口。本轮不扩大全量玩法，优先补齐“当前优先版是否真的可跑、可见、可验收”的程序入口。

## 已实装

1. 新增 `src/engine/priorityScopeAudit.ts`
   - 读取 `GameState`、主轴里程碑和当前优先版范围要求。
   - 输出五区主轴进度、里程碑完成数、主轴五区资料齐备数、骨架六区资料齐备数。
   - 发现缺失时返回稳定缺口 ID，例如 `craft:jade-carving`，方便程序和策划对账。

2. 新增 `src/engine/__tests__/priorityScopeAudit.test.ts`
   - 守卫当前优先版巡检初始状态：主轴 0/5、里程碑 0/10、主轴资料 5/5、骨架资料 6/6。
   - 守卫江南完成后下一主轴会切到巴蜀。
   - 守卫内容库缺项会以稳定 ID 报出。

3. 接入 `src/components/RegionPanel.tsx`
   - 镇务面板新增“当前优先版收束巡检”。
   - 镇务面板新增“本地工坊”，只开放当前小地区已有工艺入口，不触发地区迁移。
   - 库存中的 `coin`、`labor` 展示为“通货”“工时”。

4. 补充 `src/index.css`
   - 新增巡检卡片、进度条、主轴/骨架资料 chips 样式。

5. 补充 `src/components/CraftPage.tsx` 与 `src/components/CraftExperienceModal.tsx`
   - 工坊整备、维护成本等展示层同样将 `coin`、`labor` 本地化。

## 浏览器深烟测

测试入口使用 `http://localhost:5173/`，避免覆盖用户在 `127.0.0.1:5173` 下已有存档。

已跑通：

1. 新建/继续测试档 `浏览烟测匠`。
2. 从江南苏杭水市通过街景区内通道进入龙泉山坊。
3. 在龙泉山坊采集煤、采集铁矿、冶铁，背包显示 `铁锭 ×1`、`煤 ×2`、`铁矿 ×1`。
4. 连续推进时辰到第 2 日上午，工时恢复为 8，并处理日常事件分支。
5. 打开镇务，确认巡检显示：
   - 主轴 `0/5` 区，里程碑 `0/10`。
   - 主轴资料 `5/5`，骨架资料 `6/6`。
   - 本地工坊列出龙泉青瓷与龙泉宝剑。
6. 从本地工坊进入龙泉宝剑整页工坊，确认耗料显示 `铁锭 1/1`、`煤 2/1`、`工时 8/5`。
7. 点击“亲手制作”，产出宝剑，库存变为 `宝剑 1`，主轴目标切到秦淮灯市。
8. 点击“接订单”，宝剑库存归 0，通货从 28 增至 72，并解锁售卖相关成就。
9. 全程浏览器控制台无 error/warn。

## 发现与处理

- 浏览器自动化对 canvas 点击命中不稳定，但此前已用真实街景通道完成苏杭水市到龙泉山坊迁移，本轮未新增任何地图直跳能力。
- 镇务新增本地工坊入口后，制作闭环不再依赖调试面板或总览地图；入口仍受当前小地区限制。
- 新档完整制作龙泉宝剑需要跨日恢复工时，这是当前体力节奏的预期结果。

## 当前结论

江南新档的“街景迁移、采集、精炼、制作、订单售卖、主轴目标推进、程序巡检可视化”已经在浏览器中跑通。下一轮应优先复核主轴五区活动入口与跨地区 gate/商路入口，而不是继续扩写策划文本。
