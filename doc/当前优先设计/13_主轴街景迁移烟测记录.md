# 主轴街景迁移烟测记录

更新时间：2026-06-14

## 本轮目标

把“地区移动不依赖世界地图直跳”的规则补成自动化烟测：当前优先版主轴提示可以指出下一目标，但玩家抵达目标必须通过街景里的区内通道或跨区 gate 完成。

## 已实现

新增 `src/game/__tests__/priorityJourneyNavigation.test.ts`：

1. 覆盖当前优先版五个锚点地区的主轴迁移：
   - 江南：苏杭水埠 -> 龙泉剑坊
   - 江南：龙泉剑坊 -> 金陵秦淮
   - 江南闭环后：金陵秦淮 -> 巴蜀锦里
   - 巴蜀闭环后：茶马驿站 -> 岭南香云纱晒场
   - 岭南闭环后：海港骑楼 -> 赣鄱窑镇
   - 赣鄱闭环后：窑镇 -> 西域玉作坊
2. 测试只消费 `buildRegionSpec(state).navigationTarget` 暴露出的街景目标。
3. 区内迁移必须能在 `spec.subregionGates` 找到对应入口，并经由 `TRAVEL_SUBREGION` 完成。
4. 跨区迁移必须能在 `spec.gates` 找到对应入口；若目标区尚未解锁，先经由 `UNLOCK_REGION` 打开，再经由 `TRAVEL` 完成。
5. 加入循环保护，防止主轴 fallback 在地区/小地区之间来回指向同一条无效路线。

## 设计结论

- 世界地图继续保留为调试和信息面板入口，不参与正式主轴迁移动作。
- HUD 主轴提示、百科行脚追踪和小地图高亮只负责“指路”，不直接改变 `GameState.currentRegion/currentSubregion`。
- 当前优先版程序收束时，后续新增主线节点必须同时满足：
  - 数据目标存在；
  - 街景 `gate` 或 `subregionGate` 存在；
  - reducer 能通过正式动作抵达；
  - 自动化测试能覆盖至少一条从上一主轴阶段到本阶段的路径。
