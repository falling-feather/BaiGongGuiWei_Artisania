# N2 骨架六区不断链记录

更新时间：2026-06-14

## 节点口径

本轮进入 `21_程序极速协作指引.md` 的 N2 节点：骨架六区不断链。目标不是把黔滇、荆楚、徽州、京畿、三晋、雪域做成主轴五区同等密度，而是证明每区至少具备可进入小地区、代表工艺、NPC、活动或订单入口、资料卡和结局段落，并能通过程序守卫与浏览器短烟测复现。

## 团队分配

按 N2 要求启用 2 个骨架团队 + 1 个 QA 团队：

| 团队 | 负责范围 | 审计结论 |
| --- | --- | --- |
| 骨架团队 A / Huygens | 黔滇、荆楚、徽州 | 三地最低闭环成立；建议补六区专项 smoke 和 layout 入口守卫。 |
| 骨架团队 B / Hypatia | 京畿、三晋、雪域 | 三地最低闭环成立；建议补 N2 浏览器 smoke，并修正一处三晋测试夹具旧 ID。 |
| QA 团队 / Archimedes | 六区横向 QA | 数据、资料卡、结局、ID 守卫基础足够；缺六区浏览器可点证据和集中 reducer 矩阵。 |

## 程序补线

1. 扩展 `src/dev/prioritySmokeScenarios.ts`。
   - 新增 `node: 'N1' | 'N2'` 口径，保留原 N1 主轴 smoke。
   - 新增 N2 骨架 smoke：`?smoke=qiandian`、`?smoke=jingchu`、`?smoke=huizhou`、`?smoke=jingji`、`?smoke=sanjin`、`?smoke=xueyu`。
   - 每个 N2 场景固定当前地区、小地区、本地活动、本地代表工艺、本地 NPC、资料卡和可选任务/路线活动。

2. 强化 `src/dev/__tests__/prioritySmokeScenarios.test.ts`。
   - N2 六区均可从 smoke state 启动。
   - 每区可通过 reducer 完成本地代表工艺制作、本地活动、NPC 好感/话题反馈。
   - 黔滇、徽州、京畿等可完成本地任务；荆楚、三晋等可完成路线/商贸任务。
   - 六区的路线活动会写入 `route-known:*` flags，证明骨架区不是孤立地图点。

3. 强化 `src/data/__tests__/priorityMapLayoutEntrypoints.test.ts`。
   - 骨架入口小地区若活动就在本地，则必须有活动点。
   - 若范围定义要求的活动在同区其他小地区，则入口小地区必须有 `subregionGate` 指向该活动小地区。
   - 继续守住“从街景 gate 或交互点迁移”，不新增总览地图直跳。

4. 清理测试夹具旧 ID。
   - `src/engine/__tests__/reducer.test.ts` 中三晋联作失败夹具从 `san-jin` / `sanjin-pingyao-lacquer` 改为冻结 ID `sanjin` / `sanjin-lacquer-yard`。
