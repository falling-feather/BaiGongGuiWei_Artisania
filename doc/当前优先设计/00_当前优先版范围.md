# 当前优先版范围

## 1. 裁剪判断

完整版目标是 11 个地区同时达到端游级生活密度。按照当前代码和文档状态，这个目标仍然正确，但如果立即要求全地区等量完成，会把剩余非美术工作拉到当前已完成量的 2 倍以上。

当前优先版改为“主轴纵切 + 全地区骨架”：

- 主轴纵切负责证明完整游戏体验，做到可玩、可测、可交接。
- 全地区骨架负责保护世界规模，做到可进入、可引用、不断链。
- 美术资源暂不作为代码验收项，但所有资源 key、地区包、NPC 名称和交互点类型要稳定。

按这个口径估计，当前优先版非美术内容已经完成约 65% 到 75%；剩余工作量约等于当前已完成工作量的 0.5 到 0.9 倍。若继续追加全地区同等密度，会回到完整版工作量。

## 2. 必做主轴

当前优先版只把以下五个地区作为内容密度主战场：

| 地区 | 当前职责 | 必须跑通的体验 |
| --- | --- | --- |
| 江南 | 新手家园、基础工艺、秦淮灯市、街景通道样板 | 新档引导、采集、制作、售卖、NPC、区内通道、结局回声 |
| 巴蜀 | 山地材料、茶马商路、蜀锦/竹编 | 商路风险、路线订单、节令摊位、导师反馈 |
| 岭南 | 海贸、香云纱、骑楼夜市 | 船期订单、夜市摆摊、染织缺陷、路线后续订单 |
| 赣鄱 | 景德镇瓷、开窑会、跨区供需 | 精炼、专注校准、复烧/返修、窑期事件 |
| 西域 | 绿洲巴扎、玉作、远行风险 | 跨区移动、绿洲活动、玉作缺陷、藏客回访 |

这五区必须形成“进入地区 -> 找到交互点 -> 完成工艺或活动 -> 产生作品/订单/关系变化 -> 推进长期反馈”的闭环。

## 3. 保留骨架区

以下六区本轮不删除、不弱化世界定位，但只做到骨架验收：

| 地区 | 本轮最低完成线 |
| --- | --- |
| 黔滇 | 有代表 NPC、苗银/蜡染入口、资料卡、地区声望和结局段落 |
| 荆楚 | 有楚漆/湘绣入口、路线活动占位、资料卡和基本订单 |
| 徽州 | 保留文房长链、宣纸/徽墨入口和藏客回访，不追加大规模节令 |
| 京畿 | 保留宫造采办、景泰蓝/花丝入口和采办藏客链 |
| 三晋 | 保留推光漆、票号回访和工坊整备压力 |
| 雪域 | 保留唐卡、净室供展、藏客回访和文化谨慎文本 |

骨架区必须能被移动系统、百工志、结局报告、订单或 NPC 见闻引用，不能成为地图上的死点。

## 4. 暂缓内容

以下内容进入完整版档案，不进入当前优先版主线：

1. 每门旗舰工艺的独立实时动作小游戏。
2. 43 个小地区全部手工点位、瓦片层、季节变体和最终碰撞层。
3. 所有 NPC 的长篇人物线、完整日程和多阶段后日谈。
4. 全地区等量节令活动、全地区等量夜市和全地区等量藏客链。
5. 联网、云存档、异步社交、多人经济和实时活动。
6. 经济数值的最终长期平衡，只做通胀、库存和失败惩罚的基础压力测试。
7. 最终美术替换、动画精修、地图美化和精确装饰摆放。

## 5. 当前优先版完成定义

当前优先版完成时，应满足：

1. `npm run typecheck`、`npm test`、`npm run build` 全部通过。
2. 浏览器烟测能从新档和继续档跑完主轴流程，控制台无 error/warning。
3. 总览地图不能直接迁移，移动必须通过街景通道或交互点完成。
4. 主轴五区每区至少有一条地区活动链、一条代表工艺链、一条 NPC/订单反馈链和一条结局回声。
5. 骨架六区全部可进入、可检索、可在资料卡中显示入口摘要，并至少有代表工艺和结局段落。
6. 新增或调整的内容 ID 都有数据守卫，避免活动、NPC、工艺、订单、通道或美术 key 悬空。
