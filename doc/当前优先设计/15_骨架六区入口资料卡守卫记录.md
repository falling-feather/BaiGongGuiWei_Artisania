# 骨架六区入口资料卡守卫记录

更新时间：2026-06-14

## 本轮目标

把当前优先版六个骨架区从“路线可达”推进到“入口不是空壳”：玩家进入这些地区后，至少能在代表子区看到本地产业、代表工艺、活动/NPC落点，并能通过百工志资料卡检索到该入口摘要。

## 已实现

扩展 `src/data/__tests__/priorityScope.test.ts`，新增骨架区入口内容守卫。

覆盖六个骨架地区：

| 地区 | 代表入口子区 | 守卫重点 |
| --- | --- | --- |
| 黔滇 | `qiandian-miao-village` | 苗银/蜡染入口、银娘阿岚、本地活动摘要 |
| 荆楚 | `jingchu-chu-lacquer` | 楚漆入口、熊朱漆、本地活动摘要 |
| 徽州 | `huizhou-paper-valley` | 宣纸入口、汪纸娘、本地活动摘要 |
| 京畿 | `jingji-palace-yard` | 景泰蓝入口、蓝大器、本地活动摘要 |
| 三晋 | `sanjin-lacquer-yard` | 平遥推光漆入口、漆婆、本地活动摘要 |
| 雪域 | `xueyu-thangka-court` | 唐卡入口、洛桑画师、本地活动摘要 |

每个入口都会校验：

1. `SUBREGION_CONTENT_INDEX` 中存在该子区内容索引。
2. 当前优先版要求的代表工艺确实落在该入口子区。
3. 入口子区至少有本地产业、本地工艺、本地活动。
4. 当前优先版要求的代表 NPC 驻留或排班到该入口子区。
5. `subregion-<subregionId>` 百工志资料卡存在。
6. 资料卡绑定正确地区与子区，并随地区解锁。
7. 资料卡引用代表 NPC，且 tags/body 具备基本检索与摘要密度。

## 设计结论

- 六个骨架区不在本轮追求端游级密度，但不能只留下地图节点；入口必须能承载玩家的第一眼信息、检索关键词和后续美术落点。
- 当前测试不强制六区拥有完整主轴订单链，避免扩大当前优先版工作量；它锁住的是“能进入、能理解这里做什么、能找到代表人物/工艺”的最低程序口径。
- 后续如果替换骨架区代表子区、代表 NPC 或代表工艺，必须同步调整 `PRIORITY_SCOPE_REQUIREMENTS`，并让该守卫测试继续通过。
