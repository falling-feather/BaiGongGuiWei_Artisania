# N1 浏览器深烟测收口记录

更新时间：2026-06-14

## 本轮目标

将 N1 最大缺口从“规则层已通、浏览器不可复现”推进为“可用开发态内存场景稳定复测”。本轮不改正式迁移规则，不写入玩家存档槽，只补程序调试入口、UI 可达性与自动化护栏。

## 程序补线

1. 新增 `src/dev/prioritySmokeScenarios.ts`。
   - 支持 `?smoke=jiangnan`、`?smoke=bashu`、`?smoke=lingnan`、`?smoke=ganpo`、`?smoke=xiyu`、`?smoke=xiyu-caravan`。
   - 入口仅在 `import.meta.env.DEV` 下生效。
   - 状态为内存态：`smokeMode=true` 时 `dispatch` 不调用 `localStorageAdapter.save`，不会覆盖真实存档槽。
2. 新增 `src/dev/__tests__/prioritySmokeScenarios.test.ts`。
   - 覆盖 6 个 smoke 场景可启动。
   - 覆盖五区节庆摊位三阶段、收束选择、后续订单生成。
   - 覆盖西域驼队驿站 `xiyu-caravan-post` 与 `caravan-route-known`。
3. 补充 UI smoke 选择器。
   - 活动挑战选项：`data-smoke="activity-choice:<activityId>:<roundId>:<choiceId>"`。
   - 摊位策略：`data-smoke="activity-strategy:<activityId>:<strategyId>"`。
   - 完成按钮：`data-smoke="activity-perform:<activityId>"`。
   - 收束选择：`data-smoke="stall-closing-choice:<activityId>:<choiceId>"`。
   - 镇务本地活动入口：`data-smoke="region-activity:<activityId>"`。
4. 修复活动弹窗可达性。
   - `.modal--activity` 增加 `max-height: 84vh` 与 `overflow-y: auto`。
   - 解决三轮挑战和摊位策略较多时，底部“完成体验”在浏览器视口不可点击的问题。
