# 完整版到当前优先版裁剪矩阵

## 1. 裁剪原则

当前优先版不是把《百工归位》改成小 demo，而是把完整版拆成“可交付的第一层世界”。完整版仍作为长期目标保留，当前优先版只决定本轮需要做到什么程度。

裁剪采用四种状态：

| 状态 | 含义 |
| --- | --- |
| 完整保留 | 本轮必须按完整版精神跑通，不降级为占位 |
| 收敛完成 | 保留系统，但只做主轴地区或代表样板 |
| 骨架保留 | 保留数据、入口和世界存在感，暂不追求深度 |
| 后置冻结 | 不删除设计，但本轮不实现、不阻塞交付 |

## 2. 系统裁剪矩阵

| 系统 | 完整版设计 | 当前优先版处理 | 状态 | 审核备注 |
| --- | --- | --- | --- | --- |
| 世界规模 | 11 个大地区、43 个小地区均有生活密度 | 11 区保留，五区做主轴深度，六区做骨架 | 收敛完成 | 不牺牲世界感，但不追求全地区等量 |
| 地区移动 | 全国地图、街景、城市/小地区多层移动 | 正式移动只走街景通道、牌坊、驿站、码头、商路入口 | 完整保留 | 总览地图只用于调试和路线提示 |
| 工艺系统 | 多门工艺都有真实工序和专属操作 | 保留工序、缺陷、返修、导师、作品履历；专属动作先做规则层样板 | 收敛完成 | 不能退回材料合成 |
| NPC 系统 | 全地区人物网、日程、关系、长线后日谈 | 主轴五区做人物反馈，骨架区保留代表 NPC 与见闻 | 收敛完成 | 长篇人物线后置 |
| 节令活动 | 全地区多节令、多阶段活动 | 秦淮灯市、茶马会、骑楼夜市、开窑会、绿洲巴扎优先 | 收敛完成 | 证明 schema 可复用即可 |
| 订单经济 | 长期市场、商路、价格、库存、路线风险 | 跑通订单、售卖、押金、商路风险和基础压力 | 收敛完成 | 最终经济平衡后置 |
| 商路与危机 | 多路线、多步护商、断供、运输风险 | 主轴路线做代表事件，骨架区只保留可引用风险 | 收敛完成 | 不追求全路线同等事件数 |
| 家园与种田 | 类星露谷式农事、建设、自动化 | 保留田圃和基础资源循环，不扩大家园装修与自动化 | 骨架保留 | 美术和长线经营后置 |
| 珍品阁/藏客 | 全地区代表作、荐藏、回访、续订 | 主轴与已有长链保留，新增只补断点 | 收敛完成 | 先保证作品命运闭环 |
| 百工志/文本库 | 所有资料、人物、地区、工艺结构化入库 | 保留资料卡生成和检索，优先补主轴五区关键文本 | 收敛完成 | 文本应能引导通道，不直接迁移 |
| 地图编辑器管线 | 全小地区精确点位、瓦片、图层、季节变体 | 保留代表布局和对象 ID，完整瓦片/季节变体后置 | 骨架保留 | 给美术先锁命名 |
| 美术资源 | 全地区整图、瓦片、建筑、人物、动画、UI | 只锁资源 key、地区包和首批资产清单 | 骨架保留 | 当前阶段不做最终替换 |
| 联网社交 | 云存档、异步社交、多人经济、实时活动 | 完全后置 | 后置冻结 | 不进入当前优先版 |

## 3. 内容裁剪矩阵

| 内容层 | 完整版数量口径 | 当前优先版口径 |
| --- | --- | --- |
| 大地区 | 11 个全密度 | 5 个主轴深度 + 6 个骨架 |
| 小地区 | 43 个都要有手工动线 | 主轴相关小地区优先精修，其余保留入口摘要 |
| 旗舰工艺 | 多门都做专属操作 | 每个主轴地区至少一条工艺链完整，实时动作后置 |
| NPC | 全地区人物网 | 主轴 NPC 有关系反馈，骨架 NPC 有功能和见闻 |
| 节令活动 | 全地区多节令 | 五条活动链优先验收 |
| 结局反馈 | 全事件组合变化 | 地区声望、作品、NPC、商路、活动的基础组合必须存在 |

## 4. 明确不剪的东西

以下内容不允许为了缩小范围而取消：

1. 11 个地区的世界存在感。
2. 通过街景通道移动的正式游玩规则。
3. 工艺的材料、工序、缺陷、返修、导师、作品履历链。
4. NPC 与地区见闻对文本库和结局的影响。
5. 美术资源 ID 的稳定性。

## 5. 当前优先版交付边界

当前优先版交付的是“可审核、可美术对接、可进入程序收口”的设计冻结稿。审核通过前，不要求编译、浏览器烟测或深入代码改造；审核通过后，再按 `03_优先内容验收表.md` 进入程序验收。
