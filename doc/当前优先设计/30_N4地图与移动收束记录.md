# N4 地图与移动收束记录

日期：2026-06-14

## 节点目标

N4 负责把当前优先版的移动红线和街景入口收紧到程序可守卫状态：

1. 复验主轴五区和骨架六区街景入口。
2. 守住总览地图不得正式迁移。
3. 补人工 JSON、通道、活动点、NPC 点、资源点的引用守卫。
4. 回头审查 UI、资料卡或面板是否绕过街景通道。

本节点继续只处理程序端；最终美术替换、精确装饰摆位和非优先小地区细化不作为阻塞。

## 协作团队

按 `21_程序极速协作指引.md` 的 N4 要求启用 3 个团队：

1. 地图团队：审计 `regions`、`mapLayout`、`mapLayouts/*.json`、`subregionContent`、`regionContent`、`priorityScope` 与街景 spec 装配。
2. UI 红线团队：审计 WorldMap、RegionPanel、Lore、HUD、App 与 StreetScene，确认正式迁移只来自街景 gate。
3. QA 团队：整理 N4 自动化与内置浏览器 smoke 矩阵，指出缺少 DOM 钩子和可达性守卫。

## 剩余边界

1. 全量 43 个小地区中，本节点将当前优先人工街景推进到 22 张。其余 21 个小地区仍可由 StreetScene 自动布局进入，不是硬断链；后续属于地图点位细化和表现层微调。
2. `resource` 没有单独 interaction 类型，当前用 `industry` 点承载采集/精炼资源入口；这与现有工艺/资源系统一致。
3. RegionPanel 仍允许打开本小地区工坊/活动/采集说明，它不改变地区或小地区；若后续红线扩大为“所有体验点也必须近身触发”，需要另开节点收紧。
