# 当前优先版结局回声守卫记录

更新时间：2026-06-14

## 本轮目标

把“主轴五区和骨架六区都有结局回声”从静态字段检查推进到真实运行链路：当玩家到达阶段性结算时，当前优先版登记的地区声望与代表 NPC 好感必须能进入结局报告。

## 已实现

新增 `src/engine/__tests__/priorityEndingReport.test.ts`。

测试覆盖：

1. 当前优先版 11 个地区全部经由 `END_TURN` 触发结局报告。
2. 每个地区在高声望状态下生成 `regionalOutcomes`。
3. 地区回声必须带地区名，并读取该地区 `ending.pillar` 文本。
4. 当前优先版登记的 required NPC 全部具备 `endingInfluence`。
5. 每个 required NPC 在高好感状态下生成 `relationshipOutcomes`。
6. 人情回声必须带 NPC 名字，并读取该 NPC 的长期结局影响文本。

## 设计结论

- 结局报告现在不只是“有弹窗、有字段”，而是能从 reducer 的正式 `END_TURN` 动作读取当前优先版地区与 NPC 数据。
- 骨架六区继续保持低密度，但结局段落和代表人物后日谈必须能被阶段性结算读取，不能成为只存在于地图和百科里的静态占位。
- 后续如果替换当前优先版 required NPC，必须同步补 `endingInfluence`，否则结局守卫会直接失败。
