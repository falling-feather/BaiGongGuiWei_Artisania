# ID冻结与资源命名规范

## 1. 文档目的

当前优先版要尽快对接美术组，因此必须先冻结一批跨策划、程序、美术共用的名称。冻结不是说不能新增，而是不能随意改名；新增必须遵守同一命名规则。

本文件先作为策划审核稿。审核通过后，程序和美术按此整理 manifest、地图对象和资源包。

## 2. ID 分层

| 层级 | 示例 | 归属 | 是否冻结 |
| --- | --- | --- | --- |
| 大地区 ID | `jiangnan`、`bashu`、`lingnan`、`ganpo`、`xiyu` | `regions.ts` | 已冻结 |
| 小地区 ID | `jiangnan-longquan`、`ganpo-kiln-town` | `regions.ts` / `subregionContent.ts` | 主轴五区优先冻结 |
| 工艺 ID | `longquan-sword`、`gambiered-silk`、`jingdezhen-porcelain` | `crafts.ts` / `craftInteractions.ts` | 当前优先版冻结 |
| NPC ID | `jn-ning-ciqiu`、`ln-he-yunsha` | `npcs.ts` | 主轴 NPC 冻结 |
| 活动 ID | `ln-qilou-night-market` 等 | `regionContent.ts` | 主轴活动冻结 |
| 挑战 ID | `ln-qilou-night-market-route` 等 | `activityChallenges.ts` | 与活动绑定冻结 |
| 百工志 ID | `region-ganpo-kiln-town` 等 | `loreEntries.ts` | 可新增，不改旧 ID |
| 地图对象 key | `gate`、`npc`、`workshop`、`resource` | `mapLayouts/*.json` | 类型冻结，坐标可调 |
| 资源 key | `regions/jiangnan/...` | 美术 manifest | 审核后冻结 |

## 3. 命名规则

### 3.1 通用规则

1. 程序 ID 使用英文小写和连字符。
2. 中文名只作为显示文本，不作为引用键。
3. 已进入存档、flags、订单、作品履历或美术 manifest 的 ID 不改名。
4. 若必须改名，需要同时写迁移说明、旧 ID 映射和测试守卫。
5. 美术文件可以有中文备注，但 manifest key 必须英文稳定。

## 5. 资源 key 设计

资源 key 不直接等于文件名，而是稳定 manifest 名称。建议格式：

| 类型 | key 格式 | 示例 |
| --- | --- | --- |
| 地区母版 | `region.{regionId}.master` | `region.jiangnan.master` |
| 街景地面 | `region.{regionId}.streetTiles` | `region.lingnan.streetTiles` |
| 建筑图集 | `region.{regionId}.buildings` | `region.ganpo.buildings` |
| 道具图集 | `region.{regionId}.props` | `region.xiyu.props` |
| NPC 头像 | `npc.{npcId}.portrait` | `npc.jn-ning-ciqiu.portrait` |
| NPC 站姿 | `npc.{npcId}.stand` | `npc.ln-he-yunsha.stand` |
| 工艺工具 | `craft.{craftId}.tools` | `craft.jingdezhen-porcelain.tools` |
| 工艺缺陷 | `craft.{craftId}.defects` | `craft.gambiered-silk.defects` |
| 活动摊位 | `activity.{activityId}.stall` | `activity.ln-qilou-night-market.stall` |

## 6. 地图对象命名

地图布局对象建议按交互类型命名：

| 类型 | 用途 | 必填目标 |
| --- | --- | --- |
| `resource` | 采集点、材料点 | `targetId` 指向资源或产业 |
| `workshop` | 工坊入口 | `targetId` 指向工艺或产业 |
| `activity` | 地区活动或节令入口 | `targetId` 指向活动 ID |
| `npc` | NPC 站位 | `npcId` 指向 NPC ID |
| `subregion-gate` | 区内通道 | `targetId` 指向小地区 ID |
| `region-gate` | 跨地区通道 | `targetId` 指向大地区或路线 |
| `vendor` | 商贩/订单板 | `targetId` 指向商贩或订单类型 |
| `decor` | 装饰 | 不参与规则结算 |

坐标和美术表现可后续调整，但对象类型和目标 ID 必须稳定。

## 8. 改名流程

审核通过后，如确需改 ID：

1. 在 `06_审核问题与决策记录.md` 记录原因。
2. 列出旧 ID、新 ID、影响文件和迁移策略。
3. 程序阶段补旧档迁移或兼容别名。
4. 更新美术 manifest 和地图布局。
5. 跑 ID 引用守卫。

没有迁移策略的改名一律后置。

## 9. 审核口径

本文件审核通过即代表：

1. 主轴五区的地区、工艺、活动和 NPC 命名可以交给美术组。
2. 美术组第一轮可按 `region.{regionId}.*`、`npc.{npcId}.*`、`craft.{craftId}.*` 组织资源。
3. 程序组后续只做 manifest 接入和点位微调，不因命名重新设计系统。
