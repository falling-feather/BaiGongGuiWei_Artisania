# 美术组对接清单

## 1. 当前口径

当前优先版不要求最终美术资源完成，但代码侧要把资产 ID 和运行入口稳定下来，让美术组可以立刻按包生产。美术组第一轮只需要服务“主轴五区 + 全地区识别骨架”。

## 2. 地区包优先级

### 第一优先

| 地区 | 画面识别点 | 首批资产目标 |
| --- | --- | --- |
| 江南 | 水市、秦淮灯火、龙泉山坊、织埠 | 新手街景、基础工坊、灯市摊位、通道牌坊 |
| 巴蜀 | 竹海、山道、茶马驿、织锦坊 | 山路入口、竹编/蜀锦工坊、茶马摊位、驿站 |
| 岭南 | 骑楼、晒莨、港口、夜市 | 骑楼街、香云纱晒场、船期告示、夜市摊位 |
| 赣鄱 | 窑火、高岭土、瓷镇、河运 | 采土点、淘练点、瓷窑、开窑会场 |
| 西域 | 绿洲、巴扎、玉作、商队 | 巴扎街、玉柜、驼队入口、远行路标 |

### 第二优先

黔滇、荆楚、徽州、京畿、三晋、雪域先交“地区识别图 + 代表工坊 + 代表 NPC + 通道/入口”即可，暂不要求完整街区装饰。

## 3. 稳定资产类型

每个地区包建议先按以下文件组织：

| 文件 | 用途 | 当前要求 |
| --- | --- | --- |
| `region_master.png` | 地区整图母版 | 用于确认风格和构图，不急于切片 |
| `street_tiles.png` | 街景地面、水路、山路、墙体 | 先覆盖主轴五区基础路径 |
| `buildings.png` | 工坊、市场、码头、驿站、牌坊 | 必须能区分交互类型 |
| `props_sheet.png` | 材料点、半成品、摊位、告示、货箱 | 优先服务采集、制作、订单、夜市 |
| `characters.png` | 站姿或简易行走角色 | 先给核心 NPC 与通用路人 |
| `portraits.png` | NPC 头像或半身 | 先给主轴 NPC 和导师 |
| `craft_tools.png` | 工艺工具、材料、失败状态 | 支持工艺 UI 与后续小游戏替换 |

## 4. 需要代码侧锁定的 key

交接前代码侧应锁定：

1. 地区 ID、子地区 ID、工艺 ID、NPC ID。
2. 街景交互点类型：采料、工坊、活动、NPC、子区通道、跨区通道、商贩、装饰。
3. 主轴五区的代表工艺阶段名、缺陷名、返修方向和工具名。
4. 每个地区包的目录名和首批资源文件名。
5. 地图编辑器或运行时布局中使用的对象类型和目标 ID。

锁定后原则上只新增，不改名；如必须改名，要同步更新数据守卫和本清单。

## 5. 交接验收

美术组拿到当前优先版后，应能回答：

1. 哪五个地区要优先画到可游玩。
2. 每个优先地区的第一眼识别点是什么。
3. 哪些 NPC、工坊、摊位和通道必须先有资源。
4. 哪些资源只是占位，可等第二轮。
5. 代码侧如何从资产 key 找到运行入口。

代码组拿到首批美术后，应只需要做资源替换和点位微调，不需要再重写地区设计。
