# 活动收束入口互锁记录

更新时间：2026-06-14

## 目标

活动摊位进入最终收束选择后，玩家必须先处理当前收束，再开启新的地区活动。reducer 已经具备这条规则，本轮把它同步到 UI 入口层，避免玩家在弹窗或镇务入口看到“似乎还能继续开活动”的误导。

## 已完成

1. `src/components/ActivityModal.tsx`
   - 新增 `pendingStallClosing` 判断。
   - 未处理摊位收束时禁用“完成体验”。
   - 显示“先处理当前摊位收束，再安排新的地区活动。”反馈。
2. `src/components/RegionPanel.tsx`
   - 本地活动入口在 `pendingActivityStallClosing` 存在时同步禁用。
3. `src/game/__tests__/activityUiEntrypoints.test.ts`
   - 守卫活动弹窗和镇务入口都读取摊位收束未决状态。

## 验证

已通过：

- `npm test -- activityUiEntrypoints reducer`
- `npm run typecheck`

## 结论

活动执行链现在在规则层和入口层保持一致：摊位最终收束没有处理时，不会从活动弹窗或镇务面板继续开启新的地区活动。后续五区活动浏览器深烟测可以直接围绕“活动三轮 -> 收束选择 -> 后续订单”推进。
