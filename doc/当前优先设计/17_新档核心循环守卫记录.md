# 新档核心循环守卫记录

更新时间：2026-06-14

## 本轮目标

把“新档进入江南后能完成采集、精炼、制作、售卖闭环”落成当前优先版专门测试，避免它只散落在 reducer 大测试里。

## 已实现

新增 `src/engine/__tests__/priorityNewGameCoreLoop.test.ts`。

测试从 fresh state 出发，真实执行：

1. 初始进入江南苏杭水埠，并确认主轴目标指向龙泉宝剑。
2. 通过 `TRAVEL_SUBREGION` 进入 `jiangnan-longquan`。
3. 执行 `GATHER_RESOURCE harvest-iron-ore` 采铁矿。
4. 执行 `GATHER_RESOURCE harvest-coal` 挖煤。
5. 执行 `GATHER_RESOURCE smelt-iron` 冶铁。
6. 执行 `END_TURN` 恢复人力。
7. 执行完整 `RUN_PROCESS longquan-sword`。
8. 校验主轴目标推进到秦淮灯市活动。
9. 执行 `SELL_ITEM` 售出龙泉宝剑。
10. 校验售出履历、银钱、库存扣减和 `item-sold` flags。

## 设计结论

- 新档第一条生产链受现有体力系统约束：采集、精炼和完整三段制作不能在同一季全做完，因此测试按“采集精炼一季，次季开工售卖”执行。
- 该守卫验证的是规则层核心循环；浏览器内点击点位、碰撞距离和 HUD 遮挡仍由后续浏览器深烟测确认。
- 如果后续调整龙泉剑材料、人力成本或新手地区入口，该测试会优先暴露首小时体验是否被打断。
