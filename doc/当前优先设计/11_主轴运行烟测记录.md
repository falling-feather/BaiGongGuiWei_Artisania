# 主轴运行烟测记录

更新时间：2026-06-14

## 本轮新增验证

新增 `src/engine/__tests__/priorityJourneyFlow.test.ts`，用于证明当前优先版五区主轴不只是 HUD 文案，而是能通过真实 reducer 动作推进：

1. 进入主轴当前地区与代表工艺小地区。
2. 补足测试资源后执行真实 `RUN_PROCESS`。
3. 检查主轴目标从“工艺样品”推进到对应节会/摊位里程碑。
4. 进入节会小地区，连续执行三轮真实 `PERFORM_ACTIVITY`。
5. 检查 `stall-chain-completed:{activityId}` 和收摊待处理事件。
6. 执行真实 `RESOLVE_ACTIVITY_STALL_CLOSING`。
7. 检查主轴目标进入下一地区；西域完成后进入 `complete`。

## 当前结论

- 主轴目标计算可以随真实制作与活动结算推进。
- 五个锚点地区的代表工艺样品均能通过 reducer 产出。
- 五条节会/摊位链均能完成三段推进并触发收摊选择。
- 收摊选择能够回写完成标记，主轴最终进入 complete。
- 该测试不替代浏览器内街景入口、碰撞和实际点位验收；它证明规则层链路完整。

## 后续仍需浏览器复核

1. 五区街景内活动入口与 NPC/通道点击优先级。
2. 地区迁移是否全部通过 gate、subregionGate 或商路交互点触发。
3. HUD 主轴目标在 1280x720、移动窄屏和存档继续状态下是否都不遮挡关键交互。
4. 美术资源替换后，地图对象和 manifest key 是否仍保持一致。
