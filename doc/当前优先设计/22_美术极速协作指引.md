# 美术极速协作指引

本文只服务当前优先版：主轴五区先画到可游玩识别，骨架六区先画到不断链识别。不按完整版美术一次性扩张。

## 1. 极简进度

- 已有第一轮可用素材基础：33 个建筑运行 PNG、32 个道具运行 PNG、11 个 NPC 行走表、11 张地区全景概念图。
- 已有通用运行资源：HUD、主菜单封面、天气叠层、通用瓦片、开放世界素材、西域素材。
- 当前美术不是从零开始，但正式资源包还未成体系。
- 当前优先版美术完成度约 45% 到 55%。
- 若按最终统一风格交付，完成度约 30% 到 40%。
- 完整版美术完成度约 20% 到 30%。

## 2. 当前缺口

P0 缺口：

- 主轴五区缺正式地区包：`region_master.png`、`street_tiles.png`、`buildings.png`、`props_sheet.png`、`characters.png`、`portraits.png`、`craft_tools.png`。
- 骨架六区缺识别包：`region_master.png`、`buildings.png`、`portraits.png`。
- 11 区专属地形、道路、水岸 tile 尚未进入新美术组生成清单。
- 主轴 NPC 缺头像、半身或站姿统一规格。
- 主轴工艺缺工具、材料、缺陷、返修状态图。
- 主轴活动缺摊位、账册、货样、收束选择等活动视觉。

P1 缺口：

- 大地图底图和地区节点图标仍是 CSS 节点。
- 背包资源图标、成就徽章、订单图标尚未成体系。
- 手艺整页仍多为主视觉占位块，缺主视觉、工序小图、师傅图和成品图。
- 现有生成素材需要统一风格、尺寸、透明边、占格、碰撞和编辑器 QA。

后置缺口：

- 43 个小地区全部精修不进当前优先版。
- 全地区完整昼夜、天气、季节变体不进当前优先版。
- 所有 NPC 长篇立绘和活动插画不进当前优先版。

## 3. 最小团队分配基数

最小有效并行基数：8 个美术团队。

| 团队 | 任务 | 主要输出 |
| --- | --- | --- |
| A0 美术总监 | 风格统一、资料校准、最终审查 | 风格板、审查意见 |
| A1 江南包 | 水市、龙泉、灯市、新手街景 | 主轴地区包 |
| A2 巴蜀包 | 竹海、山路、茶马、织锦 | 主轴地区包 |
| A3 岭南包 | 骑楼、晒莨、港口、夜市 | 主轴地区包 |
| A4 赣鄱包 | 窑火、高岭土、瓷镇、河运 | 主轴地区包 |
| A5 西域包 | 绿洲、巴扎、玉作、驼队 | 主轴地区包 |
| A6 骨架六区 | 黔滇、荆楚、徽州、京畿、三晋、雪域识别包 | 骨架地区包 |
| A7 切图接入 QA | 抠绿、切片、命名、编辑器摆放、预览 | 运行 PNG + QA 图 |

极速扩容时再加 4 个团队：

- NPC 团队：专门做主轴 NPC 头像、半身、站姿、通用路人。
- 工艺团队：专门做工具、材料、缺陷、返修、工序示意。
- UI 图标团队：专门做背包、订单、成就、大地图、资源图标。
- 活动插画团队：专门做五区活动摊位、货样、账册、收尾图。

## 4. 项目节点

### A0 风格与命名锁定

团队数：1 到 2 个。

任务：

- 统一当前优先版目录、文件名、地区 ID、NPC ID、工艺 ID。
- 确认所有产物按代码 manifest 命名，原则上只新增不改名。
- 从已有 11 张全景概念图里定主色、建筑剪影、地貌关键词。

回头审查：必须。未统一命名不得并行切图。

### A1 主轴五区正式包

团队数：5 个地区团队 + 1 个美术总监。

任务：

- 五区各交一个可绘制、可拆分、可进编辑器的地区包。
- 每区优先画：整图母版、街景路径、代表工坊、活动摊位、核心 NPC、通道入口。
- 不追求全子地区精修，只服务当前优先版主轴链路。

回头审查：必须。五区完成后统一检查风格是否像同一个游戏，交互点是否一眼可识别，建筑占格是否接近。

### A2 骨架六区识别包

团队数：1 到 2 个。

任务：

- 六区只交识别级资源：地区母版、代表工坊、代表 NPC 头像或站姿、通道入口。
- 不画完整街区装饰，不补大型节令活动。

回头审查：必须。重点看六区是否有地区辨识度，是否比主轴五区明显更轻量但不断链。

### A3 NPC 与工艺补齐

团队数：2 到 4 个。

任务：

- 主轴五区导师、路线人、活动 NPC 优先出头像/半身/站姿。
- 主轴工艺优先出工具、材料、失败缺陷、返修状态。
- 工艺图先服务 UI 和活动说明，后续再升级为动作小游戏素材。

回头审查：需要。重点看人物职业道具和工艺工具是否对应真实玩法，而不是装饰。

### A4 UI 与活动视觉补齐

团队数：2 到 3 个。

任务：

- 五区活动摊位、账册、货样、封账/收灯/收摊视觉。
- 背包关键资源图标、订单图标、地区节点图标。
- 大地图底图可先出简版，不阻塞主轴。

回头审查：需要。重点看 UI 图标是否清楚，不抢正文、不混风格。

### A5 切图接入与编辑器 QA

团队数：1 到 2 个。

任务：

- 输出透明 PNG。
- 对齐占格、碰撞、底部锚点、文件名、路径。
- 在地图编辑器摆放，导出或截图验证。
- 记录哪些资源只是占位，哪些可进入首轮运行。

回头审查：必须。没有编辑器预览和资源清单，不算完成。

## 5. 每个团队的交付格式

每个团队每天只交六件事：

1. 地区或资源包名称。
2. 新增文件路径。
3. 对应的 regionId、npcId、craftId、activityId。
4. 可直接接入的 PNG。
5. 预览图或编辑器截图。
6. 剩余缺口。

禁止交付：

- 只画漂亮概念图，不列可切资源。
- 不按 ID 命名。
- 混用地区服饰、建筑或宗教符号。
- 一开始就画全 43 个小地区精修。
- 为了追求最终效果阻塞程序当前优先版。

## 6. 歧义记录

- 完工口径：按当前优先版，不按完整版。
- 主轴五区：必须优先画到可游玩识别。
- 骨架六区：只画识别包，不追求同等密度。
- 全景概念图：可作为风格来源，不等同于正式地区包。
- 运行切片：已有切片可复用，但需要统一 QA 后才能算正式交付。
- 美术终版：最终季节、昼夜、全 NPC 立绘、全活动插画全部后置。
