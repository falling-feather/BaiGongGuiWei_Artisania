# 审核问题与决策记录

本文件供策划审核使用。审核通过后，再进入程序收口、编译、测试和浏览器验证。

## 1. 建议默认决策

| 问题 | 建议决策 | 理由 |
| --- | --- | --- |
| 当前优先版是否只做五区深度 | 是 | 可以把剩余非美术工作压回 0.5 到 1 倍当前工作量 |
| 其余六区是否保留 | 保留骨架 | 保护完整版世界观，也避免美术组误以为世界被砍掉 |
| 总览地图是否允许正式迁移 | 不允许 | 正式体验必须通过街景通道、驿站、码头、商路入口 |

| 是否现在要求最终美术资源 | 不要求 | 只锁资源 key、目录、角色和交互点需求 |
| 是否现在开始编译和程序深改 | 不开始 | 先等策划审核确认范围 |

## 2. 需要审核确认的问题

1. 五个主轴地区是否合适：江南、巴蜀、岭南、赣鄱、西域。
2. 黔滇、荆楚、徽州、京畿、三晋、雪域是否接受本轮只做骨架保留。
3. 当前优先版是否以“可对接美术组”为第一交付目标，而不是以“玩家可长期游玩几十小时”为目标。
4. 美术组第一轮是否按“主轴五区完整包 + 六区识别骨架包”启动。
5. 程序组下一阶段是否严格禁止新增大系统，只修断点、补入口、补测试、补数据守卫。

## 3. 可替代方案

### 方案 A：五区主轴

这是当前推荐方案。优点是世界仍大，主体验可收口，美术组有清晰优先级。缺点是六个骨架区短期不会有同等生活密度。

### 方案 B：三区精品

只做江南、岭南、赣鄱三个地区，把巴蜀和西域也压成骨架。优点是更快，缺点是商路和远行感会明显不足，不太像大型端游世界。

### 方案 C：十一州等量

继续按完整版推进 11 区等量内容。优点是最符合终局野心，缺点是当前阶段会显著拖慢美术对接，剩余工作量很可能超过当前已完成量的 2 倍。

## 4. 审核后的动作

审核确认后，程序阶段按以下顺序启动：

1. 把本目录文档作为当前冲刺唯一范围基准。
2. 更新原交接文档中的“当前优先版”引用。
3. 只补主轴五区和骨架六区断点，不再扩大全地区内容密度。
4. 运行类型检查、测试、构建和浏览器烟测。
5. 输出给美术组的稳定 ID、资产 key 和地区包清单。
