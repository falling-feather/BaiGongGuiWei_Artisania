# 美术组 NPC 交互框分区重构记录

日期：2026-06-17

## 本轮目标

- 修正 NPC 交互首幕与二级页没有按图像 UI 框分区使用的问题。
- 确保半身像固定放入左上绿色人物框，人物身份与基础信息固定放入右上信息框。
- 将人物对白、页签入口与具体交互内容统一收束到下方内容框，避免文字和按钮压到框线外。

## 实装调整

- 接入文件：`src/components/NpcDialogModal.tsx`
- 样式文件：`src/index.css`
- 使用资源：`public/assets/game/ui/hud_v2_dialogue_panel.png`、`public/assets/game/ui/hud_v2_choice_button.png`

本轮将 NPC 交互面板从“整块内容叠在图像框上”改为三槽位布局：

- 左上头像槽：只承载当前 NPC 的 `bust.png`，失败后继续走 `portrait.png` 与同画风 fallback 链。
- 右上资料槽：显示身份、姓名、关系阶段、交谈次数、性格、好感度与知识标签。
- 下方交互槽：显示对白、`交流 / 送礼 / 互动 / 委托` 页签、当前模式内容与告辞按钮。

二级页不再使用白色大弹窗，也不再把人物半身像塞到底部。内容量较大的委托、赠礼、地方见闻改为在下方框内局部滚动，画卷框架本身保持稳定。

## 适配说明

- 桌面端按 `735x480` 图像母版的比例坐标落位。
- 窄屏端不改变图像框分区比例，仅压缩字号、按钮高度和按钮列数，避免再次出现右侧画面被裁切、人物图压出框的问题。
- 二级页底部移除重复的“返回对白”按钮，保留右上返回入口和底部告辞动作，给下方内容框释放空间。
