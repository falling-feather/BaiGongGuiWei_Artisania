# 程序端快速收束记录

更新时间：2026-06-14

## 本轮目标

文档设计接近完成后，程序侧优先把“当前优先版主轴”落为可运行、可测试、可在 HUD 中看到的目标链，保证后续美术接入时不会只剩散落系统而缺少玩家下一步指引。

## 已实装

1. 新增 `src/data/priorityJourney.ts`
   - 定义五个优先锚点地区的主轴步骤：江南、巴蜀、岭南、赣鄱、西域。
   - 每个地区拆为两个里程碑：代表工艺样品完成、代表节会/摊位链闭环。
   - 所有里程碑绑定真实工艺 ID、活动 ID、完成 flag 与百科目标词条。

2. 新增 `src/engine/priorityJourney.ts`
   - 提供纯函数 `buildPriorityJourneyGuide`。
   - 根据 `GameState` 自动判断当前未完成 step/milestone。
   - 复用现有百科行脚路线引导，只提示“怎么到目标附近”，不提供地图直接迁移。

3. 接入 `src/components/Hud.tsx`
   - HUD 常驻显示“主轴目标 x/5”。
   - 玩家主动追踪百科词条时，原有“行脚目标”仍然保留。
   - 主轴目标和百科追踪互不抢占，便于后续调试与正式游玩并行。

4. 补充 `src/index.css`
   - 新增 `.hud--priority-guide`。
   - 位置高于原有行脚目标，避免与控制提示和百科追踪重叠。

5. 补充自动化守卫
   - `src/engine/__tests__/priorityJourney.test.ts`
   - `src/data/__tests__/priorityScope.test.ts`
   - 覆盖主轴目标推进、里程碑完成条件、数据 ID 断链检查。

6. 新增当前优先版程序巡检
   - `src/engine/priorityScopeAudit.ts`
   - `src/engine/__tests__/priorityScopeAudit.test.ts`
   - 把主轴五区里程碑、主轴资料齐备度和骨架六区资料齐备度输出为可测试结构。

7. 接入镇务收束入口
   - `src/components/RegionPanel.tsx` 新增“当前优先版收束巡检”与“本地工坊”。
   - 本地工坊只开放当前小地区已有工艺，不触发迁移，不替代街景 gate。
   - `coin`、`labor` 在镇务和工艺页展示为“通货”“工时”。

8. 新增主轴活动街景入口守卫
   - `src/game/__tests__/priorityActivityEntrypoints.test.ts` 覆盖江南、巴蜀、岭南、赣鄱、西域五个主轴活动。
   - 守卫目标小地区必须暴露对应活动点。
   - 守卫活动里程碑的导航只能指向真实 `subregionGate`，不新增总览地图直跳。

9. 补齐活动收束入口互锁
   - `src/components/ActivityModal.tsx` 在摊位收束未决时禁用“完成体验”。
   - `src/components/RegionPanel.tsx` 在同一状态下禁用镇务本地活动入口。
   - `src/game/__tests__/activityUiEntrypoints.test.ts` 固化 UI 入口与 reducer 规则一致。
