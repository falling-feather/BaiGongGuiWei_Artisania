# 审核后程序收口任务池

## 1. 使用说明

本文件不是当前阶段要立即执行的开发任务，而是策划审核通过后的程序收口清单。当前阶段只确认任务是否必要、顺序是否合理、范围是否仍保持在 0.5 到 1 倍剩余工作量内。

程序阶段启动前，以 `06_审核问题与决策记录.md` 的审核结论为准。

## 2. 任务分级

| 等级 | 含义 |
| --- | --- |
| P0 | 不做就无法交付当前优先版 |
| P1 | 影响主轴五区完整体验 |
| P2 | 影响骨架六区世界完整度 |
| P3 | 可作为审核后第二轮增强 |

| 任务 | 目标 | 产出 |
| --- | --- | --- |

| 地图迁移守卫复核 | 总览地图只做调试和提示，正式迁移只走街景通道 | 2026-06-14 已新增路线可达与布局 gate 守卫 |

## 6. P3 后置增强

以下内容审核通过后也不进入第一轮程序收口：

1. 全旗舰工艺独立实时小游戏。
2. 全 43 个小地区手工地图点位。
3. 全地区等量节令活动。
4. 全 NPC 长篇人物线。
5. 家园装修、自动化经营和深度农事。
6. 联网、云存档、排行榜、异步社交。
7. 最终经济平衡和数值大规模调参。

## 7. 程序阶段验收顺序

审核通过后，建议程序阶段按以下顺序执行：

1. 文档入口和 ID 冻结表落地。
2. 主轴五区数据断点修补。
3. 地图迁移守卫和街景入口复核。
4. 工艺、订单、NPC、百工志文本入库。
5. 骨架六区断点修补。
6. 自动化测试与浏览器烟测。
7. 输出美术组资源 key 清单。

## 8. 不进入程序阶段的条件

只要以下问题未审核确认，就不进入程序深改：

1. 主轴五区是否最终确定。
2. 骨架六区是否接受当前密度。
3. 美术组第一轮是否接受“主轴五区完整包 + 六区识别骨架包”。
4. 总览地图是否明确只作为调试/提示。
5. 工艺实时小游戏是否明确后置。

## 9. 交付判断

当前优先版程序收口完成后，应能交付：

1. 一个可从新档跑到阶段性结局的生活模拟纵切。
2. 五个主轴地区的完整内容体验。
3. 六个骨架地区的世界存在感和运行入口。
4. 稳定的美术资源 key 与地区包需求。
5. 明确的完整版回收路线。
