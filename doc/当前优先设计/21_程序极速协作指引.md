# 程序极速协作指引

本文只服务当前优先版：主轴五区完整纵切 + 骨架六区不断链。不按完整版、联网版或最终美术版扩大范围。

## 1. 极简进度

- 当前不是早期 demo：新档、存档、街景、采集、精炼、制作、缺陷、返修、订单、NPC、活动、商路、结局和数据守卫都已进入可运行链路。
- 当前优先版程序完成度约 70% 到 76%。
- 基线已通过：`npm run typecheck`、`npm test`、`npm run build`。当前测试为 20 个文件、237 项用例。
- 已有五个主轴地区：江南、巴蜀、岭南、赣鄱、西域。
- 已有六个骨架地区：黔滇、荆楚、徽州、京畿、三晋、雪域。
- 正式移动红线已明确：总览地图只做提示和调试，迁移必须走街景通道、商路入口或交互点。

## 2. 当前缺口

P0 缺口：

- 主轴五区还需要一次完整深流程复验：街景入口、活动、工艺、订单、NPC 反馈、结局回声。
- 跨地区 gate 和商路入口需要继续浏览器复验，防止只在测试层成立。
- 骨架六区需要确认全部可进入、可检索、可显示资料卡入口、可被结局读取。
- 新增内容必须继续补 ID 守卫，不能出现只写文案、没有运行入口的表项。

P1 缺口：

- 工艺仍偏规则选择，需要继续强化阶段诊断、师傅反馈、返修、订单拒收和 UI 提示。
- 人工地图 JSON 只完成代表小地区样板，43 个小地区精确点位和动线尚未收口。
- 新手引导、失败提示、无材料、无订单、背包满、旧档迁移异常等状态还不够。
- 长局经济压力测试不足：金币、库存、声望、路线稳定、工时、维护费需要跑模拟。

后置缺口：

- 联网、云存档、排行榜、异步社交不进当前优先版。
- 每门旗舰工艺的实时动作小游戏不进当前优先版，只保留可承接的规则字段。
- 全 43 个小地区最终美化和完整季节变体不进当前优先版。

## 3. 最小团队分配基数

最小有效并行基数：7 个程序团队。

| 团队 | 任务 | 主要输出 |
| --- | --- | --- |
| P0 集成守门 | 统一范围、审查 PR、守住红线 | 每轮合并清单、风险记录 |
| P1 江南主轴 | 新档核心循环、龙泉/灯市/售卖复验 | 测试 + 浏览器烟测记录 |
| P2 巴蜀主轴 | 茶马会、蜀锦/竹编、茶马路线 | 数据补线 + 链路测试 |
| P3 岭南主轴 | 骑楼夜市、香云纱、船期订单 | 数据补线 + 链路测试 |
| P4 赣鄱主轴 | 开窑会、景德镇瓷、复烧回访 | 数据补线 + 链路测试 |
| P5 西域主轴 | 绿洲巴扎、玉作、商队远行 | 数据补线 + 链路测试 |
| P6 骨架与 QA | 六区骨架、ID 守卫、长局压力 | 覆盖矩阵 + 缺口清单 |

极速扩容时再加 3 个团队：

- 地图团队：专门处理街景 gate、人工 JSON、通道、碰撞、导航目标。
- 工艺团队：专门处理阶段诊断、返修、师傅反馈、订单读取。
- 浏览器 QA 团队：专门跑新档、继续档、跨区、活动、结局和控制台检查。

## 4. 项目节点

### N0 基线锁定

团队数：1 到 2 个。

任务：

- 跑 `typecheck`、`test`、`build`。
- 记录当前未提交文件和临时日志，不做回退。
- 确认当前优先版范围、ID、五区主轴、六区骨架不变。

回头审查：必须。没有绿基线不得开并行任务。

### N1 主轴五区深流程

团队数：5 个主轴团队 + 1 个 QA 团队。

任务：

- 每个主轴地区补齐一条可跑链：进入地区 -> 找入口 -> 做工艺或活动 -> 产生订单/关系/声望 -> 结局可读。
- 每个地区至少补一个自动化守卫或 reducer 链路测试。
- 每个地区至少补一条浏览器烟测记录。

回头审查：必须。五区完成后统一审查入口命名、活动结构、反馈文本和结局读取是否一致。

### N2 骨架六区不断链

团队数：2 个骨架团队 + 1 个 QA 团队。

任务：

- 六区只补最低闭环：小地区、代表工艺、NPC、活动或订单入口、资料卡、结局段落。
- 不追加大型节令链，不追求主轴五区同等密度。

回头审查：必须。重点看六区是否都能被移动系统、百工志、订单/NPC 或结局引用。

### N3 工艺与反馈收束

团队数：2 到 3 个。

任务：

- 统一阶段诊断、缺陷来源、返修履历、订单拒收、NPC 鉴评、师傅复核的显示和测试。
- 补无材料、工时不足、品相不足、重缺陷、返修后再交付等失败路径。

回头审查：需要。重点看工艺系统没有变成孤立小游戏，也没有只剩合成按钮。

### N4 地图与移动收束

团队数：2 到 3 个。

任务：

- 复验主轴五区和骨架六区街景入口。
- 守住总览地图不得正式迁移。
- 补人工 JSON、通道、活动点、NPC 点、资源点的引用守卫。

回头审查：必须。重点看是否有任何 UI、资料卡或面板绕过街景通道。

### N5 交付前压力测试

团队数：2 到 4 个。

任务：

- 新档最短主轴路线。
- 继续档跨区路线。
- 活动三阶段 -> 收束选择 -> 后续订单。
- 工艺缺陷 -> 返修 -> NPC 鉴评/订单交付。
- 结局报告读取主轴五区和骨架六区。
- 长局库存、金币、声望、工时、维护费基础压力。

回头审查：必须。通过后才进入美术替换和点位微调。

## 5. 每个团队的交付格式

每个团队每天只交五件事：

1. 完成了哪条可运行链路。
2. 改了哪些文件。
3. 加了哪些测试或烟测记录。
4. 发现了哪些缺口。
5. 是否触碰红线或产生新 ID。

禁止交付：

- 只写 Markdown，不接数据或入口。
- 只加数据，不加守卫。
- 从总览地图直接迁移。
- 重命名已冻结 ID。
- 把最终美术缺口当程序阻塞。

## 6. 歧义记录

- 完工口径：按当前优先版，不按完整版。
- 美术资源：最终资源不阻塞程序收束，但资源 key 和运行入口必须稳定。
- 实时小游戏：当前只要求规则字段和可承接 UI，不要求完成动作小游戏。
- 全地区密度：主轴五区做完整体验，骨架六区只做不断链。
- 后端能力：账号、云存档、异步社交、排行榜全部后置。
