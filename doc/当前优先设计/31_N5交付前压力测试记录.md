# N5 交付前压力测试记录
日期：2026-06-14

## 节点目标

N5 用于在进入美术替换与点位微调前，回头验证当前优先版是否已经具备一条完整、可跑通、可回归的非美术交付链。

本节点重点不是继续扩内容，而是压测以下闭环：

1. 新档最短主轴路线能从江南一路推进到西域驼队收束。
2. 继续档能按街景 gate / subregionGate 跨区移动，而不是依赖总览地图直跳。
3. 五个主轴活动均能完成三阶段摆摊、处理收束选择，并生成可交付后续订单。
4. 工艺缺陷、返修、NPC 鉴评、订单拒收和返修后交付能跑通。
5. 结局报告能在同一局读取五个主轴区和六个骨架区。
6. 长局经济能承受库存、金币、声望、工时、维护费和商路断供压力。

## 协作审计

按 `21_程序极速协作指引.md` 的 N5 要求启用 4 个只读审计团队：

| 团队 | 审计范围 | 结论与处理 |
| --- | --- | --- |
| 路线/存档团队 | 新档主轴、继续档跨区、街景 gate 链 | 发现既有测试更多使用 `placeAt` 场景定位，缺少继续档跨区路线守卫；本节点新增继续档跨区压力测试。 |
| 活动订单团队 | 三阶段活动、收束选择、后续订单 | 发现主轴里程碑使用 `stall-chain-completed:*` 会早于收束选择推进；已改为 `stall-closing-resolved:*`。 |
| 工艺 NPC 团队 | 缺陷、返修、鉴评、订单拒收/交付 | 规则层已较完整；本节点新增集中压力测试覆盖缺陷剑样、鉴评、拒收、返修与交付。 |
| 结局经济 QA | 结局报告、长局经济、维护费、断供 | 发现同局结局报告只取 top 3 地区；已取消该截断并用 11 区同局报告守卫。 |

## 程序补线

1. 新增 `src/engine/__tests__/priorityPreShipStress.test.ts`。
   - 覆盖新档主轴最短路线。
   - 覆盖继续档跨区解锁与街景路线迁移。
   - 覆盖五个主轴活动的收束选择和后续订单交付。
   - 覆盖缺陷作品 -> NPC 鉴评 -> 订单拒收 -> 返修 -> 订单交付。
   - 覆盖同一结局报告读取 11 个当前优先地区。
   - 覆盖长局经济、维护费、商路断供和后续稳定。
2. 修改 `src/data/priorityJourney.ts`。
   - 主轴活动里程碑由 `stall-chain-completed:*` 改为 `stall-closing-resolved:*`。
   - 摆摊三阶段完成仍保留事实标记，但主轴推进必须等待收束选择。
3. 修改 `src/engine/reducer.ts`。
   - `buildRegionalOutcomes` 不再只取前三个地区。
   - 当前优先版同局结局报告可以完整列出五个主轴区与六个骨架区的地区回声。
4. 同步测试夹具。
   - 更新 `priorityJourney`、`priorityScopeAudit`、`priorityActivityEntrypoints`、`priorityJourneyNavigation`、`regionSpec` 等测试中的主轴完成前置条件。

## 当前判断

N5 已将当前优先版的非美术交付链推进到可回归状态。程序侧剩余内容主要转入：

1. 原当前优先版口径下的美术资源替换。
2. 已有 22 张当前优先人工街景的点位微调。
3. 非优先 21 个小地区的表现层细化。
4. 按 2026-06-14 最新推进口径，程序侧不等待美术替换，转入 `doc/完整版设计/程序大型化协作指引.md` 的 M0.9 全地区生活规格入轨节点。

总览地图仍只作为调试和路线提示入口；正式移动链路继续以街景中的 gate、subregionGate 和交互点为准。
